Elite – Store, District, Region, and Company Security Settings



 

Overview

This article will cover information on Store, District, Region, and Company Settings in the Elite Software. There are currently no reports that represents these areas

 

 

Topics in this article include:

  • Create a new District or Region
  • Assigning Districts to Regions and Stores to Districts
  • Security Settings
  • Browser Setting Controls
  • Transactions Controls - Overbooking
  • Transaction Controls - Contracts
  • Examples
  • If all levels are unchecked



 

There are several settings in different parts of the application that control options at these levels. The structure is based on:
 

  • Store Relationship to the Operator (primary)
  • The relationship of those stores to Districts and Regions (secondary)

 
 

Some important :
 

  • A company is a collection of regions, districts, and stores.
  • A region is a collection of districts and stores.
  • A district is a collection of stores.
  • Core Reports will not represent the Different Districts, Regions. This goes by stores. 

 
 

 

Create a new District or Region
 

Districts, Regions, and their relationships to each other and Stores are managed in Store Groups.

Here's how to get there
Elite: Configuration Gear > App Configuration > System Configurations > Store Groups


 

1. Select the appropriate radio button to display the option you want to add.

 

2. Then, double-click in the next empty row and enter the name.

 

3. To remove, right-click and select Delete. 

 

 
 

Assigning Districts to Regions and Stores to Districts
 

From the District or Store areas, simply select the drop-down to the right of the Store or District you want to assign and select the option you want. You can select the empty space to remove the assignment.

 

 
 

Security Settings
 

The security settings control the ‘Store’ drop-down options. If you are searching, creating contracts, or filtering data and there is a “store’ selector, it can be controlled with these security options.


Here's how to get there

Elite: Configuration Gear > System > Security


 

Browser Setting Controls
 

BI Dashboards 

 In dashboard filters, a user can view the District and any District stores the user is related to. This is only refering to the Dashboards in the Elite Interface. 

Item Search 

 In item search filters, a user can view the District and any District stores the user is related to.

Search 

 In search filters, a user can view the District and any District stores the user is related to.

 

The drop-downs that are controlled by the Elite Settings options allow the user to select a District, Region, or Company. When this is done, you are really selecting the underlying (related) stores.

When all transaction options are turned off for a particular contract type, the Add or Create option will be hidden. With any one of them turned on, you will have the Add or Modify option available, but your options in the Store drop-down will be limited to the level you have been set at.


 

Examples
 

You could set a user to be turned on only at the District level to Create an Internal Repair. This means they could only create an Internal Repair for the stores in their District. But, you could have them set to Company for Internal Service contracts which means they could create an Internal Service for any location in the company.
 

This graphic representation of a company structure will be used as a reference in the examples below. This is a company with 12 locations organized into 4 Districts and 3 Regions. The user has related stores: 001 and 010.

 


 

Example of a Company
 

If they have Company selected, this user will be able to choose any store, district, or region in the company.

 

 

 

Example of Region
 

- If they have Region selected (and not Company), this user will be able to select:
 

- No Company (All Stores) option
 

- Regions 1 and 3
 

- Districts 1, 3, or 4
 

- Stores 001 thru 003 and 007 thru 012
 

- They will not have access to stores 004 thru 006, District 2 or Region 2 because they do not have a relationship with any stores in Region 2.

 

 

Example of District
 

- If they have District selected (but not Region or Company), this user will be able to select:
 

- No Regions or Company (All Stores) options
 

- Districts 1 and 4
 

- Stores 001 thru 003 and 010 thru 012.
 

- They will not have access to stores 004 thru 009 or Districts 2 and 3 because they do not have a relationship with any stores in District 2 or 3.

 

 

Example of Store
 

If they have Store selected (but not District, Region or Company), this user will be able to select:
 

- No District, Region or Company (All Stores) options.
 

- Stores 001 or 010.
 


 

If all levels are unchecked

 

For Item Search and View security: There will be no results shown.
 

For Search and View security: It will always show results for the operator-related store(s) but they will not be able to filter them to any particular location(s).

 

For Bi_Dashboard security: The data will filter to all related stores with no option to filter to any particular store(s).

 

There is a built-in hierarchy. The highest level controls the rest. If you have Store and Region selected and not District, you will get Store, District, and Region. If you only have Company selected, you will get all levels.
