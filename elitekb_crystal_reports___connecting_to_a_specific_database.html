Crystal Reports: Connecting to a Specific Database



Overview

This document is intended to instruct users how to connect to a specific tab in Crystal Reports. We do have customers, a very small amount, that call into POR that need assistance connecting to the POR database in Crystal Reports.


Prerequisite

The POWERBI_USERS security group needs to be created, and the appropriate users added to the group. This will give the customer read-only rights to connect to the database. Details of this are in the PowerBI setup guide and are handled by the POR IT Department.

Connecting to the Database
 

1. Open Crystal Reports.

 

2. In the 'Field Explorer' on the right hand side of the screen, right click 'Database Fields.' Select 'Database Expert.'




Note: You can also select the 'Database' tab at the top of the screen, then select 'Database Expert.'
 

3. In the 'Database Expert' window, select the SQL instance from the list. If the instance is not listed, select the '+' to the left of the 'New Connection' folder. See the additional steps below for new connections, otherwise proceed to Step 4.
 

a. Select the '+' next to the 'OLD DB (ADO)' option.
 


 

b. Select the 'Microsoft OLD DB Driver for SQL Server' option from the 'Provider' list.
 

c. Select 'Next.'
 

4. Enter the server name/IP address and the port the SQL server is using in the 'Server:' field. The format is [servername/IPaddress,portnumber]. Remove the brackets.

5. Enter the username to connect with in the 'User:' field. This should already be set up through the POWERBI_USERS security group. If not, please reach out to the IT Department for assistance with the setup of said security group.

6. Enter the password in the 'Password:' field. This should be the Windows password for the user in question.

7. Select 'Finish.'
