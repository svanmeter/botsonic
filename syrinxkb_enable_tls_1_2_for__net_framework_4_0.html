Enabling TLS 1.2 for .Net Framework 4.0


Overview

Some websites and external servers are recently changing their security requirements to only accept SSL connections using protocol TLS 1.2.  Most systems running .Net Framework 4.0 do not have the option enabled to use TLS 1.2 and will default to using TLS 1.0 and their secure connection attempts to external servers can be rejected.  If you are currently experiencing an issue with a Syrinx application such as the Transfer tool or Report Scheduler failing to connect to an external server due to a change in the security requirements for that server, the below process may resolve your issue.
 

Install .Net Framework 4.5 or Higher

In order to enable Net Framework 4.0 to use TLS 1.2, you must download and install .Net Framework 4.5 or higher.  The below link from Microsoft explains how you can check what version of the .Net Framework you currently have installed:

 

https://docs.microsoft.com/en-us/dotnet/framework/migration-guide/how-to-determine-which-versions-are-installed

 

If .Net Framework version 4.5 or later is not installed on your PC, these are available as free downloads from Microsoft.  Links to several of the later versions of the .Net Framework are available below:

https://www.microsoft.com/en-gb/download/details.aspx?id=42642

https://dotnet.microsoft.com/en-us/download/dotnet-framework/net48

 

*You only need to install one of the above versions
