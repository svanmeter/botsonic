INTERNAL eCommerce | Website troubleshooting



This article will go over troubleshooting steps for when websites are not updating.


Topics included in this article
 

  • Reasons a customer may call in
  • How do I confirm a website is not updating?
  • Troubleshooting




Reasons a customer may call in
 

  • Price changes are not showing on website
  • New pictures are not showing on website  (check for browser refresh first!)
  • New items are not showing on website
  • Web quotes are not showing in the web quote import window



How do I confirm a website is not updating? 

The best way to confirm that a site is not updating is to check the time/date stamp at the very bottom of the website. Each website has a scheduled task that runs each night; if it is successful, the time/date stamp will be up-to-date. If it is not, then the website is not updating.
 

The most common reason that a website stops updating is that the customer's IP address changes. The first thing to ask them is for their current IP address.



Troubleshooting
 

Step 1

Ask the customer to go to porhelp.com and click on the green "Show Me My IP Address" button, then send/tell you the resulting IP address.  


If the store's static IP address has changed, go to porhelp.com from a POR Terminal, click on the "Show My IP Address" button, and copy and paste the new IP address into a message to support@rentalhosting.com with the subject title "Change in Static IP address.
 

Only for Rental Hosting/IT team members

  • Search our firewall to see if the IP is already there for the customer. If it is not, enter it into the firewall and communicate to the customer to try running the Update Website process. If it works, then the problem is solved.


If the Update did not work after the IP address was added to the firewall, proceed to Step 2.


If the IP address is already in our firewall, proceed to Step 3.


 

Step 2
 

Ask for a TeamViewer session and view the log file in the .../POR/web/logs folder on their server. See if you can discern what might be the issue. If you can, proceed as necessary to fix the issue. If not, create a case for POR Support, being sure to let them know that the IP address is good.

 

Step 3

 

It could be that the store never runs the Update manually, and it's the Scheduled Task that isn't running.  Ask for a TeamViewer connection and check it. You'll find the Task Runner on the Configuration > System Configurations menu. Find the Website Update task and be sure it's active. Check the log window to see the last date it ran. See Customer Support doc "Scheduled Task Runner" for more information or check with other eCommerce team members for more information.


Reasons why the Update could stop running properly, aside from a changed IP address:

  • The store is running Mobile Workforce and POR recently re-configured MW over the Update.
  • The store recently moved or upgraded their server and tasks weren't properly finished.
  • The store did a Windows update or updated their firewall, and ports 20 & 21 are now blocked. 
  • Their scheduled task has been tinkered with.


Any of these reasons would require help from POR Support.
