Change Posting Month of a Contract

 

 

 

 

Overview

 

Occasionally, there may be a situation in which a contract that was meant to be closed at the end of one month is closed at the beginning of another. If this occurs, the posting can be moved from the current month to the last closing day of the previous month. You can only do this for contracts posted during the current month. For example, if it is November, you can only move contracts posted in November back to the last day there was a closing in October.  Once you reach December, that opportunity is lost, and you can now only move contracts posted in December back to the last closing day in November. 

 

 

Changing the posting month of a contract

 

1. To do this, access the Transaction File through File Maintenance.

 

 

2. Once you have selected the transaction file, go to the Functions tab in the top left corner. The last option in that drop-down list is “Change Posting Date.”

 

 

 

3. Once you select this, you will see a message confirming that you want to change the posting date on the contract. Select yes.

 

 

 

4. The contract will repost for the last closing day of the previous month. Both of those days will need to be reimported into your accounting software (if previously entered). When you run your end of the day for the current day, you will get the correcting G/L entries.
