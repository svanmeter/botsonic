SyrinxPDF Troubleshooting


 

Overview

Syrinx uses a version of the CutePDF “Custom PDF Writer” redistributable rebranded as SyrinxPDF in order to output Syrinx documents/reports in PDF format.  At times, this PDF writer can suffer from issues on certain PCs that prevent the printing of PDFs from Syrinx due to a variety of causes ranging from but not limited to installation of other PDF writers, issues in the PC’s printer subsystem, issues with specific printer objects installed on the PC, running registry cleaning software such as CCleaner on the PC, over-aggressive anti-virus, etc.  This guide will aim to offer suggestions on possible actions to take when a PC is encountering issues printing to PDF via SyrinxPDF.

 

Is This a SyrinxPDF Issue?

In most cases users will experience an issue with SyrinxPDF when emailing documents from Syrinx.  In order to determine if this is genuinely an issue with SyrinxPDF or other Syrinx objects related to printing, you should take emailing out of the equation and just try to print the document to PDF using the “Export PDF” button on the Syrinx Print Preview window.   If this is successful and the PDF file is produced and formatted correctly, this is not an issue with SyrinxPDF and is most likely an issue with Syrinx interfacing with Outlook or an SMTP server if using CDO for emailing documents.  If exporting the document to PDF is not successful or the resulting PDF is not formatted correctly, there is potentially an issue with SyrinxPDF, one of its components such as Ghostscript or a PDF overlay linked to the Syrinx system.  You should also attempt to print other Syrinx documents to PDF in order to determine the scope of the problem as some issues such as a corrupt SyrinxPDF installation will prevent printing anything to PDF whereas some issues may only manifest themselves on certain documents/reports.  The following Sections will detail various SyrinxPDF issues and their possible solutions.

 

System Hangs When Printing to PDF

If the system hangs or freezes when printing to SyrinxPDF this is normally an indication that there is an issue in the printer subsystem on the PC and can be caused by problems with the print spooler or corrupt printer objects that exist on the PC.  You can check the printer objects installed on the PC in “Devices and Printers” by right clicking on each entry and selecting “Printer Properties”.  If this page fails to load or causes an error then delete the printer object if the user no longer requires it or have the user contact their IT to fix the issue with that printer.  You should also consider deleting any printer objects that appear as unavailable and that are not required by the user but always ask permission first before removing anything from the user’s system.  Once either of those actions have been taken, try printing to SyrinxPDF.  If all printer objects appear OK, check the print queues for the printer objects installed on the PC by double clicking on the printer objects in “Devices and Printers” and see if there are any documents stuck in the print queues.  If yes, restarting the print spooler in the windows “Services” menu should clear those queues and you should try printing the document again.  If SyrinxPDF is still hanging then check in Windows eventvwr for any application/system errors that occurred at the same time that you were trying to print to PDF.  In most cases these errors will be linked to the splwow64.exe process which is the 32 bit interface into a 64 bit machine’s printer subsystem.  These issues should generally be handled by the user’s IT but a google search may return simple solutions/fixes for specific errors/events.

 

Large White Space in Header or Other Sections of the PDF Document 

If Syrinx is able to produce a PDF file but that PDF file has a large white space overlaying the header or other sections of the document this is normally an issue with the Ghostscript installation on that PC.  You should rename the folder containing the Ghostscript program files (C:\Program Files (x86)\GPLGS) and then reinstall SyrinxPDF which will add this back in automatically or download and install Ghostscript 9.1 from our download site:

http://www.syrinxusers.co.uk:443/Non-Syrinx%20Client%20Files/gs910w32.exe

Or, download the latest 32bit version of Ghostscript from the Ghostscript site:

https://www.ghostscript.com/download/gsdnld.html

Any version should be compatible with SyrinxPDF.

 

PDF Overlay Not Appearing on Document

If a PDF overlay has been set up in the system, the file is available to the user at a windows level and the document that they are printing is set to not display header information in “Document Header Setup” (i.e.: document should use the PDF overlay) and the document is not including the PDF overlay when printed to PDF, this is generally an issue with the format of the PDF overlay.   Some PDF writing software will create the PDF in a format that is not recognised by SyrinxPDF.  When this happens, you should grab a copy of the overlay (either linked to depot or in user options) and link it to your demo system and see if you can recreate the issue.  If you can see the problem on your own system then opening the PDF overlay in a pdf viewer and printing it through a different pdf writer such as CutePDF or the “Microsoft Print to PDF” printer and then using this new copy of the overlay should sort out the problem.

 

Odd Lines, Symbols or Colour Bars Appearing in the Corners of the PDF Document

This will generally happen if someone has printed a PDF directly from a graphic designer’s copy of a file or image and has printed the document with “markups”.  When this happens, you should grab a copy of the overlay (either linked to depot or in user options) and link it to your demo system and see if you can recreate the issue.  If you can see the problem on your own system then opening the PDF overlay in a pdf viewer and try printing it and see if there is a “Print What” option where you can select “Document” instead of “Document and Markups” (I am using Foxit PDF reader below):

Then, finish printing a new version and test on your system to see if it has resolved the problem.  If it does, replace the customer’s copy with this new copy of the overlay.

 

Pages on the PDF are Printed on Top of Each Other

If the text from one page of the PDF looks as though it has been printed on top of another page of the document (e.g.: The print preview shows 2 pages but the PDF is 1 page and looks as though page 1 has been printed on top of page 2), this is normally an issue with the PDF overlay.  As with other overlay issues you will want to grab a copy of the overlay and try opening the PDF overlay in a pdf viewer and printing it through a different pdf writer such as CutePDF or the “Microsoft Print to PDF” printer and then using this new copy of the overlay should sort out the problem.

 

Error Message During Printing and No PDF File is Produced

If the user is receiving error messages similar but not limited to the below (some details of the message may be slightly different) when printing to SyrinxPDF and no PDF file is produced, there can be a variety of causes and solutions to the problem:

 

ERROR -- 2147024809: The expression you entered refers to an object that is closed or doesn't exist (THT02A_DM)

Location lbMenuEmailDialog(line20)

 

ERROR 53: File not Found.

Location: THA70.cmdEmailInvoices_Click

 

ERROR-75:  Path/File access error.
Location: THA70.cmdEmailInvoices_Click(line 72)

 

**Before uninstalling anything verify if you are working on a terminal server or not and, if you are, whether or not the issue is affecting all users or just a specific user.  We do not want to potentially break a system that is working for the majority of users.  If the situation merits a reinstall on a terminal server, as the uninstall/reinstall of SyrinxPDF normally requires computer restarts, this work is best left to out of hours on a terminal server.

 

In this scenario, we will want to do a few basic checks:

  1. Is SyrinxPDF installed and visible in “Programs and Features” and is there a SyrinxPDF printer object in “Devices and Printers”?  If there isn’t run the SyrinxPDF installer (Run as Admin) on the PC.
 
  1. Is Ghostscript installed on the PC?  If it was installed via the SyrinxPDF installer, you will not see an entry for it in “Programs and Features” so you can check and see if the directory C:\Program Files (x86)\GPLGS exists and if there are entries in the registry key HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\GPL Ghostscript\8.15 which point back to the C:\Program Files (x86)\GPLGS directory.  If it does not appear to be installed, run the SyrinxPDF installer (Run as Admin) on the PC as this will add it back in or download the Ghostscript installer from our download site and run that.
 
  1. Are all paths to overlays at depot and user option level valid and can the user access those files?  If not, set correct paths and verify that the user can access those files via window explorer.
 
  1. When emailing from Invoice Batches, is the path to the “Save Folder” set in User Options\Batch Invoices tab correct and that the user can write to this directory.  If not, correct the path and set it to a location to which the user can write.
 
  1. Does the issue affect printing any document to PDF or just a specific document?  If the issue affects a specific document then this is normally not a SyrinxPDF issue and is generally an issue with whatever document they are printing which can be things like saved margins/printer settings in the Syrinx client for that document.  In that scenario, you can try running the Syrinx reinstall program to refresh the client (or replace the user’s adp on a terminal server) as printer settings for each document can be cached in the user’s adp file and replacing it manually or via the reinstall will get rid of these settings (along with any other document specific printer settings like label sizes, specific label printers, etc. so doing this might require certain things to be set back up in the user’s system).  Alternatively, if it is only crystal documents that are affected, you can try repairing the crystal runtime.
 
  1. Does PDF Printing work if Syrinx is set to “Run as Admin”?  If yes, then we are dealing with a permission issue and SyrinxPDF, when run under the user’s context, does not have enough permission to access the files and system locations that it needs to complete the PDF printing process.  Leaving Syrinx to “Run as Admin” is not a solution in this instance as it can create further problems such as Access not being able to link with Outlook because they are running under different user contexts but it can at least be used to determine the nature of the problem and as a temporary workaround.  The most common cause to this situation is that someone has run CCleaner on the PC and this has amended permissions on the C:\Windows\Temp folder so that Ghostscript can no longer write to it.  The solution in this case is to try creating an alternate temp directory for Ghostscript by adding a new “string value” to the registry key HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\GPL Ghostscript\8.15 and then assigning a folder path to that value (you need to create the folder in Windows too and it can be named anything as long as the registry value matches the folder name):
 

Alternatively, we have also seen situations where a user has had their permissions revoked on their own temp folder and this causes SyrinxPDF to fail as certain parts of the process will write to the user’s temp folder instead of the windows temp folder.  You can check this by navigating to the user’s temp folder, C:\Users\username\AppData\Local\Temp, and seeing if you can write to the directory.  If you can’t, then it should be handing back to the user’s IT people to fix the permissions on that folder.

 
  1. Has any new anti-virus software been installed on the PC?  You can normally check in “Programs and Features” and sort by the “Installed On” date and see if anything was installed recently which would correspond to the time when SyrinxPDF stopped working.  We have seen instances where a new anti-virus program has been overly aggressive and will either block SyrinxPDF from accessing parts of the system (The user normally gets a prompt about this but might not have been paying attention or chooses to ignore it) or is grabbing the temporary files that SyrinxPDF is creating in order to scan them and isn’t releasing them fast enough so that SyrinxPDF can continue with its work.  In that case, you can try disabling the anti-virus (some will still actively be working in the background even if they appear to have been disabled) and see if this makes a difference or uninstall the program as long as the user or their IT have the installation media to reinstall it.  If SyrinxPDF is now working then the call should be handed back to their IT in order to add in the appropriate anti-virus exceptions to allow SyrinxPDF to do its job.
 
  1. As with the “SyrinxPDF Hangs” issue mentioned at the beginning of the guide, you should check the printer objects installed on the PC in “Devices and Printers” by right clicking on each entry and selecting “Printer Properties”.  If this page fails to load or causes an error then delete the printer object if the user no longer requires it or have the user contact their IT to fix the issue with that printer.  You should also consider deleting any printer objects that appear as unavailable and that are not required by the user but always ask permission first before removing anything from the user’s system.  Once either of those actions have been taken, try printing to SyrinxPDF.  If all printer objects appear OK, check the print queues for the printer objects installed on the PC by double clicking on the printer objects in “Devices and Printers” and see if there are any documents stuck in the print queues.  If yes, restarting the print spooler in the windows “Services” menu should clear those queues and you should try printing the document again.
 
  1. Check windows eventvwr for any errors that are occurring at the same time that SyrinxPDF fails.  These issues should generally be handled by the user’s IT as they are generally windows system issues but a google search may return simple solutions/fixes for specific errors/events.
 

If you have tried all if the basic checks and there is still an issue, it is now time to try uninstalling/reinstalling SyrinxPDF.  When you do this you should always run the SyrinxPDF installer as an Administrator and make sure to restart the PC after the uninstall and then restart again after the reinstall.  This may seem like overkill but we have had instances where the restarts after each step were absolutely necessary so it is best practice to just do it as a matter of course even if not prompted to do so by the SyrinxPDF installer.

 

If there is still an issue after the reinstallation, the call should be escalated as it may now be necessary to run Process Monitor on the PC in order trace the exact calls that SyrinxPDF and its components are making to the operating system during the PDF printing process and try to identify where it is falling down.

 

SyrinxPDF Installation Fails

If SyrinxPDF fails to install or you receive an error message similar to "Install Printer Driver Failed" when installing SyrinxPDF, this is generally caused by SyrinxPDF files or drivers already being installed on the PC or anti-virus software blocking the installation.  When this happens, you should check in C:\Program Files (x86)\Higher Concept\SyrinxSP and verify if there are any files in this folder related to SyrinxPDF and delete them.  You should be careful when doing this  as there can be folders in this directory for the Syrinx Accounts Transfer program as well as the Syrinx Export program but these files are generally in their own folders and the SyrinxPDF files will be the only “files” in the root of that directory.  The SyrinxPDF files are highlighted in the below image of the C:\Program Files (x86)\Higher Concept\SyrinxSP directory:

 

 

The SyrinxPDF installer can also fail to overwrite existing versions of the SyrinxPDF drivers in the PC’s driver cache.  This can be resolved by manually removing the existing versions of SyrinxPDF driver files: CUTEPDFW.PPD, PS5UI.DLL, PSCRIPT.HLP, PSCRIPT.NTF, PSCRIPT5.DLL, from the directory C:\Windows\System32\spool\drivers\x64.

 

If SyrinxPDF is still failing to install you should see if the anti-virus is throwing any prompts/warnings when attempting to launch the installer or if there are any indications in the anti-virus’ logs that it has or is blocking an installation.  On some occasions, the anti-virus will no longer prompt that it is blocking an installation process once it has done so once already and the user may have failed to notice this was happening.  If you suspect that the install is being blocked, you can temporarily disable the anti-virus or uninstall it (if the user or their IT consents and has the installation media) and attempt the SyrinxPDF installation again.

 

Unable to Uninstall SyrinxPDF

If you receive any error messages or failures when uninstalling SyrinxPDF and the program is not removed from the PC, you can try to run a manual uninstall of Syrinx PDF using the executable unInstpw64.exe found in C:\Program Files (x86)\Higher Concept\SyrinxSP.  To do this, you should open a command prompt as an Administrator and run the following command:

 

"C:\Program Files (x86)\Higher Concept\SyrinxSP\unInstpw64.exe"  /uninstall
