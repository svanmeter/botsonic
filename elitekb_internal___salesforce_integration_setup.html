INTERNAL | Salesforce Integration Setup Guide


This article goes over the Salesforce integration setup for Elite. To set up a Salesforce integration, you'll need to set up Salesforce and then set up Point of Rental. This guide is only for setting up new Salesforce integrations. Some information in this guide may not be relevant to customers who have an older version of the integration.

Salesforce Setup

The first step is to set up Salesforce. You must be set up as a Salesforce System Administrator in the customer's Salesforce Organization in order to proceed. The Salesforce opportunity and the Point of Rental contract lifecycle are not the same. So before proceeding, the customer's Salesforce administrator needs to confirm their Salesforce Opportunity paths (path = process). 
 

High-Level Checklist

Below is a high-level checklist of the required setup within Salesforce. The bolded items are steps that can be performed or reviewed by Point of Rental. The other steps can only be performed by the customer's Salesforce administrator. 

  • Install Managed Package (create a new Connect App)
  • Assign Opportunity Compact Layout
  • Assign Opportunity Page Layout
  • Assign Opportunity Lightning Page Layout
  • Assign Product Page Layout
  • Assign Opportunity Product Page Layout
  • Configure POR Status to Salesforce Opportunity Stage mapping  


Install Managed Package

Below are the steps to install and create a new Connect App. For more information on configuring Connect Apps, see this Salesforce Article. This step can be completed by the customer.
 

  1. Go to the Managed Package Install Link: https://login.salesforce.com/packaging/installPackage.apexp?p0=04t3h0000043vfx (Updated Aug 14, 2023).
 
  1. After the App is installed, you'll get an email confirmation letting you know that the installation is complete.
 
 
  1. Once the installation is complete, you'll need to create a "New Connected App." To do this, go to Setup. In the Quick Find, search for "App Manager." For more information on configuring Connect Apps, see this Salesforce Article.
 
  1. Under the dropdown "API (Enable OAuth Settings)", check the box for "Enable OAuth Settings" and fill in the below information.
  • In "Callback URL" enter this URL: https://na.porwork.com/xeroauth/
  • Under "Selected OAuth Scopes", find "manage user data via APIs (api)" in the "Available OAuth Scopes" list and add (move) it to the right.
  • Check the "Require Secret for Web Server Flow" box.
  • Check the "Require Secret for Refresh Token Flow" box.


 
  1. Press Save to continue. 
Important! Once saved, the changes may take up to 10 minutes to update in the Salesforce Organization.
 
  1. After the Connect App is set up, you'll need to adjust the IP Relaxation for the Point of Rental app. To do so, go back to the Setup page and within the Quick Find, search for  "Manage Connected Apps." Then click on the App that you created from the App Manager.
    

 
  1. Next, click on "Edit Policies" at the top of the page.
 

 
  1. Update the IP Relaxation to "Relax IP Restrictions." 
 
  1. Click Save.
 

Configure POR Status to Salesforce Opportunity Stage mapping

      
This section will go over mapping Salesforce Opportunities to Point of Rental Contract statuses.

The Salesforce integration does not require Opportunity and Contract status mapping between Point of Rental and Salesforce. You can turn off this automation in the list of flows by deactivating PORElite: Set Opportunity Stage. This is important if the customer has their own automation to set the stage.
 
  1. Navigate to Setup Home
  1. In Quick Find, search for "Custom Metadata Types"
 

 
  1. By the record labeled "Opportunity Stage Configuration", select "Manage Records"

  1. Click the 'New' button to create a new Opportunity Stage Configuration
  2. Enter “Default” for both the Label and the Opportunity Stage Configuration Name. Then enter the Opportunity Stage Name that each POR Transaction Status should map to. If no value is entered for a given status, the stage will not change when an opportunity is moved to that status. By default, all opportunities come across as Closed Won. So if no values are entered, the Stage will remain in the Closed Won status.

    Important! The stage name must match exactly with the text entered in the text box. If it does not match, it will cause a sync error that prevents opportunities from syncing from POR to Salesforce.

 
  1. Click Save to save the changes.

  1. Go to \\porsrv1.us.corp.pointofrental.com\Download\Revision2023\@SalesforceSyncV4\POR\ and copy and paste all folders and files into the customer's local POR folder. 
Important! If the customer is in a Cloud environment, then the Information Technology Operations Team may need to get involved.[1]  
 
  1. Make a copy of the 3 executable files in the Program SaleforceSync folder. Go to each file's properties and update their "Start in" location to the customer's "Users" folder in POR. The Users folder is in the root of the POR folder. See the image below. 
 

 
  1. Once the files are copied, click on the Salesforce Sync icon. Once selected, the below window will open. Click on "Configure SFSync Parameters."

4. The Salesforce Login Settings window will open. Enter your Salesforce Login information. This information can be found in "Manage Apps" in Salesforce. It's recommended to use an integration account or the user account associated to all integrations.

Salesforce Key and Secret

 
  1. You can the Salesforce Consumer Key and Secret from within Salesforce. Go to the Setup page, in the Quick Find search for "App Manager." Next to the App that you created, click on the Down Arrow and then click on View.

 
  1. From here, underneath "API (Enable OAuth Settings)", click on "Manage Consumer Details" located next to "Consumer Key and Secret."
Important! A verification code will be sent to your inbox.
 

 
  1. After entering the code you received in your email, Salesforce Classic will appear where it will provide the Consumer Key and Secret.


 
  
Important!
 
  • Contracts must continue to be created inside of the Point of Rental. Once a contract is completed it will sync to Salesforce. Only changes done in Point of Rental will be made in Salesforce. If the customer makes a change in Salesforce, then the changes will not come over to Point of Rental.
 
  • Call Logs and Contact changes made in Salesforce will move over to Point of Rental.
 
 

Checking for Duplicates

In order for Point of Rental to properly sync with Salesforce, you will need to run the Setup Utility to check for duplicate customers/contacts. Point of Rental cannot resolve them. 

Important! Duplicates will have to be resolved before the Sync turns on. If this does not get completed then it will not sync the records. You must resolve duplicates before creating a task in the POR task runner.
 
  1. From the Salesforce Sync Screen, click Next.

 
  1. This will open up the Contact Duplication Check. Select the criteria for the customers/contacts to be reviewed by checking the boxes. The fields that you select will be reviewed for duplicates using the 'AND Command' in SOQL. The more criteria checked the fewer duplications will be found. When you're ready to begin, click on Apply to start the process. 

 
  1. Resolve the duplicates by either merging or removing the contacts from either Salesforce or Point of Rental.
 
  1. Once there are no more duplicates, you can start the process of syncing Contacts to Salesforce by clicking "Next". This will take you to a new page where you can click "Run."


 
  1. Follow the steps until you get a "Finished" box. Do the same steps for Calls Logs and Registering Contracts to Opportunities. 


 
 
Sales Reps
    
  1. Sales Reps will have to be configured in Point of Rental in order for it to sync properly. In Point of Rental, you can go to Configuration Gear > App Configuration > System Configuration > SalesReps. You will need to make sure that the Sales Rep has an assigned Operator.
 
  1. From within the Salesforce Setup, click on Manage Users.


 
  1. This will open up a new window, where you can add a Sales Rep and associate them with a Salesforce User by clicking "Add."
 
 

Creating a Task for Point of Rental and Salesforce


The last step of this process is to schedule a task in the Point of Rental Task Scheduler so that Salesforce and Point of Rental can communicate. This step has to be done AFTER resolving duplicates. After this is complete Point of Rental will start syncing your Items (Products), Contracts (opportunities), and Customers (Accounts).
 
  1. To begin, in Point of Rental,  go to the Configuration Gear > App Configuration > System Configuration > Task Scheduler.
 
  1. Once there, create the following Task from the POR Task Runner by pressing "Add."
  • Name - Salesforce Sync
  • Description - This task is going to connect Salesforce and Point of Rental
  • File to Run - <DataDrive>:\POR\Program\SalesForceSync\ SalesforceSync.AutoRun.exe
  • Start in Path - <DataDrive:\POR\Users
  • Recurrence - Minute
  • Interval - 15
  • Priority - Low

 
  1. Press save to save the task.
 
  1. After the task is created and the process is started, All Products will have to sync before Opportunities are made in Salesforce. This may take time depending on how many Item Records the customer has.

The Error Log


If you need to determine whether an Error occurred or find information on possible syncing issues, go to the Error Logs. You can find it by going to <DataDrive>:\POR\SFPORSync\AppLogs.
