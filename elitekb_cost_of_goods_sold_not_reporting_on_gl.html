Cost of Goods Sold Not Reporting on G/L

 

 

This article discusses the reasons why cost of goods are not reporting on the general ledger, and how these issues can be resolved.
 

If your problem is that the COGS report shows entries being done but nothing reflects on the G/L, there are two possible reasons for this. We will discuss them below.


Topics in this article include
 
  • Home Location
  • Default G/L
 





Home Location


Make sure that in the item file there is a Home Location, and it is not set to 000. The home location should reflect the store that buys the item. 

 




Default G/L


Make sure there is a default G/L for Sales Inventory (Default) and for Cost of Goods Sold.

Here's how to get there
Expert: Program Menu > Configuration > Accounting Configurations > Account Numbers
Elite: Configuration Gear > App Configurations > Accounting Configurations > Account Numbers
