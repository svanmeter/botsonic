Introduction

In some instances it may be necessary to change the default rate of VAT used in the Syrinx system such as the government raising or lowering the amount of VAT that must be charged against goods and services.  The following will guide you through the process that can be employed in Syrinx to achieve this.

Actions Before Changing VAT rates on Syrinx

Before changing the default rate of VAT, it will be necessary to complete the following actions so that all invoices created prior to the effective date of the new rate of VAT have been exported from Syrinx and will not be affected by the change of the default tax code.

- All unconfirmed Offhires, Advice Notes and Customer Service jobs, for which an invoice dated before the date that a new rate of VAT is to come into effect and has been printed and issued to the customer, should be confirmed and posted to the accounts system.

- An invoice batch for onhires, offhires and advice notes should be created with an “invoice up to” date equal to the date that a new rate of VAT is to come into effect and then review, print and confirm the batch.  These invoices should then be posted to the accounts system.  

Note – If you normally invoice hire continuations at the end of each month, you may simply want to invoice your onhires up to the previous month end if you have not done so already and allow for any continuation hire invoices created after this point to be invoiced at the new rate of VAT as long as the next period to be invoiced will extend past the VAT change date.

 

THESE TASKS MUST BE COMPLETED ON OR BEFORE THE DATE THAT THE NEW RATE OF VAT IS TO COME INTO EFFECT OR PRIOR TO MODIFYING THE RATE OF VAT WITHIN SYRINX.  ONLY ONCE THESE TASKS HAVE BEEN COMPLETED SHOULD YOU PROCEED WITH THE FOLLOWING PROCESS. 

There are two methods that can be used to achieve this change.  You should consult with your accountant or accounts department to determine which method is most suitable for your business.

Method 1: Creating a New Tax Code and Setting it as the Default Tax Code for Syrinx

If you do not have an existing tax code that corresponds to the new VAT rate, it will be necessary to create one (*note - If you already have a tax code entered in your Syrinx system for the new rate, please verify that the code is marked as current and skip to the next paragraph which describes setting the default rate of VAT used in Syrinx).  New Tax codes are created in the Syrinx SP Configuration program by selecting the “Accounts/Invoicing’ tab and then selecting the “Tax Codes” menu option.  On the “Tax Codes” screen, the new code along with its associated rate can be entered on the blank line at the bottom of the screen.  It is important to note that this code must match the corresponding tax code that has been set up in the Accounts program.  The code T20 used below is merely an example and any new codes that you create should follow your existing coding structure.  Please consult with your accountant or accounts Department before creating a new tax code in Syrinx.

After adding the new code to the Syrinx system, it will then be necessary to set this new tax code as the default tax code within Syrinx.  This is also set in the Syrinx SP Configuration program by clicking on the “System Options” and selecting the “Accounts” menu options.  The “Default Tax Code” field should then be set to the new standard VAT code.  You may also want to change the “Default Tax Code (Purchase)” field as well but this may need to wait until you have finished processing purchase invoices that were charged at the old rate of VAT.  The ‘Close’ button at the bottom of the screen should then be clicked to apply the change.   

Users should then restart the Syrinx program on their PC’s after this change has been completed.  From this point forward, all invoices produced from Syrinx will be charged at the new rate of VAT where applicable.

The original VAT code will still be available in the Syrinx system and can be manually applied to invoices where necessary.

You may also want to verify that no sales stock items have been set to explicitly use the old tax code.  You can do this by using the Amend Sales Stock Items menu and querying by the old tax code.  If any sales items are returned, you can click on the “Accounts” tab of those item records and clear out the value held in the “Tax Code” field.  This will force these items to use the new default rate of VAT.  Specific Tax codes need only be applied to sales items records if the item is normally charged at a non-standard rate of VAT and leaving this field blank implies that the item is charged at the standard rate of VAT.
