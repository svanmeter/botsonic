Usage Items

 

 

Usage items are items that have a per-unit fee usually in addition to rental fees. They are commonly added to a rental package or a suggested item, as a header item. When sending out the contract, you can select the serialized item. This article will walk you through creating a usage item and configuring free units per hour/week/month, if applicable.



Topics in this article include

  • Creating a Usage Item
  • Configuring free units
  • How do Usage Items work?
  • Removing a Usage Item from inventory
  • How do I partially return a Usage item?





Creating a Usage Item

 Follow the steps below to create a Usage Item. 

How to get there
Expert: Program Menu > File Maintenance > Item File
Elite: Gear icon > File Maintenance > Item File
 

  1. Select Add Item. For example, if you have several diamond blades, each one must have its own key (i.e., DB12-1, DB12-2, etc.).
 
  1. Enter general information regarding the item like the Key and Name. Each Usage Item must be individually listed in the inventory (i.e., quantity = 1). 
 
  1. To use the usage item feature, set up the item with the type: Type of Rental – Usage. 

 
  1. In the Details tab, Add your rental rates. Usage items only have 7 definable rental periods instead of the standard 10.  

 
  1. (OPTIONAL) The sales price of a usage item defines the maximum amount to charge. For example, if you enter $300.00 in the sales price field and your customer has a usage charge of $400, Point of Rental will only charge $300.00. If you do not want a maximum, then leave the sales price blank.


 

  1. Once you select the Usage item type, four new fields will appear below the period and rates sections:  # readings, free units, +/- max, and each unit. Fill in each option.

 

  • # readings - This is the number of readings taken each time this item is rented. For example, most stores measure diamond blades three or four times and record the average, whereas mileage and helium are read just once.
  • free units - This is the number of free units given per day with a unit being the base charge such as 1 mile, 1 thousandth, or 1 pound.
  • +/- max - This is the maximum reading allowed and indicates whether readings should get larger or smaller with use. The measurements must be entered in a full value, not a fractional value.
    • In the case of a blade, if you are charging the usage by millimeters used on a 3" blade, in the Max field you would enter -76 (3 inches equals 76.2 millimeters). 
    • For items that use miles, or count becomes larger use a positive number. For example, if an odometer maximum is 99999, enter “99999”. 
  • each unit - Enter the charge for every unit used.

 
  1. It's recommended to add a line in the Print Out Notes stating the charge per unit and the number of free units per day.
 

 
  1. When you're finished, press OK to save the record. 
 
 
 

Configuring free units

You need to set up the usage parameters under Rates in Parameters. These numbers determine how many free units are given per hour, per week, and per month. These parameters are only used if you are giving free units. If the “free units” field in the item record is 0, then these parameters are ignored.

How to get there
Expert: Program Menu > Configuration > System Configurations > Parameters > Item Rates tab
Elite: Gear icon > App Configuration >  System Configurations > Parameters > Item Rates tab


For example:

6 hours, 3 days, 3 weeks - For every hour charged in rent on a usage item, give the customer 1/6 of the free units allowed per day. For every day charged, give the customer the number of free units per day as defined in the item record. For every week, give the customer 3 times the number of free units per day.  And for every month, give the customer 3 times the number of free units per week (i.e., 3 x 3 x free units per day).

0 hours, 5 days, 4 weeks - The customer gets 0 free units for each hour charged, but will get the number of free units per day for each day charged. For every week, give the customer 5 times the number of free units per day. And for every month, give the customer 4 times the number of free units per week (i.e., 4 x 5 x free units per day).

0 hours, 0 days, 0 weeks - This is the most commonly used setting. The customer gets only the defined number of free units per transaction. He will not receive any more free units the longer he has it.

 

Examples:

Suppose Joe rents a diamond blade with free units = 6, per unit = 400, and the usage parameter = 6 hours, 3 days, 3 weeks.

Joe is charged a 1-day rental and uses 4 thousandths. He doesn't have any usage charge since he receives 6 free units with his rental.

Joe is charged for 1 day and 2 hours and uses 10 thousandths. He receives 8 thousandths free (6 per day + 2/6 x 6 per day), so he has to pay for 2 thousandths at the rate of $4.00 per thousandths, resulting in a wear charge of $8.00.

Joe is charged 1 week and 1 day and uses 30 thousandths. He receives 24 thousandths free (3 x 6 per day + 6 per day), so he has to pay for 6 thousandths resulting in a wear charge of $24.00.



 

How do usage items work?
 

When the usage item is rented out, the counter person will be asked to enter the current reading(s). The computer will then average the reading(s) and prompt you to accept the average reading(s). The average reading when the item was last returned will be the default.

 

Usage items create two line items on the contract: one for the rental of the item and the other for the sale of the per-unit usage. The two-line items are integrally connected (i.e., you can't delete one line item and leave the other). If you don’t want to charge for rental and/or sale just change the price to 0 instead of trying to delete the line item. 

Both the rental line item and the sale line item have the same taxable status. In other words, if the usage item is non-taxable, both the rental and sale will be non-taxable, and vise-versa for a taxable usage item.  However, if the tax jurisdiction defined on the contract does not charge tax, both line items will not be taxed, regardless of the item's tax status being taxable.

 

When sending out a reservation, the quantity of the sale line item will be equal to the number of free units allowed for the specified time period. You can't directly change the quantity of the sale line item. If you try to change the quantity, it will force you to enter the reading(s). This is to ensure that the proper reading is held in the item file. Also, the quantity of the sale line item will always be greater than 0. The average unit reading will be printed on the contract above the rental prices when a usage item is opened


The amount charged is based on the unit difference between the beginning and end reading. The quantity of the sale line item is equal to the total number of units used. The price of the sale line item will be equal to the total number of units used minus any free units given multiplied by the charge per unit [equation: (total units used - free units) x charge per unit]. The total income from the sale and the rental will be included in the item income figure. The rental line will be included in the rental totals and the sale line item will be included in the sales totals. 

  • Important note, if you change the quantity of the sale line item while the rental line is still open, the OUT unit reading will be changed. If you change the quantity of the sale line item while the rental line is closed, the IN unit reading will be changed.
 

When closing a contract, you'll be prompted to enter the current reading(s). If a usage item reading decreases and the average IN reading is greater than the average OUT, the reading will not be accepted, and the average IN reading will default to the average Out reading. The average unit reading will be printed on the contract above the rental prices when a usage item is closed. After the usage item is closed, the unit reading will not appear on subsequent printings of the contract because the program does not retain that information.




Removing a Usage item from inventory


When your Usage item is no longer usable, you can remove it from inventory by an asset sale or through an inventory count update. 




How do I Partially Return a Usage Item?

Usage items cannot be returned using the Close Contracts option, Partial Return. To return your Usage item on an open contract, right click on the item and select "Close Item."
