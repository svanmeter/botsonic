Data Management Tools | Rental Rate Adjustment


 

In this article, we go over making rental and sale price adjustments using the Rental Rate Adjustment feature in Database Management Tools. You should always do this in the "Training Database" or in "Training Mode" BEFORE you work with live data. This feature is only available with the Database Management Module. Accessing the Training Database or Training Mode

In Expert, press the Training Database checkbox in the Program Menu. 




In Elite,  before you sign in, press the Training Mode check box.





 

Getting there
 

Expert: Program Menu > Database Management > Data Management Tools > Rental Rate Adjustment

Elite: Left Menu > Tools > Database Management > Data Management Tools > Rental Rate Adjustment


 

 

Who can access Rental Rate Adjustment?
 

In order to use this module, you MUST have an owner's level password and a license for the Data Management Module. It is recommended that no one is in the Point of Rental System when performing this action.

You should always do this in the "Training Database" or in "Training Mode" BEFORE you work with live data.

 



How will items on a contract be affected?

By default, items on Reservation and Open contracts will be updated. However, before starting the updating process, you can check a box to "Hold Pricing on Reservation" contracts. This can be found in the Update section.



 

Rental Rate Adjustment

You should always do this in the "Training Database" or in "Training Mode" BEFORE you work with live data. It is recommended that no one is in the Point of Rental System when performing this process.  Follow the steps below to perform a rental rate adjustment.


 

1. If you are updating your rental price, select Rental Adjustment under the Load field.
 

 

2. Under Rental Adjustment, enter the rental price % change and the rental price round-off.
 

 

 

Rental Price % Change - This parameter defines the percent change being applied to your rental rates. It may be specified up to two decimal places. For example, 5.25 designates a 5 and ¼ percent increase. A negative number is used for prices to decrease. For example, -4.5 designates a 4 and ½ percent decrease.

 

Rental Price Round-Off - This parameter defines the round-off factor to be applied to the adjusted rental prices. For example, 0.50 causes all rental prices to be rounded off to the nearest 50 cents.
 

Note: If you round to the nearest dollar, items with low rental rates may round up/down in an undesirable fashion. For example, suppose you rent plates for $0.35/day, $1.00/week, and $3.40/month and round to $1.00 with a 10% price increase, the rates will be $0.00/day, $1.00/week, and $4.00/month.

 

3. Under Selection, choose the store and category or categories to schedule for the rate changes.

 


 

 

Choosing all stores (as opposed to an individual store) - Selecting all stores will update the header record with the new price. Any record that does not have a header will also update with the new price.

 

When choosing a specific store - When updating the price on the header record, a new store special rate will be created for the specific store in the header record special rates. If you have an item that does not have a header, a special rate will be added to the item file (the price on the item will not update).


 

4. Click the Load Data button once selections have been made. A list of items will be displayed.
 


 

 

5. Check the box next to the items that will be changed. Double-clicking on ‘Select’ will check or uncheck all the boxes. 
 

 


 

6. Within the Update section, you can tell the system when and on which transaction these prices will take effect.



 

Effective Date - This will allow you to choose the dates when the new rates will take effect.

 

Permanent Price Change - This box is automatically checked. If this box is not checked then it will create a Special Rate during that time.

 

Hold Pricing on Current Reservation - With this box checked, pricing on future reservations will not be affected. This box will only appear if the effective date selected is a future date.


 

7. After you confirm everything, click on Update Rates.
 


 

8. A message will appear asking you to confirm that your new rates will begin immediately or at the set date that you chose.
 

 

Note: If you set up an additional rate change for the same item for the same start date/period, the second rental rate change entered will override the first.


 


 

Sales Price Adjustment


You should always do this in the "Training Database" or in "Training Mode" BEFORE you work with live data. It is recommended that no one is in the Point of Rental System when performing this process. Follow the steps below to perform a sales price adjustment.


 

1. If you are updating your sales price, select Sales Adjustment under the Load field.
 

 

2. Under Sales Adjustment, enter the sell price % change and the sell price round off. Using the respective radio buttons, you can define if these changes will be reflected on the sell price, purchase price, or retail price. 
 

 

 

Sell Price % change - This parameter defines the percent change to be applied to your sell prices. It may be specified up to two decimal places. For example, 5.25 designates a 5 and ¼ percent increase. A negative number is used for process decreases, for example, - 4.5 designates a 4 and ½ percent decrease.

 

Sell Prices Round Off - This parameter defines the round-off factor to be applied to the adjusted sale prices. For example, 0.50 causes all sale prices to be rounded off to the nearest 50 cents.


 

3. Under Selection, choose the store and category or categories to schedule for the rate changes. You can also enter a customer number if you want to change the price for an individual customer.

 

 

 

Choosing all stores (as opposed to an individual store) - Selecting all stores will update the header record with the new price. Any record that does not have a header will also update with the new price. 

 

When choosing a specific store - When updating the price on the header record, a new store special rate will be created for the specific store in the header record special rates. If you have an item that does not have a header, a special rate will be added to the item file (the price on the item will not update).

 


 

4. Click the Load Data button once selections have been made. A list of items will be displayed.
 





 

 

5. Check the box next to the items that will be changed. Double-clicking on ‘Select’ will check or uncheck all the boxes.
 

 

 

 

 

Effective Date - This will allow you to choose the dates when the new rates will take effect.

 

Permanent Price Change - This box is automatically checked. If this box is not checked then it will create a Special Rate during that time.

 

Hold Pricing on Current Reservation - With this box checked, pricing on future reservations will not be affected.  

 

 

6. After you confirm everything, click on Update Rates.
 


 

7. A message will show up asking you to confirm that your new rates will begin immediately or at the set date that you chose.
 

 

Note: If you set up an additional rate change for the same time period  the second sales rate change entered will override the first.



 

Filtering data by importing and exporting CSV files
 

Rates can also be adjusted through an import/export feature that is also available. You can filter the data you want to adjust and then export it to a spreadsheet.
 

 

The rates can be manipulated in a spreadsheet and then saved. Once you select “Import CSV,” you will be given a list to choose from.

 

 

Once loaded, specify when the price change should occur and click Update Rates.
