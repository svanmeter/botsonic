Syrinx® - Driver App

 

 

 

Overview: This guide goes over how to install the Driver App on your mobile smart phone or tablet.

 

Topics included:

  • Download and install the App from the Google Play Store
  • Enter Connection Details to the Syrinx Android Web Service
  • Using the Driver App

 

 

Download and Install the App from the Google Play Store

 

Google Play Store: https://play.google.com/store/apps/details?id=com.syrinxda.main

Apple App Store: https://apps.apple.com/us/app/syrinx-driver/id1460269940

 

Navigate to the Google Play Store on your device:

 

 

 

Then, search for Syrinx and select the Syrinx Driver App from the list of results:

 

 

 

Press the Install button to begin the installation of the Syrinx Driver App:

 

 

 

 

Press Accept to allow the Syrinx Driver App to access the required areas of your Android device:

 

 

 

Once installed, you can press Open to open the app and begin working in the Syrinx Driver App:

 

 

 

The Driver App will be available in the list of apps and can be run from there. If you would like to see this on your home screen, you can do this by opening the app list. The icon looks similar to the image below (see icon below):

 

 

 

 

The Syrinx driver icon will be at the end of this list. As there may be multiple pages, keep scrolling until you find it. Hold down on the icon until the screen changes and then drag the icon onto the home screen, or wherever you would like it to be stored. This procedure may vary based on device manufacturer, so if required, guidance can be found online by searching with the device’s model name in the search string.

 

The Syrinx driver app is now ready for use.

 

 

Enter Connection Details to the Syrinx Android Web Service

 

When launched for the first time, click on the Change Server option on the main screen to enter the connection details to the web server hosting the Syrinx Android Web Service:

 

 

 

Enter the IP address or host name of the web server and press Save:

 

 

 

If you are unsure of the connection details for your web service and have other active driver app devices, you can check the details of one of the existing devices and you will find this information.

 

The Enter Connection screen allows you to enter multiple addresses for your web server in cases where your router does not allow web traffic originating from within your network to access resources using your external IP address of your web server. In that type of scenario, you can enter your external IP address in the 1st Web Service URL box and then the internal IP address of the web server in the Alternate Web Service URL box.

 

 

When the driver is out of the office and connecting over 3G, it will use the 1st connection string and when the driver returns to the office and is connecting over the internal wireless network, it will connect to the alternate URL.

 

If successful, you will then see a Connection successful message when the app moves back to the main screen:

 

 

 

All that is required now is to set up a staff member in Syrinx with a Syrinx Mobile Username which can be used to log in to the app. This is done by going to Syrinx > Staff > Staff Members and selecting an existing staff member who has been enabled as a driver and adding a value to the Syrinx Mobile Username: field on that staff member’s record:

 

 

 

 

This username can then be entered into the driver app in order to complete the connection:

 

 

 

Congratulations! You have now set up the Syrinx Driver app.

 

 

Using the Driver App

 

This guide is designed to help you through the stages of the Driver App. The App can be installed on Android devices.

 

Once installed and configured, the user should be able to log into the app.

 

Once you tap on the app you will see the login screen. Enter the Syrinx mobile username and tap Enter:

 

 

 

 

When the app loads you will be presented with your Job List. As you can see below, you are able to do vehicle checks through the app. The vehicle check makes the driver answer pre-defined questions about their vehicle before they can commence their daily duties. If any of the checks fail, we can set it to create a repair job, which is added to the open jobs list in the workshop section of Syrinx.

 

 

 

 

 

Once the vehicle check has been completed, the checklist will automatically be removed from their job list. Only the remaining outstanding jobs will be left on the jobs list.

 

 

 

 

The driver will need to tap on the job that they would like to do next. This will bring them to the front screen of the job. On this screen they have all the information required, such as site address, site contact, etc. The driver can swipe across and see the details of what they are delivering on the Item List screen.

 

When the driver is on their way they should tap the En-Route button:

 

 

 

 

Once the driver arrives at the customer site they should press the Start Job button. This will take them straight to the Item List page. Depending on how your app is configured, the driver can tap Mark All and this will automatically tick all items as delivered.

 

Alternatively, if you would prefer the driver to manually tick each item, this can be set up as well. When the driver taps onto a line, it will take them into the details of that item, as shown in image 2. In here the driver can edit the stock number (this can be turned off if required) change the status of the item, take photos (if there is a red exclamation point over the camera, photos must be taken). Mark as Done and add any specific notes to that item. The Driver can then continue to swipe left to finish all items on their delivery. Once they have completed all of the items, the Items List will appear as per Image 3.

 

 

Image 1 Image 2 Image 3

 

 

The driver will then need to swipe left again and include any overall comments relating to the job in the Notes tab. There is a Status dropdown option, if, for example, the job was aborted.

 

 

 

The next option is the Photos tab. If the driver taps the camera symbol they will be asked to either take a photo using the devices camera or upload one from file. If the driver takes a photo, it will automatically upload to the server. The image can be deleted, and a new photo taken if the driver is unhappy with the quality of the picture. Any photos that do not upload due to low signal strength can be uploaded at a later stage using the Unsynched photo section in the menu.

 

 

 

 

The final screen is the customers Signature screen. If there is a contact linked on the contract in Syrinx, the app will automatically ask “is Mark Whiteman signing for this”. If this person is the correct person, click Yes. If not click No, and you can manually type the person’s name in.

 

Once this has been done, tap on the signature box and the customer can sign in this space. If you would like to increase the size of the signature box, simply tilt the device to a landscape position and the signature box will be bigger. There is also a Nobody on Site option, which can be disabled if required. Once the signature has been obtained then the driver can tap on Mark Complete.

 

 

 

The driver can access the Menu at any point by clicking on the three lines in the top left corner.

 

 

 

The below list details the options available in the Menu tab:

 

Job List

Shows a current list of outstanding tasks.

Create Off Hire

This allows the driver to create an off hire if the customer asks them to remove an item from site. Once completed, this will sit in unconfirmed off hires until a Syrinx user processes the off hire.

Create Vehicle Defect

If the driver notices a defect during their working day they can report this back to the office by using this feature. This will appear in mobile work schedule.

Unsynched Photos

Any photos that fail to upload due to low signal/connectivity issues will still be stored on the device. When the user selects this option, the photos will be synced back to the server.

Print Settings

This can be used to configure wireless/Bluetooth printers to produce paperwork for the customer on site.

About

This will show the app version, Device ID, Database it’s connected to, WEB URL and any system options. Note that the options cannot be changed from here, only viewed.

Logout

This will log the mobile user out of the driver app.
