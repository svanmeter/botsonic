Customer Authorisation

You can now authorise a customer to allow more orders to be raised when the customer is on hold or over their credit limit.  You must first grant the privilege ‘Admin.Authorise Use of On-Stop Customer (any)’ or the privilege ‘Admin.Authorise Use of On-Stop Customer (own)’.  The latter will only let a user authorise those customers where they are the salesperson.

Once the privilege has been granted, there is a new ‘Auth’ button next to the ‘On Hold’ box that can be used to authorise the customer to allow new orders:

 

When you press the button, it will show the current ‘on hold’ message, and then you must enter a reason for authorisation and how many new orders that are allowed:

 

A four digit code is then provided to the end user which should be passed on to the person who wants to raise the new hire contract/advice note:

 

When a new contract or advice note is raised for that customer, the system will prompt for the authorisation code:

 

If the code matches the one generated, then the system will allow the user to enter the new order and deduct the amount of authorisation usages by one.

If a contract was created before a customer was put on hold (or they went over their credit limit), you will still be able to see the details of the order but you must use an authorisation code to be able to confirm the contract on the system.  This is done by pressing the ‘Authorise’ button on the ‘Details’ tab of the ‘Enter Hire Contract’ and ‘Enter Advice Note’ screens.
