INTERNAL | Troubleshooting System Updates


This article will show you how to check update eligibility, how to run them, and troubleshooting methods. To run updates, all users must be logged out of Point of Rental.


Topics included into this article

  • Determining if a customer is eligible to run updates
  • Getting the updates to push over
  • Where do customers go to run updates?
  • How do I know if all users have logged out?
  • Possible reasons why the update did not go through


 

Determining if a customer is eligible to run updates

In Expert and Elite for a customer to be eligible to upgrade to the next version they must have a software maintenance or support subscription. 
 

Within Salesforce
 

You can view this information under the Software Info tab on the Asset page.
 


 

Within the Software
 

In the Expert and Elite Platforms, there are two ways that you can see if the customer is eligible for updates: through the File Sync and License area of the program.

 

Licensing

 

Expert: Program Menu > Configurations > Licensing

Elite: Configuration Gear > App Configurations > Licensing

 

Under the License Information, you will see a Software Support Expires row.
 


 

File Sync
 

To see if the customer is eligible for updates you can also check for updates in their system. If the customer is in a Farm or Point of Rental Managed Windows Virtual Desktop Solution, this process might be different. Please consult with our CEX IT Team for information.


Expert: Program Menu > Configurations > Other Updates > Check for Updates

Elite: Configuration Gear > App Configurations > Other Updates > Check for Updates

 

On the top of File Sync, you can go to Help > About and look for the Software Dates.
 

 


 

Getting the updates to push over
 

If the customer is up to date on their subscription payments, the accounting team will need to update GAdmin with the new dates. Contacting David Jensen, Global Director of Product Development, is NO longer needed.
 

 






Where do customers go to run updates?

Before running updates, all users must be logged out of Point of Rental. 

In Rental Expert,  go to 9. Configuration > Other Updates > Install Updates

In Rental Elite, go to Left Menu > Configuration > App Configurations > Other Updates >  Install Updates.

              


How do I know if all users have logged out?

To check if all users are logged out, go to Task Manager to see what is running (depending on the server). Applications can be found on either the Processes tab or the Details tab. Armenu.exe is the program for Point of Rental. If this is running it means someone is still logged into the program.  Terminate that process from that machine if possible.


Important! We have customers who operate on a server farm, meaning they have more than one server. Running updates for these customers is a more complicated process.  These customers usually have an IT group that coordinates running updates. They have to coordinate to ensure all users are out of Point of Rental.

You can confirm who is on a server farm by looking in their account in Salesforce >  Asset tab > Software info.

 

Possible reasons why the update did not go through
 

Scenario:  The customer is getting a .Net Framework install message

Solution: If this is the case, create a ticket for the IT department for follow-up. This involves Windows updates for the customer’s server.

Scenario: Updates are not available.

Solution: The result of this is the customer may be behind on their payment to Point of Rental. A case for the Accounting team needs to be created for follow-up.

Scenario: You are at the version edits screen, but it fails.

Solution: Most likely a user logged into Point of Rental during the update process. You will have to log that user off. You can confirm the user from the task manager on the server.

Scenario: Unable to run updates due to user in the system.

Solution: All users must be out for updates to run.
