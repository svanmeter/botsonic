Internal | DevOps Work Statuses



This document is for internal use only.

This document will define DevOps Work Statuses. An issue is NOT resolved for a customer until there is a release version listed on the DevOps tab AND we are on that version or a version past it.




Status

New It has not yet been assigned to a developer. We only assign a ticket to a developer when they are ready to actually start working on it, so most tickets stay in this state for a while.
Work In Progress The developer has been assigned and is working on the issue.
To DoSet into the current sprint. The goal is always to get all items done in a sprint but that can never be guaranteed (or communicated).
Needs more workMore detailed development testing, needs more specific information from support agents, maybe a TV needs to be provided, etc.
Dev CompleteThe developer believes that they have coded a fix that solves some or all of the problem, but that has not yet been tested by QA or other developers and is not yet scheduled for any release.
Done The fix has been tested and deemed to resolve the issue OR the case is unreproducible, fixed by another ticket, or otherwise work is complete. (NOT complete for the customer yet.) All steps after "Done" happen via automated processes. Any ticket which has been in the Done status for more than 1 week would be worth asking the development team about. (Which is why the known issues list has dates on it - to track when Support notes the status change).
Done + comment saying "@support The fix for this development ticket will be released in the vXX.YY.ZZ release. We cannot provide an exact timeframe for the release of this version but it should be out within the next 2 to 3 weeks"This fix will definitely be out in the specified version (if that version is not specifically released, the fix will be in the next higher version that is released).
Done + comments saying "@support This ticket has been released to SERVER in version vXX.YY.ZZ"The fix is live on the specified server. We put a separate message for every server the fix is released to, so look for the message affecting the appropriate client. E.g., if it is an AU client, look for the message saying the ticket has been released to prod, etc.
QA (quality assurance)/QA In Progress Testing is underway to make sure the issue is resolved and should be ready to release in the next update.
QA PassedQuality assurance testing has passed & the issue is resolved. It should be released in the next update. 
Rejected -Rejected for various reasons.
