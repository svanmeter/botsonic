Enabling Syrinx Version 3 Menu in Version 4 Client


Overview

Syrinx version 4 uses office style ribbons to access the various menu options.  If you wish to use the classic Syrinx menu style this can be enabled using the following procedure.


Granting Users Permission to Alter the Startup Menu

In order to have the option to enable the classic Syrinx menu in version 4, a user will require the system privilege ‘Prv.SysAdmin.Ribbon Cfg Startup’.  This privilege will need to be added to an existing user-defined role linked to the user’s logon or a new role will need to be created that contains this privilege and then that new role linked to the user’s logon.  This can be accomplished in the Users/Logons menu of SyrinxSP Configuration.


Enabling the Classic Syrinx Menu

Open the Syrinx version 4 client and connect using a logon who has been granted the system privilege ‘Prv.SysAdmin.Ribbon Cfg Startup’ and then click on the “File” menu option in the upper left corner of the screen:

 

 

Enable the “Display Main Menu on Startup?” option and then close and reopen the Syrinx client program.  The logon user should now have access to the Syrinx classic menu:
