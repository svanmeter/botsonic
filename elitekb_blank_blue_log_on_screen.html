Übersicht

 

Dieser Artikel beschreibt den Fehler, der auftritt, wenn Sie sich bei Ihrem Windows-Server anmelden, aber Ihre Desktop-Symbole nicht geladen werden und Sie an einem blauen Bildschirm festhängen. In diesem Artikel enthaltene Themen:

 

  • Leerer blauer Anmeldebildschirm Fehler
  • Lösung

 

 

Leerer blauer Anmeldebildschirm Fehler

 

Dieser Fehler wurde vor allem auf Windows 2008 R2-Servern beobachtet. Er tritt auf, wenn ein Benutzer versucht, sich per Remote-Desktop an seinem Server anzumelden. Die Remote-Sitzung wird aufgebaut und der Benutzer sieht, dass der Server „Computereinstellungen anwenden“ anzeigt, aber danach wird derselbe blaue Bildschirm ohne jeglichen Text angezeigt. Der Benutzer kann die Sitzung über die Leiste für die Remote-Desktop-Sitzung am oberen Rand des Bildschirms immer noch minimieren oder die Verbindung trennen. Selbst wenn der Benutzer die Verbindung trennt und erneut versucht, sich per Fernzugriff anzumelden, erhält er denselben leeren blauen Anmeldebildschirm. Aufgrund des unmittelbaren Auftretens dieses Fehlers scheint es, dass es sich um das Ergebnis eines Windows-Updates handeln könnte. 


 

Lösung

 

  1. Melden Sie sich bei Ihrem Windows-Server als administratives Benutzerkonto an.
  2. Öffnen Sie Verwaltung > Remotedesktop-Services> Remotedesktop-Services-Manager. Alternativ können Sie auch den Task-Manager öffnen und den Tab „Benutzer“ wählen.
  3. Klicken Sie mit der rechten Maustaste auf den Benutzer, der den leeren blauen Anmeldebildschirm erhält, und klicken Sie auf „Abmelden“.
  4. Melden Sie denselben Benutzer erneut von seinem Arbeitsplatz aus an. Er sollte sich nun anmelden können.
