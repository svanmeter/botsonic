What3words integration - Users now have the option to use what3words coordinates against addressees in Syrinx:

The button linked to the W3Words field on the address can be used to either get the what3words value for the site's existing Lat/Long values or to generate the site's Lat/Long value from the what3words coordinates.  The option is available on customer/supplier addresses, global sites, staff member addresses and depot addresses.

In order to record a what3words value against addresses, the system must have a valid Syrinx mobile app license (e.g. Driver, Workshop or Operator app).  The translation of what3words values into valid LAT/Long coordinates and vice versa is then enabled by entering a valid what3words api key on to the new "Misc(2)" tab of Syrinx SP Configuration\System Options\General (this tab is only visible if the database holds a valid mobile app license):

Quickbooks Online Transfer Tool - There is now a version of the Transfer Tool that can link with Quickbooks online.  This tool is licensed and will require a valid license to run with a Syrinx database.  On initial release, the Quickbooks Online transfer tool will have the same functionality as the existing Quickbook for Windows (QBW) transfer tool meaning that it will support the direct transfer of customers, suppliers, sales/purchase invoices and depreciation (journals) into Quickbooks online and will work with Syrinx business areas if enabled. It will also pull back customer balances, aged debt information and unpaid invoices into Syrinx. The tool will also have the ability to run as a service if required.    

Option to Link Countersign Tablets to Depot Records - Users now have the option to link countersign tablets to depot records for instances where they perhaps do not use cash tills or have more tablets than tills.  The countersign tablets can be linked to a depot via the new countersign tab on the "Amend Depot" screen:

When a new tablet is added, the "Register Key" value for that tablet will be randomly generated and is unique to the database.  The button to the right of the "Register Key" value can be used to copy the key to the clipboard:

The "Register Key" value should then be used to identify the device in the countersign website either via "Add Register" if a document has not already been sent to countersign or by selecting the value from the list of available registers that have already sent a document to countersign:

Set Timesheets to be Reviewed When Imported via SyrinxWebApi  - The Timesheets POST endpoint now includes a "ToBeReviewed" value that can be set to determine whether or not the timesheet requires review after being imported.  If the value is true or not defined in the payload, the imported timesheet will be flagged for review.

POST ChargeUnits Endpoint Added to SyrinxWebApi Along with Option to Enter Charge Units via POST Timesheets Endpoint - A new POST ChargeUnits endpoint has been added to SyrinxWebApi which will allow for the import of charge units against hire contracts.  A "ChargeUnitEntries" section has also been added to the POST Timesheets endpoint which will allow for the import of charge unit entries when importing timesheet information.

Users GET Endpoint Added to SyrinxWebApi  - A Users GET endpoint has been added to SyrinxWebApi which will allow for details on Syrinx system users to be extracted from the system.  Users can be queried via the parameters for user name (can be partial), server logon and an IsActive flag.

Virtual Directories Enabled for Syrinx Mobile Service - Virtual directories can now be configured for the "EmailTemplates" and "Reports" directories on the Syrinx Mobile service site so that these customer specific files can be held in a central location instead of having to reside on the web server.

Syrinx Task Scheduler No Longer Requires Admin Access to Configure and Run - The Syrinx Task Scheduler no longer requires Admin Access in windows in order to run and configure the tool prior to starting the Task Scheduler service.  If the Task Scheduler is run as Administrator, the UI and service will continue to store and read its configuration information in the Local Machine registry key.  If the Task Scheduler is not run as Administrator, it will store and read its configuration information in an encrypted config.dat file in C:\ProgramData\Higher Concept\Task Scheduler.  

*Note - Existing customers who have the Task Scheduler and upgrade to this new version should set their Task Scheduler to "Run as Administrator" if they can no longer view/edit its configuration information through the user interface, the service itself should still run as expected as it will continue to read the configuration information from the Local Machine registry key.
