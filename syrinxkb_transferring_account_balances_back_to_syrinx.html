Overview:

As well as transferring new and modified customer details from Syrinx to your accounts package, you can also transfer the credit limits and account balances back into Syrinx, so that hire staff can receive a message when entering a new hire / sale if the customer is over their credit limit.

If this option is set, then the balances are transferred back into Syrinx automatically at the end of each invoice transfer.  You can also transfer the balances back at any other times (see below).

To configure the system to transfer balances / credit limits:

1. Using the Syrinx Configuration program navigate to the System Options/Accounts menu.

2.  Go to the tab titled ‘Direct Transfer’ and tick the option ‘Balances/Credit Limits’

3. You may optionally set the following options:

Even if no Credit Limit Set?

If this option is set, then the customer balances are transferred back to Syrinx even if there is no credit limit set within the accounts system.

Unpaid Invoice List?

If this option is set, then the list of unpaid invoice numbers and invoice dates are also transferred back to Syrinx.

‘On Hold’ status?

If this option is set, then as well as transferring the credit limits and account balances, the ‘On Hold’ status is also transferred from the accounts system to Syrinx.

New Customers?

If this option is set, any new customers added to the accounts database that do not exist in Syrinx will be transferred back to Syrinx

Modified Customer Name/Addresses?

If this option is set, any changes to customer invoice addresses or customer names will be transferred back to Syrinx.  When this option is enabled, the customer invoice address in Syrinx is read-only and cannot be amended.  Changes to this information must be made in the accounts package and then transferred back to Syrinx.

Don’t Auto-run with each Transfer?

If this option is set, the Accounts Transfer Tool will not automatically pull back this information when a user runs the ‘Process all Tasks’ option.

If Over Credit Limit – Issue Message to User

This option lets you tailor the message that the hire staff will receive when entering a new hire / sale if the customer is over their credit limit.  If the text ‘|1’ or ‘|2’ is present in the message text, then it will be replaced with the customers balance or credit limit (respectively).  For example, you could set this option to “Customer balance (|1) is over credit limit (|2)”

         If Over Credit Limit - Stop with Error?

If this option is set, then the hire staff will receive an error stop rather than a warning when entering a new hire / sale if the customer is over their credit limit.

         If Over Credit Limit – Show Aged Balances/Unpaid Invoices?

If this option is set, then the hire staff will be shown the customer’s aged balances and unpaid limits when entering a new hire / sale if the customer is over their credit limit.

 

4. Press the OK button

To transfer balances / credit limits back to Syrinx:

1. On the PC that is used to transfer the customers / invoices to your accounts system, start the Transfer program.

2. Select the ‘Transfer Customer Information from…’ task, and press the Run button:

Alternatively, this task will also be run automatically if the user presses selects the ‘Process All Tasks’

 

Refreshing all aged balances

The transfer program will bring back aged balances as well as the total customer balance.  However, it will only retrieve the aged balances from the accounts system if the balance on the account has changed – i.e. if any new invoices have been posted, or payments entered on the accounts system.  This is by design, as the retrieval of aged balances can slow the transfer down significantly.  However, if there is no activity on an account over a period of more than 1 month, the aged balances in Syrinx can get out of date.  There is an option in transfer tool to force it to retrieve all aged balances from the accounts.  This option may be set before running the transfer by pressing the Edit button, and then choosing the Run Options tab:

 The option will automatically unset itself at the end of the transfer, so you just need to set it manually when moving to a new accounting period.
