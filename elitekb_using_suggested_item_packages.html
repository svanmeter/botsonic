Using Suggested Item Packages

 

Please read through this entire process so you understand how each step will affect the next.

Suggested Items Packages can be used in several ways. The most common is for creating a sleek and sophisticated-looking website. Secondly, if you have 50 items that have the same suggested items, you could create 1 suggested items package, then assign the suggested items package to the 50 items as a package. 6 months from now when you want to add a new suggested item to those 50 items, all you need to do is add it to 1 record instead of all 50. Lastly, they can be beneficial to your counter staff when searching for inventory and extremely beneficial to new employees learning your inventory.

 

Here is an example of what a ‘Suggested Item Package’ will look like upon completion.

Note the grouping and sorting. It is a good idea to create an intuitive grouping or sorting scheme and stick to it.

 

Overview:

1. Create an Item Record for the suggested item package.

2. Build your Suggested Items List.

3. Determine the correct way to hide your items from the website.

4. Create Item Records for all suggested items packages.

  • The item type should be ‘Rental – Package’.
  • You must determine whether you want this item to show in the inventory selection screen when searching for items. If you do not want it to show, mark the item ‘No Print on Contract’.
  • Change the quantity to 9,999 to indicate that this is not an actual inventory item.
  • Be sure the Header Key is blank
  • If you are using Global Headers set the Home and Current Loc to ‘000’. Otherwise, create this record for each store. (Global Headers make this process much easier.)
  • Determine a category for these items. (if you need to create a new category, do it now).
  • Repeat Step 1 until all packages have been entered.

TIP: Right-click a like item and select copy; Change the Item Key, Item Name, and Qty.

Example: Suggested Item Record

2. Build the suggested items list for each package.

  • Access the first Suggested Items Package.
  • Select TOOLS at the top of the screen then choose SUGGESTED ITEMS.
  • Right-click in the gray area of the box and choose ‘Add’.
  • If you use Global Headers insert the Global Header. If you use Store Headers insert the Store Header. If you do not use Headers insert the actual item. Assign the appropriate action from the drop-down in the action type column. The user can choose to have this item presented On Add, On Send Out, or On Close.
  • Enter all items required to build this suggested item package. Be consistent in the order of the items selected. (i.e. Round, Square, Rectangle, etc.…)
  • Make sure the Special Box is not checked.

3. Determine if it is necessary to hide items on your website.

  • If you do not have an integrated website with Point-of-Rental™ Systems you can skip step #3.
  • If you do not want the actual items to list on your website and only want the packages to show, you must do 1 of two things:

o, Mark, each actual item ‘Hide on Website’.

o Or, during the web update process you can choose to not upload that category.

  • You must determine the best way to accomplish this.

Remember, this is one demonstration for creating Suggested Items Packages. Depending on your setup and situation slight adjustments to these instructions may be necessary. This procedure can be modified to accommodate most situations that may be required. By applying the procedures discussed in this document, the possibilities are limitless. Use your imagination.
