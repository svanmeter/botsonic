Installation

After downloading the required version of the Syrinx installation program.  Double click on the SpV4Setup.exe file to begin the installation.  Verify that the version number listed in the lower right corner of the start screen matches your current Syrinx version and then click on the “Install” button to begin the installation:

 



 

The system will then begin the installation of the Syrinx and Syrinx Configuration programs, Syrinx PDF printer driver, Crystal Reports Runtime XI and Microsoft Access Runtime 2010 32 bit.  Please note that if you are running a 64 bit version of Office 2010, the installation of the Microsoft Access Runtime may fail as Microsoft does not permit the installation of 32 bit and 64 bit versions of the same version of office.  It may be necessary to uninstall any 64 bit Office components and replace them with the 32 bit versions before you will be able to install Syrinx on the PC.

Troubleshooting

1.  When attempting to connect to the database, you receive the error message: “The PC ' computername' has not been granted access the system. Please contact your local system administrator.”  This is a licensing issue and means that the full number of licensed PCs has been reached on the database.  The list of PCs currently holding licenses can be viewed in the “Registered PCs” screen in Syrinx SP Configuration\System Administration.  If necessary, a PC can be removed from the list by selecting the name of the PC that no longer requires a license and pressing the “Remove PC” button.  The new PC can now connect to the main Syrinx program and will take the free license.  If all PCs in the list require a license to access the main Syrinx program, an additional license will need to be purchased.  Please note that you will not be able to remove a PC from the Registered PCs list if you are connected to the Syrinx SP Configuration program with that same PC.

 

2. When attempting to connect to the database, you receive the error message:  “Unable to connect to server: [DBNETLIB][ConnectionOpen (Connect()).]SQL Server does not exist or access denied. (-2147467259).”  This error normally indicates that either the Syrinx connection details are incorrect or the PC cannot connect to the server over the network or the appropriate ports have not been opened on the database server’s firewall.   

If the connection details are correct, you should then verify that the PC can connect to the database server on the network by pinging the server or browsing to the server.  If you are unable to establish connection to the server by either of these methods, you could attempt to ping or connect to the database server using its IP address instead of its computer name as name resolution may not be working properly on the network.  If you are still unable to establish any sort of connection to the server, your IT administrator should verify that the PC has been linked correctly to your internal network and can communicate with other computers on the network.

If you can access the server on the network, you should then verify that the connection attempts are not being blocked by a firewall.  You should enable exceptions on the firewall for the SQL server service and also the SQL Browser service as inbound access to SQL server is not configured by default when the SQL database engine is installed on the server.

 

3.  When attempting to connect to the database, you receive the error message:  “Unable to connect to server: Login failed for user 'user'. (-2147217843)”.  This normally indicates that either the username or password is incorrect.  Please check with your System Administrator and verify that they have provided you with a valid username and password for the Syrinx system.  Another common cause is having CAPS lock enabled on your keyboard as username and passwords are case sensitive.
