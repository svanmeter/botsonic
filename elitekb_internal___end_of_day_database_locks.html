INTERNAL | End of Day Database Locks




This article will go over what End of Day database locks mean. It is not recommended to clear a lock but it is important to understand what each lock does. Below are the definitions of each lock. 



Topics in this article include
 

  • Database Lock Definitions
  • Lock Number Definitions





Database Lock Definitions

400 - Count drawer is done.  Asked, "Are you done?".  No more prompts. Update Daily Totals, Update Register, Register file lock cleared.

401 - Move Daily Totals to Compare Totals.

402 - Create Totals Summary.

403 - Close Credit Card Batch, Unlock Totals file, Create Transaction Edit "E" Close on Terminal xxx.

404 - Create Deposit Slip, Print Deposit Slip.

405 - Print End Of Period Reports - current store.

406 - Print Register Summary.

407 - Check all stores closed.

408 - Print "Totals Report".

409 - Update values: Package Rates, Quantity Out, Customer Balance, Create Service Contracts.

410 - Clear End of Periods such as End of Month and End of Year, Store 1 - clear all locks.

411 - from this point forward, only remote stores create the locks, Update Dates table.

412 - Print GL Distribution, Print Category Income Report, Print End of Period Reports - Store 000.

413 - Send System Notifications.

499 - Send MassFax/Emails.

0 - lock 499 cleared, Rebuild locks table, Run PORFileSynch if expiring in next 19 days, Run PORFileSynch if it is their date to run - once every two weeks If Backup\EndOfDayBatch.bat or Backup\EndOfDayBatchXX.bat (for multi-company) exists, it is run.
