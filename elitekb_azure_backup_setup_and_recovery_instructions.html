Azure Backup Setup and Recovery Instructions

 



 

Overview

Azure Backup for offsite backups. This article covers the sign-up and configuration of an Azure account and backup client.

What is Azure Backup?

 

Azure Backup is the Azure-based service you can use to back up (or protect) and restore your data in the Microsoft cloud. Azure Backup replaces your existing on-premises or off-site backup solution with a cloud-based solution that is reliable, secure, and cost-competitive. Azure Backup offers multiple components that you download and deploy on the appropriate computer, server, or in the cloud. The component, or agent, that you deploy depends on what you want to protect. All Azure Backup components (no matter whether you're protecting data on-premises or in the cloud) can be used to back up data to a Recovery Services vault in Azure [1]. Store plans start from $5/month. For Storage Plans and Pricing, please visit: https://azure.microsoft.com/en-us/pricing/details/backup/

I. Create Azure Account

 

1. Go to https://azure.microsoft.com/en-us/services/backup/

 

 

2. Click the Get Started button.

 

3. An Azure account will be needed to use Azure services. Azure provides 12 months of free services and $200 to use for 30 days.

 

 

4. Click Start free.

 

5. This will take you to the Microsoft Sign In page.

 

 

 

6. If you already have a Microsoft account go ahead and sign in and skip to step 20. If you do not have an account go on to the next step.

 

7. Click Create One!

 

8. Input your email address on the following screen.

 

 

9. Input a password for your new Microsoft Account and click Next.

 

 

10. Microsoft will need information about you. Go ahead and fill the next form out.

 

 

11. Input a good number to verify it is you. Click Verify once you receive the code.

 

12. Input your credit card information on the next step.

 

13. Check the agreement box and click Sign Up.

 

 

14. If you get the following message, click Sign Up not enable the Azure subscription.

 

 

15. Choose your payment information.

 

 

16. Select No technical support on the next screen unless you wish to have the technical supports offered and click Next.

 

 

17. Check the agreement box and click Sign Up.

18. Once the subscription is done setting up you will be logged in to the Azure Portal.

 

 

 

II. Create a Recovery Services Vault

 

1. You will first have to create a Recovery Services vault.

 

2. log in to your Azure Portal https://portal.azure.com.

 

3. Click on All Services.

4. Type “recovery” in the search box.

 

5. Select Recovery Services vaults.

 

6. On the Recovery Services menu click Add.

 

7. Enter the desired name, subscription, resource group, and location of the recovery services vault.

8. If you do not have a Resource group available you can click Create new to bring up the pop-up window to create one.

 

 

9. Click OK on the pop-up window.

 

10. Once all required fields are filled go ahead and click Create.

 

11. The deployment process will begin. Wait a couple of minutes until it is complete.

 

 

12. Once the deployment is complete you will see the newly created recovery vault listed.

 

 

 

III. Setting Storage Redundancy

 

When a new recovery services vault is created it must be configured on how storage is replicated for redundancy. Microsoft currently offers four options: locally redundant storage (LRS), Zone-redundant storage (ZRS), Geo-redundant Storage (GRS), Read-access geo-redundant storage (RA-GRS). By default, when a new recovery services vault is created it is set to Geo-redundant Storage (GRS). GRS is the second-best redundant option and it is recommended to leave it as is. The price varies depending on the replication option.

 

For more information about storage replication please visit https://docs.microsoft.com/en-us/azure/storage/common/storage-redundancy. If you wish to change it you can do the following:

 

1. Click on All services.

2. Type “recovery” in the search box.

 

3. Select Recovery Services vaults.

 

 

4. Click on your Recovery Services Vault that you created in the last section.

 

 

5. You will see an overview in regards to the recovery vault.

 

 

6. On the middle panel scroll down until you see Backup Infrastructure. Click it.

 

7. Click Backup Configuration.

 

8. Select the desired backup configuration.

 

 

IV. Vault Configuration

 

1. Click on All services.

2. Type “recovery” in the search box.

 

 

3. Select Recovery Services vaults.

 

 

 

4. Click on your Recovery Services Vault that you created in the last section.

 

 

 

5. You will see an overview in regards to the recovery vault.

 

 

6. On the middle panel scroll down until you see Backup. Click it.

 

7. A panel will open asking you where you will be running the workload and what you would like to backup.

 

8. Select On-premises and Files and folders.

 

 

9. Click Prepare Infrastructure. A new panel will open.

 

10. Download and save the agent.

 

11. The agent gets installed on the server, but before then you must download the vault credentials. Check the box and download the vault credentials.

12. Do not open the credentials, and if you accidentally open the credentials let the dialog that attempts to open the vault credentials fail.

 

13. If you are not on the server that will be backed up, go ahead and transfer the files downloaded from the previous steps.

 

 

V. Install and Register the Agent

 

1. Run the MARSagentinstaller.exe that you downloaded from the previous step.

 

 

2. The installer will extract all files and follow up with the wizard.

 

3. Take the defaults for the Installation Folder and Cache Location unless you wish to change the path to something else.

 

4. Click Next.

 

5. Click Next on the Proxy Configuration screen.

 

 

6. On the Installation screen make sure that the required software is all green check-marked and click Install.

 

7. The wizard will now begin the installation.

 

8. Once completed, it will ask you to go to register the software. Click Proceed to Registration.

 

9. On the Vault Identification window it will ask for the location of the downloaded vault credentials from the previous section. Browse to where you placed the vault credentials.

 

10. Click Next.

 

11. On the Encryption Setting you will be asked to put a passphrase. You can either create a minimum of 16 characters in length or have the wizard generate a passphrase for you.

 

12. You will also be asked to save the passphrase somewhere. Your passphrase will be needed if you ever need to recover your data. If you lose this passphrase your data cannot be recovered. Store this passphrase somewhere safe.

 

 

13. Click Finish.

 

14. The wizard will begin the server registration.

 

 

15. Click Close once it is finished.

 

 

VI. Create an Azure Backup Policy

 

1. Open the Microsoft Azure Backup agent.

 

2. On the Actions section, click Schedule Backup to launch the Schedule Backup Wizard.

 

3. The backup wizard windows will come up.

 

4. Click Next.

 

5. On the Select Items to Backup window select the files and/or folders that you wish to be backed up. We recommend backing up the whole POR directory if possible.

 

 

6. Click Next.

 

7. We recommend setting the backup schedule to run once daily at 1:30 AM or anytime when the system is not busy and is not being used.

 

 

8. Click Next once you set the time.

 

9. On the Select Retention Policy (Files and Folders) screen specify the retention policy for the backup. Since we are doing daily for now, we recommend unchecking the weekly, monthly, and yearly retention policy and to set the daily retention to 60 days.

 

10. Click Next.

 

11. Select Automatically over the network on the next screen and click Next.

 

 

12. Click Finish to confirm the backup settings.

 

 

13. Click Close once the modification progress is finished.

 

 

14. To ensure that the backup policy is working, click Back Up Now on the Actions.

 

 

15. Click Next on the Select Backup Item window.

 

 

16. Click Back Up to confirm the backup.

 

 

17. Once the backup is completed, click Close.

 

 

18. To ensure that the scheduled backup is working as intended, open the Microsoft Azure Backup UI and verify that you have successful backups on the time the backup is supposed to start.

 

 

 

VII. How to Recover Files

 

1. Open the Microsoft Azure Backup agent.

 

 

2. On the Actions section, click Recover Data to launch the Schedule Backup Wizard.

 

 

3. The Recover wizard will initiate and will ask you which server you would like to retrieve files from.

 

4. Click Next.

 

5. You can select what type of recovery you want. For this example, we will choose Individual files and folders.

 

 

 

6. Click Next.

 

7. Select the volume you wish to recover.

 

 

 

8. The wizard will provide details on the backups available. You can choose a different date (if you have backups for that date) to recover from.

 

 

9. Click Mount and the wizard will begin the process to mount the backup on your computer.

 

 

10. Click Browse to navigate through the files backed up.

 

 

11. This will open a file explorer window where you can recover specific files from the date and time the backup took place.

 

 

12. Once you are done, click Unmount.

 

 

13. And confirm the unmount.

 

 

The Microsoft Azure Backup UI will keep track of any backup and recovery attempts for your records.
