How to Integrate with Point of Rental

Here's how to get there
Elite: Configuration Gear > App Configuration > Accounting Configurations > Account Numbers

To integrate with your Point of Rental system, you will need to go into your accounting numbers configuration for store 1. 


Under Miscellaneous choose the API – MS Dynamics GP and click on Settings. After that fill out the required information needed.




After that is done, click OK and restart the program.

 

Setup Tax Schedule

The tax schedule for the vendor is read directly from the GP SQL database rather than through eConnect. The tax detail in GP breaks out the components of the tax schedule assigned to the vendor. This will be needed for your accounts payable and purchase orders.



 

 

 

General Ledger Sync to Great Plains (GP) Flow
 

1. Validate balanced entry

2. Serialize GP Journal Transaction

a. Get next JE number from GP

b. Populate JE header

i. JrnEntry = JE number from GP

ii. BachNumb = JE type code + JE PK

1. Tnnnnn = Real-time transaction entry

2. Dnnnnn = Bank Deposit entry

3. Bnnnnn = Batch entry

iii. TrxDate = Real-time event date, deposit date, or batch closing date

c. Populate JE distribution

i. Load the JE detail lines from the GL queue.

ii. Add a distribution line for each JE detail line.

1. ActNumSt = GL Account from detail line

2. Dscriptn = Memo from detail line

3. DocDate = TrxDate from header

4. DebitAmt = Detail line amount, if the amount is greater than 0.00

5. CrdtAmt = Detail line amount, if the amount is less than 0.00

3. Export the JE Transaction via eConnect

a. If successful, update the GL queue record as Posted.

b. If not successful, update the GL queue record as Error and update the error description. 

 


Purchasing/Accounts Payable (AP) Integration

Independent of the GL Mode, completed purchase orders are exported to GP as purchasing transactions, which result in vendor payables. Verified PO's are added to the export queue when they are finalized.
 

Accounts Payable to Great Plains Flow
   

1. Validate PO amounts

a. Sum of line item detail = Total PO amt - PO Tax amt - PO Shipping amt - PO Misc amt

2. Load PO Vendor record

3. Serialize GP Vendor update

a. This will create the GP Vendor card if the Point of Rental Vendor record is not mapped to an existing GP Vendor. (Vendor mapping:  Point of Rental VendorFile.Gllink = GP Vendor.VendorId)

b. These fields on the GP Vendor card will be updated from Point of Rental if the Point of Rental Vendor record is mapped to an existing GP Vendor:

i. Name

ii. Contact name

iii. Account number

iv. Notes

v. Minimum order

vi. Address

vii. Phone

viii. Contact 1 Phone

ix. Contact 2 Phone

x. Fax

4. Serialize GP PMTransaction

a. Get next voucher number from GP

b. Add tax to the PM Transaction

i. Set PO Purchase Amount = (Total PO amt - PO Tax amt - PO Shipping amt - PO Misc amt)

ii. Check for a tax schedule with the required detail assigned to the GP Vendor.
 

This step requires the configuration of the GP database server, database, and credentials in Point of Rental. The tax schedule for the vendor is read directly from the GP SQL database rather than through eConnect.
 


The tax detail in GP breaks out the components of the tax schedule assigned to the vendor.
 

 

5. If GP Vendor tax schedule was found

a. Calculate GP Tax amount for each detail line = Detail line % * PO Purchase Amount (rounded to 2 decimal places).

b. Accumulate total tax as each line item is calculated.

c. Add the tax node to the PM Transaction.

6. If no tax schedule is assigned to the vendor.

a. No tax node is added to the PM Transaction.

b. TaxAmnt on the PM Transaction is populated with the Point of Rental PO Tax amount .

7. Populate PMTransaction data.

a. VendorID = New or existing Vendor Id from step 3

b. DocNumbr = Vendor invoice number

c. Pordnmbr = Point of Rental PO Number

d. DocAmnt = PO TotalAmount

e. ChrgAmnt = PO TotalAmount

f. DocDate = PO Vendor Invoice Date

g. PrchAmnt = PO Purchase Amount = (Total PO amt - PO Tax amt - PO Shipping amt - PO Misc amt)

h. TaxAmnt = Total tax from the tax node added above or the PO tax amount from the PO record.

i. FrtAmnt = PO Shipping cost

j. MscChamt = PO Miscellaneous amount

k. DocType

i. If PO Total Amount > 0, then 1, Payable

ii. If PO Total Amount < 0, then 5, Credit Memo

l. Build PM Distribution (the PM Distribution adds the line item detail from the Point of Rental PO)

i. Add a line to the distribution for each line item in the PO detail.

1. DistType = Purchase

2. DistRef = PO line item reference

3. ActNumSt = GL account number assigned to the PO line item

4. DebitAmt = PO line item amount, if the amount is greater than 0.00

5. CrdtAmt = PO line item amount, if the amount is less than 0.00​​​​​​

ii. Add a line to the distribution for tax.

1. DistType = Taxes

2. ActNumSt = PO Tax GL account assigned to the PO store

3. DebitAmt = Tax amount calculated above, if the amount is greater than 0.00

4. CrdtAmt = Tax amount calculated above, if the amount is less than 0.00

iii. Add a line to the distribution for freight.

1. DistType = Freight

2. ActNumSt = PO Shipping GL account assigned to the PO store

3. DebitAmt = PO shipping amount, if the amount is greater than 0.00

4. CrdtAmt = PO shipping amount, if the amount is less than 0.00

iv. Add a line to the distribution for miscellaneous.

1. DistType = Misc

2. ActNumSt = PO Misc GL account assigned to the PO store

3. DebitAmt = PO misc amount, if the amount is greater than 0.00

4. CrdtAmt = PO Misc amount, if the amount is less than 0.00

v. Add a line to the distribution for the vendor payable amount.

1. DistType = Pay

2. ActNumSt = Accounts Payable GL account assigned to the PO store

3. DebitAmt = PO Total amount, if the amount is less than 0.00

4. CrdtAmt = PO Total amount, if the amount is greater than 0.00
