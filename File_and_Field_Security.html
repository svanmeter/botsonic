File and Field Security

 

This selection allows you to establish the security level required to view and/or change the data file and each field within the data file.

First, you must enter the owner's level password.  In order for the changes to be implemented, you must Quit out of the Program Menu at all other terminals.  Next, you will be presented with:

 

EXIT returns you to the Security Menu.  The other selections specify which data file you wish to change.  Next, each password level will be shown along with File Maintenance functions permitted on the file.  You may change the allowed functions to define which password levels may Inquire, Modify, Add, Delete, or Merge records.  **Merge is only available for Item and Customer File.**  After you have made any required changes click OK to continue to the next screen.

                                 

 

The Field Numbers and Names of each field for the selected file will be displayed on the screen, along with its current security level for both Display and Modify, and if data is required to be in the field. 

    

 

Click on the field you wish to view.  You are able to specify the new Display security code followed by the new Modify security level.  These codes define who can enter or view data in the field.  The Field Length is also viewable.  This cannot be changed but you can see the maximum amount of characters that can be inputted into the field.  Click OK to exit the Security File Menu. 

 

**If the password level is set to Not Allowed, you will not be permitted to change it, nor will you be permitted to display/modify that field. After you have entered the new password levels, click OK and you will be returned to the Field listing.

By default, the Password to Modify the ‘inc life’ field (Income Life) is set to Not Allowed. This means when accessing the Customer Record, the Life To Date field will be greyed out and unmodifiable.
