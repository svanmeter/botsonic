Web Pay Troubleshooting Guide



Overview

This guide will go over the basic steps for troubleshooting when payments using web pay do not come through to Point of Rental. This results in credit card inquire becoming out of balance.
 

Note: Do not do a credit card bypass before following the below steps. This can cause double payments to happen.
 

1. Go to update the website and ensure it is working properly.
Expert: Configuration > Other Updates > Update Website 
Elite: Configuration Gear > App Configurations > Other Updates > Update Website

a. If you receive an error, more than likely the IP address has changed.

i. Go to porhelp.com on the customer’s server side. Select Show My IP Address. Make a note of the IP address and create a case for Rental Hosting with the new IP address in the case.


2. Verify that the task runner is turned on.
Expert: Configuration > System Configuration > Task Runner
Elite: Configuration Gear > App Configurations > System Configuration >Task Runner


 

a. The task this service uses is Quote Import. If this does not show, then search for the task using the Show Inactives box. Edit the task to uncheck the inactive box.
 


3. Verify that the import file does not show as an XML file. This can be found on the POR Folder on the customer server. This will be the location of the file. POR: Pictures: Quote: WebPay.

 

a. By design the file should automatically be saved as a .imported file type.

 

b. If it does show as an XML you can open the file via Notepad to see the customer number and contract number to notate which customer the payment belongs to.
 


c. In the Windows search bar on the customer’s server side, search for the program called RUN. Then enter this path 'C:\POR\Program\aracs.exe DDE:WebPay.' Press OK. This will reimport the web pay file. The C in the path name is the drive letter of where POR is stored on the customer’s server.
 


4. When running a credit card inquire, the merchant transaction field will show 1234. That will represent the web payment for that entry. Web Pay will always post to store 1. There are a few customers who have their web pay configured to be store specific.
