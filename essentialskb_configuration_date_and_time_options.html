The date and time options in Configuration allow you to set your grace periods for how long customers have to return items before late fees accrue. You will also have the ability to set expirations for Quotes that have not been converted.
 

To get to this area, go to Configuration > Company Configuration > Date/Time Options.

 

 

Hide Start Time: If you only rent in whole days and do not need to know when your client is picking up the item, you can hide the start time selector.
  

Hide Due Time: If you only rent in whole days and do not need to know when your client is returning the item, you can hide the due time selector.
 

Show event start and end time: If checked, start and end times will show up when creating contracts, and will be shown on contracts if available.
 

Send Out - Reset Start Time Default Value: Enabling this feature tracks a more precise real-time rental period. When enabled, this feature auto-changes the start date of the rental period to the time the contract was marked SENT OUT.
 

Workbench - Always Show All Transactions: You can set the default Workbench overview to just see the current week. With this set to yes, then all transactions will show including past contracts.
 

Default Transaction Duration: This allows you to set a predetermined period as the default transaction period upon contract creation. 
 

Note: Most commonly seen in event rentals for a weekend rental period.
 

Closed Day Rollover Time: Time on a closed day when rollover should occur to the next day’s rental period.
 

Overnight Rollover Window: Duration after the close when rollover should take effect.
 

Due Window: Duration after open when items are due back.
 

Grace Period Percentage: Percentage of the rental period that will be considered part of the grace period. 
 

Example: Entering 50 would mean there would be a 1-hour grace period on a 2 hour rental.


Start Time Delay in minutes: With this option, if you want to send an item out at the counter, but want the start time to be fixed later, you can enter how much later that time should be.
 

Note: Setting should not be set to be more than 15 minutes.
 

Grace Period Minimum: Settings are in minutes. The least amount of time customers can bring back items before late fees accrue.
 

Grace Period Maximum: Settings are in minutes—the most amount of time a customer has to return items before late fees accrue.
 

How old should a quote be before it is canceled: Quotes will be canceled if the start time and due time are older than the specified duration.
 

When canceling old quotes should old unpaid reservations also be canceled: If checked, then both quotes and reservations that are older than the duration specified for quote expiration will be canceled.
 

Default Transactions to start at the current date and time: If checked, contracts going out will default to the current date.
 

Don’t allow variant due dates: When creating a transaction, the Duration and End Time fields will be forced to contain equivalent values.
 

Max Rental Duration in days: Rentals may not exceed these many days in total length. This configuration setting will also enable the configuration setting ‘Don’t Allow Variant Due Dates.’
 

Maximum period to be considered a Last-Minute Reservation: New reservations with less than this time until the start date will be considered last minute. Under User Settings, you can set up a notification to be sent to your inbox as well.
 

Browser Idle Timeout (in minutes): Determines how long an employee using the Counter Mode can be inactive before they are logged out. The maximum idle timeout is 60 mins.
 

Purchase Availability Delay Duration: The period will be used to apply automatic availability delay to items after purchase. If the field is blank or zero, then there will be no delay applied.
 

Enable manual stock activation for automatic purchase availability delay?: This option will allow you to go to the item and manually place the item into stock.
 

Delivery/Pickup Windows: When marking a contract as either a delivery or pickup, the counter person needs to select the delivery/pickup date and time. The time can be selected from a drop-down box that has multiple time windows defined. Those delivery times are defined in this section. See our documentation on Delivery\Pickup windows for more information about this feature.
