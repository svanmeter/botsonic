Overview

The following process will detail how to create a global database entry in the CRM for a database and then how to link users/staff members within your demo Syrinx database with global user accounts for use with the POR One app or Syrinx 365 application as well as creating the required patches\files for enabling a Syrinx database for Syrinx 365 or the POR One App.

Create Global Database Entry and Create Global Admin Account

Find the customer record in the CRM and add a contact record for whomever will be the global admin for the customer and verify that an email address has been set against that contact.  Next, select the database entry that is going to be enabled for Syrinx 365/POR One access and then click on the “Global Databases” tab of the database record:

Then use the “Add” button to begin creating a global database entry for that specific database:

Once the database entry is created, you can now create the patches and json file that need to be used as per the Syrinx 365 API installation guide.  The Syrinx 365 API and json file are created via the “Generate DB Patches” button on the Global database record and the Syrinx 365 application patch is generated via the “Patch License Info” button (You do not need the Syrinx 365 application patch if you are just installing/testing for the POR One app.  You only need this if you want to use global logons or global messaging in Syrinx 365 desktop):

Note: Each installation of a database will require a unique “name” when creating the global database entry.  So, if you have the same database running on two separate PCs (i.e. one as live and one as demo), you will need to create a global database entry for each of those installations and the patches/json file created during this process should only be used against that specific databases and should not be applied to any other version of that database on a separate PC.
 

The global database entry will also need to be enabled for an Operator app license in http://gadmin.pointofrental.com/ via the “Licenses” tab of the global database record:

 

The “Operator App” license should be enabled and set as a “site” license:

Once enabled, an “Operator App” column should be visible on the “Users” page in gadmin:


Note: If the Customer does not already have a valid Operator App license from the HCS CRM, this will also need to be generated and applied to the Syrinx database.

For pre 4.41.1 systems, the customer contact who will be the customer’s Global Admin should then be set as the main contact for the global database by selecting the global database entry and then clicking the “Add” button on the “Global Database Users” screen that is available to the right of the database entry (if you’re zoomed in on the CRM, you might not see it and might need to zoom out):

This will then create or link that contact’s Global user account to the global database and, if they do not have a global account yet, it will show the default password for the account and will potentially email the contact a new user link:

After getting the required patches, json file and creating the global database “admin” user, you will then want to install the Syrinx 365 API and make sure that is up and running properly before beginning the next step where we will link global users to existing users/staff members in the database.

 

Creating Global User Accounts for Local Logon Users and Staff Members in Syrinx 4.41.1 or Later

Global User accounts for logon users and staff members can be created in the Syrinx client for Syrinx365 and POR One Access.  Users can create thei own Global users for staff members and/or user logons once the Syrinx database has already been registered with Global via the CRM and the appropriate patches have been applied to the Syrinx database.

 

A new "Global User" button will appear on the "Amend Staff Member Details" screen and will be enabled only when an email address value has been populated:

If the staff member is not already present in Global, the system will attempt to create a Global user when the button is pressed.  After creation, the Global “PersonId” will be saved against the staff member record in the Syrinx database.

 

On creation of a new Global user, an email will be sent by Global as below if that user has not already been linked to a Global database entry:

 

If the staff member is already present in Global, then the user will be presented with the option to update or delete the user:

Selecting "Update User Names" will update the first and last names of the Global user entry with the current First/Surname values set on the staff member record.  Selecting "Delete User" will remove the user's link to the Global database instance.

 

The same functionality exists for logon users in the Syrinx configuration program:

A new general system option has also been added to the "Misc(2)" tab of Syrinx SP Configuration\System Options\General which will allow a user to override the Global endpoint URL but this should not be necessary in most cases:

Once global user accounts have been created, staff members who are operators should be able to connect via the Operator app (POR One) and logon users should be able to connect to the Syrinx 365 desktop application using a global logon instead of a SQL logon as logon as the Syrinx365Api has been installed and the associated SQS listeners are running.

 

Creating Global User Accounts for Local Logon Users and Staff Members on Pre-4.41.1 Systems

Once Syrinx 365 API installation is complete, the customer contact can then use their Global Database account to log into the global admin site: http://gadmin.pointofrental.com/ : 

 

If it detects a user logon in the Syrinx database with the same email address, it may issue the below prompt and the user should click “Yes” to link their global user account with the local user account in the database:

Note: There is currently a bug where it will fail to link your logon to local user account in your Syrinx db if your email address is linked to a Syrinx User Logon as well as a Staff Member record so make sure that the customer’s email address is only linked to one User Logon in your Syrinx db and is not linked to any staff member records.

 

You can then use the “Register New User” option to create global user accounts for the staff members/logon users that are returned to the global admin site via the Syrinx 365 api:

 

Note: If you have not installed the Syrinx 365 api properly, there will be no users/staff members in the list as the global admin site is not able to connect to your local database to retrieve the list of users/staff members

 

To register users/staff members, the user simply needs to select them from the list and click the “Register” button at the bottom of the screen (The operator IDs for logon users will be prefixed with “U” and staff member operator IDs will be prefixed with “S”).   Any users selected should have valid email addresses before attempting to register the account:

 

 

The user should then appear in the list of linked users for the global database:

 

 

Once global user accounts have been created, staff members who are operators should be able to connect via the Operator app (POR One) and logon users should be able to connect to the Syrinx 365 desktop or web applications using a global logon instead of a SQL logon.
