Transfers - Elite Browser

 



Security and configurations
 

  • Default security allows a user to Create and Update Transfers at the Company, Region, District or Store level.  If you are using Regions and Districts, you can control which locations a user can Create and Update by setting Company to No and then any others as needed.  Having them all set to No will hide the options to create or update a Transfer.
  • Transfer Types - System default will expose the One Way Transfer option in the main search options.  If a system also wants to expose Round Trip Transfers, the Transfer Types configuration can be set to Both.  Or, only Round Trip.

  • Transfer Delay - there are two options within this configuration (located in the Core configuration interface).  
    • One is to set, in hours, the default transfer period between two locations.  This will be the default for the transfer period.
    • The other is to set ‘No Transfer’ should be done between these locations.  In browser transfers, this will hide that location as an option when selecting the transfer customer.
    • If no transfer delays are set, the system will default to your default rental period set in parameters.  If neither is set, this value will be 1 hour.


 

Transfer Customers
 

When setting up ‘transfer customers’ (which are really your store locations) they must be configured with a key number that is formatted with ‘XFER-<storenumber>’ ie: ‘XFER-2’.

 

Transfer options: One Way or Round Trip

 

‘One Way Transfer’ is the typical method used.  The purpose of this is to change the (current) location of an item (or quantity of bulk items) until they are moved back or to another location.

 
  1. Create a new One Way Transfer
  2. Select the store you are transferring to
  3. Set the status - the status can be Planned (the transfer will happen in the future) or Open (the transfer is happening now).
  4. Ensure a Transfer Period is set (see Configuration options above to auto-set it)
  5. Add items to the transfer
    1. When the status is Planned, sales items will default to ‘In Stock’ but can be pulled from the Action options.  All sales items will automatically flip to pulled when the transfer is set to Open.
  6. Select Done to save the Transfer 
  7. Closing the Transfer while logged in as a user of the destination location will move the items to that location.
    1. Serialized items will see an update to the Current location
    2. Bulk items will increase in Quantity by the amount transferred to the location.  And decrease in Quantity in the Transferred from location.

‘Round Trip Transfer’ is less commonly used and is for bulk items.  The way this works is that it will set a positive line for the lending store (decreases available) and a corresponding negative line for the receiving store (increases available).  The Transfer is left in the open state until they are returned to the lending store.  Then the transfer is closed and available quantities are adjusted back. Round Trip transfers are only for bulk rental items where a Header record is related.
