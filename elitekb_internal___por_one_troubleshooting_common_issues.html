INTERNAL | POR One Troubleshooting Common Issues


 

In this article, we are going to show you common troubleshooting issues that might occur for the Elite and Syrinx platforms. POR One runs off a Global API Service on your server.



Topics in this article include
 

  • Users cannot log into the application

  • Users are missing a particular feature when they log into the application

  • The page doesn't load or show any information

  • POR One App API Error

  • Syrinx and Syrinx365

  • Elite 



 

Users cannot log into the application


There may be situations when users cannot log in to the application. Here are common troubleshooting steps you can take.


 

Forgot Password
 

Passwords can be reset by the user by clicking on “Forgot Password” on the login page or they can be reset by POR Staff.
  

1. To reset the password for a user, log into GAdmin with a Point of Rental Staff account.
 

2. Click on Organizations.
 


 

3. Search for the user’s organization. After you find it, click on the company name.
 


4. Click on the “Users” button. This will take you to a page with all the registered users for an organization. 
 

 

5. Once you find the user (the one with their email), click on the Edit Icon and click on Reset Password.
 


 

Check the User's Registration
 

Assuming the user hasn’t just forgotten their password, you can check to see if they have been registered to an organization by logging into GAdmin.

 

1. Login to GAdmin using a Point of Rental Staff Account.
 

2. Click on Organizations.
 


 

3. Search for the user’s organization. After you find it, click on the name of the company.
 

 

4. Click on the “Users” button. This will take you to a page with all the registered users for an organization.
 



5. If they don’t appear on the list you can register them manually by clicking on the Register New User button on the user's list.
 



 

Users are missing a particular feature when they log into the application


If the user is registered to multiple organizations, make sure that they are logging in to the correct organization where the feature should be enabled.  

 

1. Login to GAdmin using a Point of Rental staff account.
 

2. Click on Organizations.
 


 

3. Search for the user’s organization. After you find it, click on the name of the company.
 


 

4. Click on the Licensing button. They need to have licenses for the feature and the user needs to be allowed to access the feature.
 

 

5. Then check a user’s access to a feature by clicking on the Users button from the organization page.

 

6. Next, check if their account has access to the feature.

The page doesn’t load or show any information
 

If a page in the app displays any kind of error message or does not load any information, first assume the user does not have a connection. This could be a lack of a general connection to the internet or to a server that is not available.

 

  • If it isn't a lack of internet connection, then you can check to see if the user’s remote server is online.
    • Log into GAdmin (POR Account) and check the organization’s online status by clicking on the globe icon.
 

 

  • If the globe icon is red, it means that we did not find the server online. If the globe is green then they are online.

 

  • If the server is showing as offline you will need to get access to the remote server that is hosting the API and SQS service. Depending on their organization type (Expert or Syrinx) their on-premise server’s API and SQS service setups will differ. Both APIs should be apps that are hosted on IIS and can be managed and configured via IIS Manager (7.0+).

 

Note: The organization type for Expert is for Elite Customers.

 


POR One App API Error
 

If you get a message stating there is an API Error Message on the POR One App, this means that the SQS Service on the server is not communicating with our API Server. The recommendation is to reset the POR SQS Service on the server. You will need to be logged in as an administrator to perform this action. 


 

Syrinx and Syrinx 365
 

  • The default name of the Syrinx API should be “Syrinx365Api” and an application pool with the same name. The application pool name doesn’t always have to be the same you can always just right-click on the application and select Manage Application → Advanced Settings and check the actual app pool.

 

 

  • If the application pool has not been started, it needs to start for the API and SQS service to run. You can right-click on the app pool or use the taskbar on the right to start it.

 

 

  • Syrinx’s SQS service is built into the API and will only start after the API has been “connected to” once you can just open the API’s landing page (http://localhost/Syrinx365Api/index.html, Syrinx365Api is the app’s name in this case but the name will depend on what it has been named as in IIS) and “start” up the API.
     

  • If you still cannot see a “green globe” in GAdmin for this server then the issue might have to do with the API’s security credentials and you will need to speak to a developer from the appropriate team to further debug the issue.  Syrinx API also provides a logs folder that might contain log files with useful debugging information that can be provided to the developer to assist in debugging

 


 

Elite


Similar to Syrinx in the respect that it is also an app hosted in IIS and has an SQS service that connects the local API to Global API, however, Expert’s SQS is not part of the API but runs as a separate service.

In Elite, Relationship ID in GAdmin goes to the OperatorID table relationshipID field, the Remote user ID in GAdmin goes to the Employee system-generated number. For the organization in GAdmin, the Remote Company ID field should be the POR customer number in Parameters.
