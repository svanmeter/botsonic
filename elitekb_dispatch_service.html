Dispatch Service

 

Overview: Configuring the Dispatch Service to autorun will allow your routes to be sent to TSO overnight. This eliminates the need for someone to manually open the Dispatch Center each morning for your trucks to receive their route information.

1.   Program Menu > Configuration > Store Configuration > Configure Task Runner

2.   Ensure that the Task Runner is running in the bottom right corner under the Status section.

a.   It not go to your server and login as an administrator and Create / Start the service.

3.   Click the Add Task button.

4.   Create a name – Dispatch Service

5.   Create a description – Dispatch Service runs at 3 am

6.   Click the button to the right of the File to Run box and navigate to x:\POR\Program\DispatchService.ex.(where x is the drive letter Point-of-Rental is installed on your server).

7.   Click the button to the right of the Start in Path box and navigate to x:\POR\Users (where x is the drive letter Point-of-Rental is installed on your server).

8.   Command Arguments should be set to ’RefreshRange’ (without the quote marks)

9.   Recurrence is set to ‘Days’

10.  Interval is set to 1 (every 1 day)

11.  Click Save and answer the prompts that follow.

 

Based on your settings, the Dispatch Service will run and send the necessary information to TSO Mobile allowing routes to be uploaded and ready when drivers get in their trucks.

Information passed to TSO is as follows:

Description- Customer, Delivery/Pickup company name, contact name, and number

Name- Trip, route#, sequence, contract number, and promised arrival
