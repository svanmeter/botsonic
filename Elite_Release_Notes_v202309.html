We are listening!  Items with this label are a result of your suggestions in uInnovate.  Please continue to share and support those ideas that help your business.

 

  We recommend reading these and understanding the impact prior to updating.  These features may require adjustments to security or configuration and may have implications for your operations or current workflows. 


  
 


Sorting on Delivery/Pickup Dates Column in Day at a Glance:  You know that thing?  Where you’d sort on the delivery/pickup column?  And the navigation bar would leap to the right?  Yeah, we fixed that. 

 

Browser Availability Store Drop Down:  This store drop-down was found to not conform to the store, district, region, or company user permissions.  Now it does and will match the store drop-down found in the item search grids.

 

Automatic Report Selection Criteria:  The selection criteria for automatic reports have been extended for SQL users past the 255 characters constrained in a JET db.  As a reminder, selection criteria can now be extracted from any crystal report generated to the screen by selecting the appropriate export icon at the top left.  This makes it very easy to set up an automatic report that has just what you’re looking for. 

 

Call Log Notifications from POR One:  Users can set up system notifications to be sent when a call log is created for a specific call log type or customer.  Now, call logs created against either of these criteria in POR One will also send the notification(s) to the appropriate people.

 

Price Override Tracking Password Field:  On the tracking screen for system security, Elite users will now notice that the password drop-down is grayed out.  This is due to the fact that this security no longer applies to Elite users.  Instead, ad hoc pricing security should be used.  However, price overrides will still be tracked in the edits for the boxes that are selected.

 

  Sales Rep in Dispatch:  The sales rep column was still labeled ‘salesman’ in our dispatch center.  It has been updated to ‘Sales Rep’.

 

            

POR One Release Notes
        
These are updates for Elite users utilizing the POR One app.

 

Linking Serialized Items on Task: When completing a task on POR One, such as Driver Delivery or Customer Pick up, and the item has not been serialized in Elite the user has the ability to select a serialized item from POR One. Once they have done this, the specific item page can now be viewed by tapping on the Item Name. This allows you to see all of the item details to determine if it is the correct item to associate with the Contract. 


Mobile Permissions Improved Design: The Mobile Permissions that were recently added to Elite have been updated to be even easier to use and navigate. Clicking on the menu icon next to any of the usernames will show additional details such as Email, Relationship ID, and Remote User ID. The last two don’t sound important but may be requested by Support when helping you troubleshoot. 


FIX Serialized Image Tagging: When adding a picture attachment for an unserialized item it was being associated with all of the serialized items erroneously. This has been fixed to only be associated with the item's serial number that has been selected.

 

 

Memory Lane Monday

This is a fun corner where we republish release notes from previous sprints you may have forgotten about.

   

Tax Exempt Customers: Users could always run the Expired Tax Number report to manage which customer's tax exemption status was about to expire, or had already expired. However, we wanted to give this old, dependable trail horse a bit of a nudge. Now, if the user is writing a contract to a customer with an expired tax exemption, they will be warned and the contract will be taxed. If the user has the proper security, they can always override this and set the contract to taxable.


Sequential Invoicing: A longtime request has been the ability for our users to assign a unique invoice number to a contract in addition to the Point of Rental appointed contract number. Well, friends, sometimes dreams really do come true. This option can be turned on on the accounting tab of your parameters. When this is enabled, a unique invoice number will be assigned to a contract when its income posts. We have added columns to the customer dashboard and account payments to display this information. Any credit adjustments made from the contract will also reference the invoice number. We have also added it to the contract format, statements, aged customer balance, and accounts receivable history reports. This is the first phase of implementation for this feature, and there will be more development to come.
