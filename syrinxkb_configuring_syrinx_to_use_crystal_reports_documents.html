Configuration

In order to use Crystal Reports template files rather than the inbuilt Syrinx documents some configuration changes must be
made. Firstly launch the Syrinx SP Configuration Program and select the ‘Documents’  Tab then the ‘Options’ icon in the Custom Documents section:

Tick the ‘Use Crystal Reports for document printing?’ option, and specify the network folder location where the report files will be kept. This is normally on the Syrinx server in the SPDocs\CRDocs folder. Please note that all Syrinx client PCs must be
able to read from this folder over the network:

You will need to enable the specific document type for which you will be using a Crystal Report template. For example, if you wished to use a customised Hire Contract document, you will need to select the ‘Hire’ tab, and tick the ‘Hire Contract’ box. 

 

You can enable as many docs as you wish as long as you have a corresponding Crystal Report file for that document type.
Finally you will need to copy the Crystal Report template (*.rpt) files into the shared folder location specified in the ‘General’ tab of the CR Documents screen (as in the second screen shot above).

Printing Crystal Report documents

Crystal documents are printed in exactly the same way as internal documents. The only difference is seen when Syrinx
presents the print preview change. This will look slightly different with a ‘Business Objects’ logo on the top right of the
page.

If you wish to export the document to a file e.g. xls, csv; Crystal provides its own export function that can be accessed

using the button on the top left of the preview screen. For exporting to PDF file, you can still use the ‘Export PDF’ button shown on the main Syrinx ribbon or you can also export to PDF from Crystal’s own export function but if you have configured Syrinx to add an overlay on PDF/Email documents, then using Crystals export function will not include this.

Please note that printing a Crystal document for the first time may take longer than usual. This is because the Crystal Reports runtime application must be loaded by the PC first. This should stay resident in memory for subsequent prints and therefore the print will appear quicker to respond.
