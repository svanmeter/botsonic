Overview

Syrinx can transfer customers, suppliers and sales/purchase invoices directly into Quickbooks, and can also pull customer balances, aged debt information and unpaid invoices back to Syrinx.  The transfer program works with Quickbooks and Quickbooks Pro, version 2008 and later, and uses the QBFC SDK (Quickbooks Foundation Classes) to integrate directly with the Quickbooks program.  There are different versions of the QBFC for the different versions of Quickbooks; for example, Quickbooks 2008 requires QBFC v7.

[Note:  for Quickbooks versions prior to 2008, Syrinx can export customers and sales invoices to .iif format text files, which can then be loaded via Quickbooks’ own import facility.]

Customer/Supplier Records

Syrinx and Quickbooks have different rules for uniquely identifying customers and suppliers.  In Syrinx, each account customer/supplier is identified by their account number, whereas Quickbooks requires each customer/supplier name to be unique, and the use of account numbers in Quickbooks is optional.

For the transfer program to work correctly, each account customer/supplier record in Syrinx must have a unique account number and a unique customer name.  Also, the name of any customer/supplier cannot exceed 41 characters in length.  These names and account numbers must then match one to one with the customer/supplier records in Quickbooks.  

For a new Syrinx install, this means that the customer list in Quickbooks must first be checked to ensure that all customers have unique account numbers.  To do this, first export the list to Excel:

Then in Excel, Select All (Ctrl-A) and Data / Sort by Account No:

You can then go down the spreadsheet and identify any missing or duplicated account numbers, and make the necessary adjustments in Quickbooks to ensure all customers have unique account numbers.

If it is not a new Syrinx install, then both systems must be rigorously cross checked before using the transfer program.

Cash Accounts

The Depots screen in Syrinx is used to record the cash account numbers that non-account invoices are to be posted to.  The transfer program identifies customer records in Quickbooks using the customer name, so it must be able to determine the customer name that corresponds to each cash account number.  To meet this requirement, an account customer record must be created in Syrinx for each of your cash account numbers, making sure that the customer name in Syrinx matches exactly with the Quickbooks name.  These account customer records in Syrinx should then be made non-current to ensure that no contracts/advice notes are entered against them; they will be used solely by the transfer program to do the name lookup from the account number.

Handing Customer/Supplier Name changes

The transfer program uses the customer name as specified in Syrinx to search for the corresponding customer record in Quickbooks.  It is essential that the names in Quickbooks match the names in Syrinx for both customers and suppliers.

When transferring customers to Quickbooks, if a customer is not found with the given name a new customer will be created; this will cause duplicate records in Quickbooks for the same customer if a name change is made in Syrinx without the corresponding change also being made in Quickbooks.  Similarly, when posting invoices, the transfer program will search for the customer using the customer name as defined in Syrinx; if it cannot find one then a ‘No such customer account’ error will be returned.  If it finds a customer with the given name but the account numbers don’t match, then an exception error will be raised.

To minimise the possibility of any operating issues such as these, it is strongly recommended that Syrinx is configured as follows.  On the Accounts tab in System Options, set the Accounting System to Quickbooks, and un-tick the option that would permit duplicate account numbers:

Running the Transfer program

To configure the transfer program, you simply need to specify the location of the Quickbooks data file:

On Windows 7 and above, the data files are generally located in “C:\Users\Public\Documents\Intuit\QuickBooks\Company Files” but in a multi-user environment they can be held in a network location.  This path must be entered exactly as listed in Quickbooks and is case sensitive.

Before running the transfer program, you must already have Quickbooks open with the appropriate company file open.  The transfer program communicates directly with the (running) Quickbooks application, so for optimum transfer speed, it is recommended that the Quickbooks program is minimised during the transfer.

The first time the transfer program is run, Quickbooks will ask for confirmation that the transfer program is indeed allowed to connect to your Quickbooks data file.  This is a Quickbooks security check, for the user to acknowledge the validity of the integrating application.

Select the third option, and press Continue, and confirm the next dialog windows.
