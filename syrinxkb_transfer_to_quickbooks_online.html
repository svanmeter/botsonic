Guide on the Syrinx Transfer Tool for Quickbooks Online


Summary

On initial release, the Quickbooks Online (QBO) transfer tool will have the same functionality as the existing Quickbook for Windows (QBW) transfer tool meaning that it will support the direct transfer of customers, suppliers, sales/purchase invoices and depreciation (journals) into QBO and will work with Syrinx business areas if enabled. It will also pull back customer balances, aged debt information and unpaid invoices into Syrinx.  The tool will also have the ability to run as a service if required. 

 

Prerequisites

  • The Syrinx version must be v4.40.5 or above.
  • A valid license code for the QBO Transfer Tool must exist in the Syrinx database.
  • Syrinx and QBO should both be configured as per the below recommendations prior to attempting to transfer any information between the two programs
  • QBO will only allow a user who is of type “Admin” to connect via the transfer tool and will only allow one admin user to connect, so the recommendation would be to create a specific user for use by the transfer tool and all installations of the tool should use the same QBO user to connect

Nominal Code Configuration

QBO does not have “nominal codes” like Syrinx but instead has entries in its chart of accounts which are identified by name.  A nominal account can also be a sub account of another nominal account and this must be reflected in Syrinx as the nominal account name must contain the main account name and the sub account name separated by a colon (i.e. mainaccount:subaccount).  The names of the nominal accounts in QBO must match the names of the nominal codes defined in Syrinx. Creating Service Items for Nominals in QBO

In QBO, transaction lines do not allow for the separate allocation to a nominal (chart of accounts) and are instead linked to a “service item” which itself is linked to a nominal account.  Because of this, it is required to create a “service item” for each nominal code used in Syrinx via the settings icon and selecting “Product and services”. In order to pick the correct service item, the item name must be the same as the Syrinx nominal code. The service item is then linked with the correct income/expense account by selecting the previously created chart of account entry for the nominal code.

 

 

Customer/Suppliers

QBO does not have account numbers for customers or suppliers (vendors) and its records are referenced by name. The suggestion by Intuit for recording account numbers is to prefix the customer/vendor name with its account number.  The referenced field used for this in QBO is the “Customer display name” and it is this that Syrinx will use when initially referencing customer/vendor records.

 

For customers new to Syrinx and QBO, the recommendation would be to manage the customers / suppliers in Syrinx, allowing it to create records in QBO as it will correctly create the formatted display names as per Intuit’s recommendations.

 

For existing QBO customers, customer and vendor records will not be automatically pulled back into Syrinx. Previously with QBW, the requirement was to set an account number on the QB record, and the transfer tool would pick up that record and create it in Syrinx. With QBO, a similar mechanism occurs but it relies on the fact that the record’s “Customer display name” has the format “AccountNumber – Customer or Vendor Name” (e.g. “A000 – ABC Plant and Tools”). It will delimit the full name by the first hyphen and assume the text to the left is the account number and the text to the right is the record name.

 

Once a customer or supplier has been posted from Syrinx to QBO through either creating a new record or an update, the QBO internal record id is captured and stored against the customer/supplier record in the Syrinx database.  From that point forward, updates will be done using the QBO internal id as a reference rather than looking up the name, allowing name updates to also take place. Hence it is preferred to always use Syrinx as the master customer/supplier record source.

QBO - Test Account

For testing purposes, a free developer QBO user account can be created by visiting https://developer.intuit.com/app/developer/qbo/docs/develop and using the signup page. There is no need to create an “app” and all you need is the sandbox company that will be created following successful signup. By default QBO will create a US based sandbox company, it is suggested that you create your own local company sandbox in the region that most closely matches your physical location. In order for the transfer tool to connect to QBO, you need to use a user for the chosen sandbox company who is of type Admin.
