Overview

The Syrinx Mobile App web service is a web site that must be made available to the internet but must also have access to your Syrinx database server on the internal network.  Prevention of unauthorised access to your internal files and data (e.g. by hackers) requires specialised hardware and hardware configuration – this must be in place before the Syrinx Mobile App web service is installed.  In a recommended configuration, the Mobile App web service requires: 

  • A separate server PC running IIS, onto which the Syrinx Mobile App web service will be installed 
  • A connection from this separate web server PC to the internet with a static external IP address 
  • A connection from this separate web server PC to your main server that runs the Syrinx database, with the ability to establish a SQL Server connection (typically via port 1433).
  • Suitable firewall(s) and DMZ configuration that protects your internal network from the internet 

System Requirements Checklist

The Following should be in place prior to the installation of the Mobile App Web Service on your network:

 
 

Requirement

Completed

1.

The designated web server must be in place and available on your network.

 

2.

The web server should have a fixed IP address on the internal network.

 

3.

The internet connection should have a fixed IP Address.

 

4.

The Syrinx system should be upgraded to the latest version.  If you are unsure of what version of Syrinx you are running, please contact our Support team on +44 (0)118 9569577 or email uksupport@pointofrental.com .

 

5.

Port forwarding rules should be in place on the router in order to forward external web traffic to the web server.  The web server will generally be listening on TCP ports 80 or 443 (if SSL is enabled) although this can be changed. If other services on the network listen on TCP ports 80 or 443 and port forwarding rules are already in place for those services which direct traffic to a different web server, an alternate port may be forwarded to tcp port 80 (or 443) on the web server.  Users would just need to be made aware of this port number as they will need to add it to the connection strings on their devices.  

 

*Note - Point of Rental will not supply or install SSL certificates.  It would be the responsibility of the customer to set up and configure SSL connections on the web server

 

6.

If the web server is isolated from the rest of the internal network within a DMZ, the appropriate exceptions will need to be added in order to allow the web server to communicate with the database server through the DMZ.  The database server will normally listen on ports UDP 1434 and TCP 1433 but the TCP port can vary depending on how the database engine was installed.  If you do not know what port the Syrinx database server is listening on, please contact our Support team on +44 (0)118 9569577 or email uksupport@pointofrental.com .

 
 

Failure to implement any of the above may result in delays to your installation or training.
