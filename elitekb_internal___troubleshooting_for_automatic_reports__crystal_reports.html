INTERNAL | Troubleshooting for Automatic Reports - Crystal Reports

 

In this article, we're going to walk you through troubleshooting custom and Crystal Reports within Automatic Reports and also show you what to look for in a SQL Server.


Topics in this article include
 

  • Troubleshooting hardware (SQL) 
  • Troubleshooting within Automatic Reports




 

Troubleshooting hardware (SQL)


Here are some things that you can check for within the SQL Server. Please reach out to CEX IT for information on how to perform these steps.
 

  • Downloaded Crystal Reports (version 2022) and install.
 
  • Verify that user por reportuser is setup in SQL.
 
  • Verify that the remote connection is enabled on the SQL server.  SQL management studio > right click on server> connection > allow remote connection.
 
  • Verify that SQL port 9767 is enabled.  SQL configuration manager > SQL server network configuration > protocols for sql > enable TCP port 9767.
 
  • Create odbc 32-bit connection on windows to connect Crystal Reports & SQL.  Point of Rental can only read the 32bit (not 64bit) odbc driver when opening reports in the default location.

 


 

 Troubleshooting within Automatic Reports

  • Ensure custom reports are saved to Point of Rental reports' default location on the SQL server. Here is the path: c:\por\reports or c:\por\reportscustom. Please note, the customer may be using another drive letter. 
 
 
  • Ensure the following naming convention is used when saving custom reports: client#.reportname.sql.rpt
 
  • Verify that the custom reports uploaded within Point of Rental. You can do this by testing the report.  
 

 
Here's how to get there
 

Expert: Program Menu > Configuration > Reports and Notification > Automatic Reports 
Elite: Configuration Gear > App Configuration > Reports and Notification > Automatic Reports 

 
  • To resave a custom report within Automatic Reports, click the Reload Params button. Then click Ok.
