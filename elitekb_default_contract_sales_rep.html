Default Contract Sales Rep

 

This is a Configuration setting that will determine how the ‘Contract Sales Rep’ is set, by default, when creating a new contract.  This is a company-wide default.

 

Security

To view the option in Configurations, set the Transaction Configuration area and the option: Default Contract Sales Rep to ‘Yes’.

 

Options

From the Configuration option, there is a dropdown with 4 options to choose from:

 
  • None - This option will not set the Contract Sales Rep by any means other than a user choosing to do so.
  • Territory - This option validates with the Zip/Postal code of the Customer on the contract.  If that Zip code is assigned to a Territory AND there is only one Sales Rep assigned to that Territory, this Sales Rep will be set as the Contract Sales Rep.
    • If multiple reps are assigned to the Territory, none will be set and any could be selected.
  • Jobsite - This option validates when a Jobsite is added.  If there is a Sales Rep assigned, that rep will be set as the Contract Sales Rep.
  • Customer - If the contract Customer has a Sales Rep assigned, that rep will be set as the Contract Sales Rep.
  • Plus...A rep can be updated that was auto-set.
 

Related Options

 

Contract Sales Rep Required - This option is set in the Legacy App Configurations - Store Parameters.  When true, this field is required.  This is a separate feature from the Default Contract Sales Rep.  If one is set automatically, the requirement is fulfilled.  If not, then it will show as required.

 

Add/Modify Security - There are two security options in the Transactions Area that work with this field:

 
  • Contract Sales Rep - Add New:  This controls the ability to add a Contract Sales Rep when set to None.
  • Contract Sales Rep - Modify: This controls the ability to change the Sales Rep when one is already set.
