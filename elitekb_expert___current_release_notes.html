Release Notes (informational only)


In the past, we have kept the Expert and the Elite release notes in separate camps to avoid confusion.  However, we think it’s time that the fence comes down between The Sandlot and The Beast.  After all, we’re all on the same baseball..er…software team. Because Elite is the next evolution of Expert, the latest and greatest technology moving toward the future, we thought it would be beneficial for our Expert customers to be in the know.  As Elite features release and mature you may find functionality that you just can’t live without.  While new features in Expert are limited due to its older technology, fear not, Expert will continue to be supported.  Below are the release notes that our Elite customers received for this release; there is also a link to the Elite Release notes on the home page of this site if you are interested in learning more about past Elite releases.

 

V2023.09

 

Sorting on Delivery/Pickup Dates Column in Day at a Glance:  You know that thing?  Where you’d sort on the delivery/pickup column?  And the navigation bar would leap to the right?  Yeah, we fixed that. 

 

Browser Availability Store Drop Down:  This store drop down was found to not conform to the store, district, region, company user permissions.  Now it does, and will match the store drop down found in the item search grids.

 

Automatic Report Selection Criteria:  The selection criteria for automatic reports has been extended for SQL users pastthe 255 characters constrained in a JET db.  As a reminder, selection criteria can now be extracted from any crystal report generated to the screen by selecting the appropriate export icon at the top left.  This makes it very easy to set up an automatic report that has just what you’re looking for. 

 

Call Log Notifications from POR One:  Users can set up system notifications to be sent when a call log is created for a specific call log type or customer.  Now, call logs created against either of these criteria in POR One will also send the notification(s) to the appropriate people.

 

Price Override Tracking Password Field:  On the tracking screen for system security, Elite users will now notice that the password drop down is grayed out.  This is due to the fact that this security no longer applies to Elite users.  Instead, the ad hoc pricing security should be used.  However, price overrides will still be tracked in the edits for the boxes that are selected.

 

Sales Rep in Dispatch:  The sales rep column was still labeled ‘salesman’ in our dispatch center.  It has been updated to ‘Sales Rep’.

 

POR One: These are updates for Elite users utilizing the POR One app.

 

Linking Serialized Items on Task: When completing a task on POR One, such as Driver Delivery or Customer Pick up, and the item has not been serialized in Elite the user has the ability to select a serialized item from POR One. Once they have done this, the specific item page can now be viewed by tapping on the Item Name. This allows you to see all of the item details to determine if it is the correct item to associate with the Contract. 

Mobile Permissions Improved Design: The Mobile Permissions that were recently added to Elite has been updated to be even easier to use and navigate. Clicking on the menu icon next to any of the usernames will show additional details such as Email, Relationship ID and Remote User ID. The last two don’t sound important but may be requested by Support when helping you troubleshoot. 

FIX Serialized Image Tagging: When adding a picture attachment for an unserialized item it was being associated to all of the serialized items erroneously. This has been fixed to only be associated to the items serial number that has been selected. 


V2023.10

Receive PO Security:  The receive PO security is intended to prevent users from receiving inventory into the system without proper clearance.  It was discovered that this did not prevent users who were modifying the purchase order from double clicking on the quantity and receiving the inventory.  We went ahead and caulked this hole. 

 

uLearn:  In the future, uLearn will become the single source of truth for program information and education.  As such, when users click on the NetHelp option, they will be notified of this and given the opportunity to either continue on to NetHelp or go over to uLearn.

 

Mobile Permissions UI Updates:  Updates were made to the mobile permissions app that launches from the browser.  Aesthetically the colors were refreshed and the column sizes adjusted.  There are now helpful tool tips that give information about the license when hovered over.  Selecting the ellipse next to the user will spin up a screen that has additional user information.  If there is capacity for more users to be added to a license for an organization, the column name will be in green.  If all licenses have been used for that feature, the column name will appear in gray.

 

Bulk Uploads to AWS Servers:  When users upload more than one file to their AWS server, they only receive one prompt for the files being uploaded rather than one for each file being uploaded.  A progress pop up will allow the user to see when the upload completes or if one errors out.

 

Closed Contract Mods and the Taxable Flag:  One of our users uncovered a scenario when modifying a closed contract with their contract modifications turned on. They marked the contract to be nontaxable, which created the appropriate credit.  However, any price modifications to the original contract after that point, did not take into account the credit that had marked the contract nontaxable and were creating new credits with tax.  Going forward, the program will now handle for these closed contracts that have been marked non-taxable.

 

Closed Contracts and External Tax:  Users discovered that they were able to change delivery information on a closed contract after the tax call had been sent.  The issue that this causes is that for destination based tax, the delivery address and whether or not the contract was delivered at all plays into the tax calculated.  We’ve removed this ability from closed contracts for our external tax users to prevent any ‘oopsies’.

 

POR One: These are updates for Elite users utilizing the POR One app.

More Details on Stops: The list of Stops on a Route now has more information shown along with an accordion for the delivery details. This allows a driver to know more about a Stop at a glance without having to go into the Stop details. The cards now show Customer Name, Status, Planned Arrival, Required Arrival Time, Address Accordion & Map Link for Navigation and the first few Line Items on the Contact.

Dates on Task Filters: The Tasks filter now has a more robust set of options for dates. This now has all of the date filter options available on other page filters in One such as Today, Last 7 days, Last 30 days and a Custom Search range. 

Failed Task Creation Timeout: If a Task, like Drivery Delivery for example, fails to load due to a lost connection to the server the user will be prompted to delete the task and try again. This allows you to quickly see there was an issue creating the Task and to not wait assuming that it is loading. 

About Page: An “About POR One'' page has been added to Settings. The page displays device and organization details and other information about the application. 


V2023.11

Column Color Update:  There has been a visual update in the base core forms.  The column color has been switched to a darker gray for a greater contrast on these screens.

 

Core Window Focus in AWS:  On cloud hosted Elite environments, users were confronted with a scenario in which core screens that were opened from the browser, would display ‘out of focus’ and would have to be clicked on in order to be interacted with.  This was corrected.

 

Inventory Search and Special Rates:  We discovered that users with hundreds of thousands of special rates could see a reduction in inventory search performance.  We tweaked this to improve speed.

 

On the Way Notifications:  The ability to edit the phrasing of the ‘On the Way’ notification has been added.  This is accessed through Configurations>System Configurations>Notification Messages.  If no custom text is added, it will use the default.  On the Way notifications are also now being sent when initiated from POR One.  Two for one special.

V2023.12
 


Multi-Select Attachments:  When adding attachments to a record through the ‘Set File’ option, users can now hold down the control key to select multiple records at one time.  They will be added as individual attachments under the record.

 

Closed Contract Mods and Marking Nontaxable Contracts Taxable:  Two sprints ago we did work around marking a closed contract nontaxable and having the subsequent closed mods also be nontaxable.  We have now done the work in reverse.  When a nontaxable closed contract is marked taxable, the subsequent closed mods will also be taxable.

 

Pronoun Update on Contract Update Message:  When a customer is changed from ‘Cash’ to ‘On Account’, the user is given the following message: “Customer Is now On Account. Do you want To update all of his current contracts to On Account?”  Now, obviously, this does not make much sense in relation to all customers, so we’ve updated ‘his’ to ‘their’.

 

Support Request Callback Number:  When an operator accesses the ‘Support Request’ option in the right hand help menu, the form will reference their operator record when populating the call back number.  It will use their cell number and if no cell number exists, their office number.  If neither of those exists, it will populate the store number as it has in the past. 

 

Day at a Glance Tabs: Users have expressed the opinion that it is difficult to see which tab they have selected in day at a glance as they all blend together.  We have made them more visible by bolding the text of the tab selected. 

 

Item Options:  Users now have the ability to assign options to their items.  Think of this a bit like doing a search on Amazon.  You search for ‘shirt’ and get all the shirts.  To narrow things down, you select for certain brands or colors, only including shirts that fit your criteria.  This enhanced search tool is available wherever the inventory selection screen exists.  For a detailed explanation of how to assign options and how this works, check out the ‘Item Options’ document in NetHelp or uLearn.

 

Credit Card Retrieval:  For Moneris and Cayan users, the credit card retrieval screen only displayed a set number of previous valid credit card tokens.  This screen has been enhanced and users can scroll through as many tokens as are available.  

 

Minimizing ID/Password Prompts:  For our Elite browser users, credentials are required to log into the browser.  As such, many have expressed a desire to not have to reenter ids and passwords in various places in the program as the interface already knows who they are.  We started the process of eliminating these prompts.  Account payments, file maintenance, and end of day are a few of the places we touched this sprint.  Is there a place you have to enter your credentials that really bugs you?  Let us know!  This will not apply to users on shared terminals-for obvious reasons.

 

Xero API Accounting Integration:  We’ve added a new horse to the API accounting integration stable.  Aside from a small difference in the export credential screen, it works basically like our other accounting integrations.  If you keep buying bay horses, can anyone even tell the herd is growing?  Lol.  Sorry.  Horse girl joke.  

 

Memory Lane Monday: This is a fun corner where we republish release notes from previous sprints you may have forgotten about.

uInnovate eSign Initials/Signature Update: People don’t read the fine print. *I* don’t read the fine print. “Do you agree to these terms and conditions?” “Absolutely, I do.” Of course, when later they want to question something they signed off on, it is helpful to point to specific places they were required to stop, read, and initial. We have enhanced our eSign/Countersign feature to allow users to collect initials/signatures in multiple places on a document. Within the Company Configurations of your eSign account, certain text tokens can be assigned to indicate whether a signature or initials are required. Insert these text tokens into various places (Legalese/terms/printout fields of items/etc) and the customer will be prompted to sign/initial at each of them when signing the contract through eSign or Countersign. Please refer to the help documentation to learn more.

uInnovate Special Order Part: We all know that when we have to go back to something, it falls through the cracks. In order to make the creation of special order item records even easier, the options to mark the item as a bulk item and/or a part item have been added to the screen that appears when the Special Order option is selected from the contract or purchase order.

POR One: These are updates for Elite users utilizing the POR One app.

Easier to tab Checkbox -  The size of the area for you to check off bulk item on a checklist task, such as Driver Delivery and Driver Pickup, has been enlarged to make it easier to check off quickly. You can still tap in to complete a full checklist of the cognition status, if needed by tapped on the name of the item undergoing a condition check. Otherwise, tapping on the left side of the card will complete the task and you can move on to the next. 

Delivery Accordion Hidden when not needed -  The delivery and pickup details accordion on a Contract will not be shown anymore if the contract is set to be picked up and dropped off by your customer. This was causing confusion and the Delivery and Pickup tabs will now only be shown when relevant.
