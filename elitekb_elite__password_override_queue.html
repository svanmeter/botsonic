Password Override Queue

 

This feature allows for an Operator that does not have permissions to proceed due to an Override prompt to request an approval from others in the company from a button on the override option.

 

 

Currently, this feature is only for overrides that are presented while writing or modifying a contract.  

 

By selecting ‘Override Request’ from the override prompt, you are adding a request to those that have the security enabled to view the queue. The security setting to allow access to the queue is in Browser security. The Remote Password queue is accessed by selecting the ‘shield’ icon in the header area by the Operator name.

 

From the ‘approver’s’ point of view, a notification badge will display as new requests come in.

 

 

When viewing the feature there are options to Approve, Deny or Refresh.

 

 

When the request is made from Browser contracts, there will also be a ‘reference link’ that allows you to get more information about the Customer or Contract related to the request.  This link will take the user to a Browser customer record or the contract (browser) in inquire mode.

 

Requests made from CORE app will not display a reference link.

 

Requests made will only send the request to those that have permission to approve it. Requests from anywhere that are ‘Denied’ will send a message to the operator.

 

If the requestor cancels out of the override, the request will be removed from the queue.
