Granting a Windows Authentication User Access to Syrinx

Before adding a new windows auth user to Syrinx, you should make sure that the user has been made a member of a security group that has been granted login to the SQL server. 

 

To add a new user, connect to the Syrinx SP Configuration program using a login that has been granted Admin rights on the Syrinx database.  At a minimum this account will require the db_securityadmin and db_accessadmin security roles in order to add users to Syrinx.

 

First, click on the “Users” tab and then select the “Logon Users” menu option:

 

 

Now click on the “Add User” button on the bottom left of the Logon Users screen:

 
 

On the resulting “Add New Logon User” screen, enter the user’s name in the “User Name” field.   This name will be used to indicate which user has performed specific actions in the system and will usually be entered in First Name + Surname format.  Then, enter the user’s windows username in the “Server Login Name” field.  This should be entered in domain\username format.  Next, link the user to their specific depot if necessary and select the “General Menu Access” options applicable to the user.    These “General Menu Access” options are built-in roles within Syrinx that control a user’s access to specific menus in the Syrinx programs as well as granting them privileges to perform certain actions.  The roles linked to a user’s account can be changed after the user has been created.

 

If Successful, the system will issue the message and the user will be granted access to the Syrinx system:

 

 

The user should then appear in the list of users on the “Logon Users” screen.  

 

Granting a User Admin Rights on Syrinx Database

If a user requires Admin access on the Syrinx database such as the ability to create new users and run patches against the database via the Syrinx Configuration program.  You can select the user from the list of users on the “Logon Users” screen and then press the “Show Detail” button to open the “Amend Logon User” screen.   You can then use the “Set Admin” and “Remove Admin” buttons to grant or revoke Admin rights for the user.

 

Possible Issues with the User Creation Process

1.  Creating user with non-admin account – If you attempt to create a Syrinx Logon using an account that does not have admin rights on the database, you will receive the following error upon pressing the “OK” button:

 

Assertion failure - 'SProc Error: User does not have permission to perform this action.  (ADO Error(s): 0:Command contained one or more errors.)'.
Location:
THC21_1.cmdOK_Click
lbDbRunStoredProc(spUsrCreate)

 

You should click “OK” on the error message and then “Cancel” on the “Add New Logon User” screen.  The system will have created an entry for this user in the “Logon Users” screen.  This entry should be deleted using the “Remove” button at the bottom of the “Logon Users” screen and the user creation process started again using a Syrinx logon that has admin rights on the database.

 

2.  Incorrect Windows Username Entered in “Server Login Name” Field – If you attempt to create a Syrinx Logon using a windows username that is incorrect i.e. Spelling error, username does not exist, group not granted SQL login, the system will issue the following error:

 

Assertion failure - 'SProc Error: Windows NT user or group 'higherconcept\joeblggs' not found. Check the name again.  (ADO Error(s): 15410:User or role 'joeblggs' does not exist in this database.,0:Command contained one or more errors.)'.
Location:
    THC21_1.cmdOK_Click
    lbDbRunStoredProc(spUsrCreate)

 

You should click “OK” on the error message and then “Cancel” on the “Add New Logon User” screen.  The system will have created an entry for this user in the “Logon Users” screen.   In this instance, you simply need to close and reopen the “Logon Users” screen, select the user’s account from the list and press the “Show Detail” button at the bottom of the screen.  This will allow you to correct the value held in the “Server Login Name” Field.  If you then close and reopen the record, you should notice that the user has been assigned the “Syrinx User” database role and will be able to access the system:

 
 

If the SQL Server Role area for the user remains blank, the user should be deleted and recreated.

 

3.  Non Admin User Attempts to SET or REMOVE Admin from User – If a user without admin rights attempts to Set or Remove Admin rights from a user, they will receive the following error:

 

Assertion failure - 'SProc Error: User does not have permission to perform this action.  (ADO Error(s): 0:Command contained one or more errors.)'.
Location:
THC21_1.cmdOK_Click
lbDbRunStoredProc(spUsrCreate)

 

The process should be repeated using an account with admin rights on the database.

 

Possible Issues when User Attempts to Connect to Syrinx

1.  Incorrect Database Server Name or Network/Firewall Issues Preventing Connection – When a user attempts to connect to Syrinx and receives one of the following errors verify that the SQL server name specified in the connection details is correct and that there are no network or firewall issues preventing connections from the user’s client to the SQL database server or the port on which the instance of SQL server is listening:

 

Unable to connect to server: [DBNETLIB][ConnectionOpen (Connect()).]Specified SQL server not found. (-2147467259)

Or

Unable to connect to server: [DBNETLIB][ConnectionOpen (Connect()).]SQL Server does not exist or access denied.  (-2147467259)

 

2. User Attempting to Connect to Syrinx with a Program File that is Currently in Use -  If a user opens Syrinx and receives the error message “This file is not in the correct format for a Microsoft Access Project”, this means that they are using a Syrinx program file that is currently in use by another user.  They will not be able to connect to Syrinx until the other user logs off or until a unique copy of the Syrinx program file is created for them and their Syrinx shortcut/rdp file is modified to point to their specific version of the file. 

 

3.  User Enters Username and Password on Connect Database Screen – When using Windows Authentication with Syrinx, it is not necessary to enter a username or password on the “Connect Database” screen  when logging in to Syrinx as the user’s windows credentials will be passed to the SQL server when attempting to establish a connection.  If the user enters a username and password, they will receive the error message:

 

Unable to connect to server: Login failed for user ‘domain\joebloggs’. (-2147217843)

 

The user simply needs to remove any values held in the Username and Password fields and attempt to connect again.

 

4.  All Syrinx Licenses Have Been Allocated to Other Clients – If a user attempts to connect to Syrinx and all licenses have been allocated to other client connections the user will either be presented with a screen allowing them to remove a license from an existing client (this is temporary and the system will stop issuing this screen after the licenses have been reallocated a set number of times) or they will receive the following message:

 

The PC ‘ExamplePC’ has not been granted access to the system.  Please contact your local system administrator.

 

In order to allow the client to access Syrinx, an admin user will have to access the “Registered PCs” menu found in Syrinx SP Configuration\System Administration and remove a license from a PC that no longer requires access to the system.  The client should attempt to reconnect at which point they will take the free license and be added to the Registered PCs list.

 

5.  User Attempts to Access Syrinx Without and Account in the Syrinx Database – If a user has not bee set up in Syrinx and they attempt to connect to the Syrinx, they will receive the following error:

 

Unable to connect to server: Cannot open database "SyrinxDM" requested by the login. The login failed. (-2147467259)

 

Create an Account for the user in the Syrinx Configuration program and have them attempt to connect again.
