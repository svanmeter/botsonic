INTERNAL | Force log off a user's Remote Desktop Session from the Server



This article will provide the means to show support or allow the customer to self-service logging off a user using an Administrative login on the server.


Topics in this article include:

 

  • Identifying your server (internal only)
  • Force log off a user's Remote Desktop Session
  • Method 1 to open Task Manager
  • Method 2 to open Task Manager




 

Identifying your server (internal only)

Verify on the Salesforce Software Info page if the customer has a POR-provided server, a user-supplied server or a POR Could server (Salesforce Account->Lowest Number Asset->Software Info).
 

POR CLOUD

 

Note: If the customer is using a POR Cloud Server, send the ticket to IT Support.
 

IMPORTANT: POR Managed Azure Windows Virtual Desktop with Site-To-Site VPN
 


 

Web Address to log in: https://rdweb.wvd.microsoft.com/arm/webclient/index.html 
 

You can also scroll down to the current hardware/operating system information and look at the domain and server name.
 

Domain/Server Name: PORCLOUD\669-WVD-0 (POR Cloud V1 will show the domain to have PORCLOUD and the Server Name will be in the form of CNUM-WVD-0. WVD means Microsoft Windows Virtual Desktop.)
 

POR Provided - On-Premises
 

Domain/Server Name: RENTAL\ENTERPRISE1 (domain is RENTAL and the computer name is Enterprise, Enterprise1, or Enterprise2)
 

Server Origin: Most likely a POR provided Server. Verify contracts required.
 

Model: PowerEdge T620, T630, T640, T110 II, T130, T140 (POR Provided server Models. PowerEdge means it is a Dell server.)
 

User Supplied - On-Premises and user-supplied hosted

If POR Provided - On-Premises or POR CLOUD do not match up, then you are most likely dealing with a user supplied server. They may or may not have an IT department that assists. The rules are the same as a POR provided server, an administrative login is needed to get onto the server to be able to log off users.
 

Important: Different versions of the operating system will present different methods to launch Task Manager. In the end, Start->Run TaskManager.exe is what needs to happen.



Force log off a user's Remote Desktop Session
 

1. Log in as Admin:
 

If the customer server was provided by POR, the customer should have access to an admin account.

If the server is user supplied, ask the customer to request help from their IT department.

If they do not have an IT department, have them log in as Admin and follow the steps below.

If they do not have an Admin log-in, they will need to find one and call back. 

 

2. Once logged into an admin account, open task manager.


3. After the Task Manager has opened, select the “Users” tab.
 


 

Note: If no tabs are available, click “More Details” on the bottom left of the window.
 


4. Under the user's tab, right-click the user you want to log off.

5. Select “Sign Off.”
 



6. Repeat this step for all users you wish to have off the server.
 

Note: Be mindful not to select the account you are logged in on.

 

 

Method 1 to open Task Manager


1. Right click the taskbar and select Task Manager.

 



 

Method 2 to open Task Manager


1. Open the start menu.

2. Search for Task Manager from the start menu.

3. Click Task Manager.
