4/05/2023 v2023.05

 

We are listening!  Items with this label are a result of your suggestions in uInnovate.  Please continue to share and support those ideas that help your business.

 

  We recommend reading these and understanding the impact prior to updating.  These features may require adjustments to security or configuration and may have implications for your operations or current workflows. 

 

  

 

 

 Sales Rep Goal/Widget:  A new widget has been added that allows users to configure and compare sales goals for sales reps.  It can be filtered on a single sales rep or store view.  The goals themselves are set on the sales rep record.  They are compared against transactions for a time period both for actual revenue and probably revenue. Within the sales rep record itself, the user can view how over/under a sales rep comes by either a dollar amount or a percentage.  The ability to set/update these goals is controlled by a security setting under the Sales Rep security area labeled ‘Update Sales Rep goals’. For additional information, please refer to the uLearn or NetHelp document. 

 

Browser Dashboard Sales Rep Filter:  This filter has been streamlined to either display an individual sales rep or the option for ‘All Sales Reps’. Simple. Straightforward. Satisfying.

    

Stock/Product API and Parts:  We now support part numbers and the parts item flag in these endpoints, allowing users to search/filter on part numbers within POR One.

   
Core Design Update:  The core screens within Elite have gotten a She’s All That style makeover.  We endeavored to make the font/ colors more conducive to what exists in the Elite browser.

    

Translation Speed for Shared Users:  As a follow on to an earlier adjustment, the speed core windows load for shared users using a language other than English, has been improved. Heigh ho, Silver!

    

Sorting on Labor Updates:  In the labor updates screen the only sortable field is the Task #, and so we’ve updated the UI to reflect that.

    

Vertex and Canada:  The vertex integration was enhanced to handle Canadian zip codes. Before, only US zip codes would successfully return a tax amount. Call it an early chocolate bunny for your Easter basket.

    

POR One:  The app, the myth, the legend is here. Reach out to your friendly neighborhood sales rep to set up a demo. 

    

Deposit Slip Timeout:  When writing a deposit slip, the form will save and exit after 30 minutes of inactivity.  This is to prevent the batch numbers from being off after end of day if this form is accidentally left open.

   
       
       

POR One
        
These are updates for Elite users utilizing the POR One app.
 

Depot (Store) shown on Routes Details: Drivers can now see the starting and ending Depot locations on the Routes details pages. The search filter has been enhanced with Start Depot and End Depot fields so that a user can see Routes for a specific depot location.


Optimized Searches & Filters: Each search page has been optimized to reduce response times and return the most relevant searches based on the input.


Offline Notifications: One notifications that do not rely on network connectivity are now shown to a user when they are offline. 


Search now available on Task lists: This can be used to make condition checking of Contract or large Inventory Counts with many line items faster and easier. This supports barcode scanning, if the items have one. Available for any tasks checklists created. 


Recently Viewed toggle on Search Pages: POR One search pages now have a toggle between recently viewed and search results. This is for all pages (Contracts, Customers, Inventory, Work Orders, and Routes). 


Saved Search Filters: When you apply search filters they are saved when applied and will be set for the next session. This is for all pages (Contracts, Customers, Inventory, Work Orders, and Routes). To start fresh, there is an option to “Clear” any search filters that have been set. 


Duplicate Task Warning: To reduce user error, a warning is now displayed if a user attempts to create duplicate tasks.


 

Memory Lane Monday

This is a fun corner where we republish release notes from previous sprints you may have forgotten about.

 

Item Maintenance:  In the past, users could only set one maintenance schedule per description, per item.  But, what if you need an Oil Change at 3 months or 250 hours?  Now, users can set up an Oil Change multiple times with unique intervals (cannot duplicate an interval - 1 time and 3 times); when the machine hits 3 months or 250 hours the maintenance will be triggered.  When the Oil Change is completed and the maintenance is cleared, all Oil Change maintenance schedules will be reset.  

 

Special Rate Update: Data management tools - Special Rate Update - End Date can now be set to blank en masse. Blank is used to denote a special rate that does not expire.
