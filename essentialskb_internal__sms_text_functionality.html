INTERNAL | SMS Text Functionality



The SMS text functionality allows you to send notifications to employees and customers via text. This feature also allows you to text customers' contracts and ask for signatures instead of waiting for an email. This feature is currently feature wrapped and a Product Support Tier 3 Agent will have to enable this feature. 

 

Topics in this article include

 

  • Setting Up Employees to Text as Default for Notifications

  • Notifications Available for SMS Text Messages

  • Customer Notification Setup

  • Sending SMS Messages via a Transaction

 


 

Setting Up Employees to Text as Default for Notifications

For each employee, you can choose whether to use a primary phone or cell phone as the notification method. This will work for any notifications the employee has set up.
 






Notifications Available for SMS Text Messages

You can set up different notifications that are set within your Configuration Gear > User Settings.
 

 
 

Customer Notification Setup

You can also define at the customer level how each customer receives their notifications that apply to the Reminder Email Timer in Configuration.

 

 
 

Sending SMS Messages via a Transaction

At the transaction level, you have the ability to choose SMS texting instead of email.
