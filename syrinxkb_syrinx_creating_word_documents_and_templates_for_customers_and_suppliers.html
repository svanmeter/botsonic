Overview

If you regularly create standard letters (or other MS Word documents) for your customers / suppliers, then Syrinx can be configured to let you create these directly from the contacts tab of the Customer / Supplier screens.  Syrinx can also be set up to create central folders to hold documents relating to specific customers and suppliers, so that when creating a new letter for a customer, the system can save the letter automatically in the central folder for that customer.  This makes it very quick and simple to file, and then later find previous correspondence for a particular customer or supplier.

System Configuration – MS Word Templates

The integration with MS Word is turned off by default.  To turn it on, use the ‘Document Mngmt’ tab of the System Options screen:

You must specify a central folder for the word templates, as shown above.  For consistency, we suggest that you create a folder under the SpDocs share on your server to hold the templates.  However, this folder can be anywhere as long as it is accessible by all Syrinx users that will be using this feature.

You can also override this setting for your PC only, using the ‘User Options’ screen.  This may be useful for testing some new templates prior to making them generally available.

System Configuration – Customer Document Folders

The ability to create customer / supplier document folders is turned off by default.  To turn this feature on, use the ‘Document Root Folders’ screen on the ‘Document Management’ menu in Syrinx Configuration:

This setting specifies the root folder for the sub-folders that will contain documents relating to Customers and Suppliers.  The sub-folders beneath this root folder are created automatically when you press the 'Documents' button on a particular customer or supplier.  You must ensure that all users that require use of the Documents button have read/write permissions on this root folder.

Having specified the root folder, those users that are to be allowed to use the feature must also be granted the ‘Admin.View Customer Documents’ privilege.  Any roles that have access to the Amend Customers screen will already have been granted this privilege.  

Customer Documents.

Once the ‘Central Folder for Customer Documents’ option has been set, the Query / Amend Customer and Suppliers screens show a Documents button:

 

When this button is pressed, the system generates a folder name based on the internal customer id and the customer name, and then looks in the central customers folder for a folder that has the same id prefix.  If the folder does not exist, then it is created automatically.  The system then opens the folder in a standard ‘My Computer’ window:

 

You can then open any of the files, or drag new files into the folder, etc.

Supplier documents.

The same Documents button is available on the Query/Amend Suppliers screens.

MS Word Integration - Templates.

The MS Word Integration feature is designed to let you quickly create standard letters and other correspondence from the Contacts tab of the customer/supplier screens.  You first select a contact, then pick the template to use, and the system creates a new word document using that template, and passing in the name and address of the contact.

Three sample templates are provided, and are installed into the ‘C:\Program Files\Higher Concept\SyrinxSP\extras’ folder:

Letter.dot – default letter

Fax.dot – default fax template

CollectionNote.dot – This sample may be adapted if you want to create a blank document for a given customer, such as a blank collection note. 

Template Document Properties - Definition

Information, such as the customer name and address is passed into the word template using custom document properties, that are set up using the File/Properties menu option within MS Word:

Note: In MS Word 2007 and later, you need to go via the ‘Office’ button in the top left, then Prepare / Properties.  The click on the Document Properties drop-down and chose ‘Advanced Properties’.

The set of custom properties that Syrinx passes in (if present in the template) are:

uCustomerName

Customer Name

uCustomerMainAddress

Invoice Address

uCustomerAddress

Contact’s (site) address

uCustomerPhoneNumber

Main Phone Number

uCustomerFaxNumber

Main Fax number

uContactFullName

Contacts full name

uContactTitle

Contact Title

uContactFirstName

Contact first name

uContactSurname

Contact surname

uContactPhoneNumber

Contact phone number

uSysUserName

Current Syrinx Username

uSysCompanyName

Current Syrinx Company Name

 

In addition to these properties, you can also add properties for information that is to be prompted for each time a letter is created.  For example, you may have a standard letter to confirm a forthcoming site visit, and you may want to enter the date of the visit on each letter.  To achieve this enter the name of the property and a display value as shown on the right, then press the ‘Add’ button.  Note the name must be prefixed with ‘uPrompt’.

Once you have added the custom property, you can use it within the text of your template by using the Insert / Field… option on the menu, and then chosing DocProperty in the list on the left hand side, and your property name on the right hand side:
