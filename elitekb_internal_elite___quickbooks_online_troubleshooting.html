Errors & Troubleshooting


Error Logging:

You can check the following places to see if any errors have been logged.
 

  • QB Log in the Programs folder - C:\POR\Program\logs\QuickBooksAPI.txt
     

  • Sync Service Log in the Programs folder- C:\POR\Program\logs\AccountingSyncService\AccountingSyncServce.txt
     

  • In the Database (SQL) - Select * from ErrorLog Order By DateTimeStamp desc
     

  • In the Database (SQL) - Select ErrorMsg, * from AccountingAPIQueueGL Where DateCancelled is Null and NoExport = 0 Order By Id desc
     

  • In the Database (SQL) - Select ErrorMsg, * from AccountingAPIQueueTR Where DateCancelled is Null and NoExport = 0 Order By Id desc
     

  • In the Database (SQL) - Select ErrorMsg, * from AccountingAPIQueuePO Where DateCancelled is Null and NoExport = 0 Order By Id desc

 
 

How the System Matches Fields

 

  • The system automatically sends customers and vendors to QuickBooks Online as they are used on contracts or purchase orders. It has a process of verifying if they have already been sent, and if they have, it matches them to the existing one. If it does not exist yet, it creates the customer or vendor in QuickBooks when it posts the contract or purchase order.
     

  • Customer - Looks for Customer by NAME field.
     

  • Accounts - Looks for existing accounts by Account Number.
     

  • Locations - Looks for existing locations by Name.
     

  • Vendor - Looks for Vendor by Vendor Name.
     

  • Item - Looks for Item by ItemDetail > Item Memo



Common Errors


Error: System.IO.FileLoadException: Could not load file or assembly 'Microsoft.Extensions.Configuration, Version=2.2.0.0, Culture=neutral, PublicKeyToken=adb9793829ddae60' or one of its dependencies. The located assembly's manifest definition does not match the assembly reference. (Exception from HRESULT: 0x80131040)
File name: 'Microsoft.Extensions.Configuration, Version=2.2.0.0, Culture=neutral, PublicKeyToken=adb9793829ddae60'

Solution: Clean and rebuild the project, which will remove and replace old DLLs with new ones. Copy DLLs to the Program folder.


Error: Validation failing on a property, but property exists.

Solution: Make sure property [property]Specified = true
