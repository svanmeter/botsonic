We are listening!  Items with this label are a result of your suggestions in uInnovate.  Please continue to share and support those ideas that help your business.

 

  We recommend reading these and understanding the impact prior to updating.  These features may require adjustments to security or configuration and may have implications for your operations or current workflows. 

 
 


Dispatch Columns for Cubic Size and Dispatcher:  Two new columns have been added to the dispatch center routing screen.  ‘Cubic’ calculates the total size of the contract based on the corresponding field on the options tab of the items.  ‘Dispatcher’ displays who was responsible for creating the route so it is easy to know where to go to get any questions answered.

 

Information in Title Bar:  You may notice that we have added the information for machine, terminal, and LAN IP to the title bar so that this information is more easily accessible for those needing to find this information.

 

New Smartequip Keys:  The behavior around creating header keys for new parts has changed slightly to accommodate the part numbers that exceed the database’s fleng constraints.  How the item lookup is set in the web config file will determine whether new part headers are created using the item number for the key or if the part number is used and truncated on the right-hand side of the part number to conform to fleng constraints.  Please refer to the Smartequip Installation document for more details and examples.

 

Update Email Password from Operator Information:  We originally created a stop that if a user had more than one email setup record, they did not have the option to update it from the left-hand menu under operator information.  However, this was causing the option to be unavailable for users who may have a Store Specific email record and a general Store email record that uses the same email address.  Now, the user will be able to access the update password option if they have multiple email records that all have the same email address.  If they have multiple email records with different email addresses, they will not have the option from the left menu.

 

Cloud Server Update Messaging:  Update messaging has been updated for users on cloud servers.  The message at login notifying them of an available update has been removed as they are no longer responsible for those updates.  The prompt that there has been an update and there are new release notes to read remains.

 

Minimizing ID/Password Prompts:  We are continuing our work to minimize the number of times you have to enter your ID/password in nonshared user browser sessions.  How did you like the first round of our updates around this?


Browser Grid Exports:  In the past, exporting from a browser search grid would only export the first page of results.  For example: If the page was showing 1-100 of 1000 results, the export would only export the first 100 results.  Now, exporting will export all 1000 results at once.

 

 
          

POR One Release Notes
        
These are updates for Elite users utilizing the POR One app.

 

Show Stock Quantity on Inventory Items - The current stock quantity is now shown on each Stock item on the Item Detail screen. The value shown is for what the quantity is at that moment in the Elite database. 


Checklist enhancement for Serialized Items - When completing a Check Out or Delivery task, when a header item is part of the list then the serialized item must be selected before the rest of the checklist can be completed. 


Disabled Users - Now, when a user is disabled in Elite, it will remove all permissions that the user had for POR One. The number of available licenses will increase by 1 for each user that is disabled that had the permission and it can then be applied to another active user. 


AU/UK Date Format - The dates shown throughout the app will now be displayed in DD/MM/YYYY when selecting “English (UK)” language on the settings page. The default for “English (US)” will display the date as MM/DD/YYYY.

 

 

Memory Lane Monday

This is a fun corner where we republish release notes from previous sprints you may have forgotten about.

u Item Exchange History Report: We have added a report so that users can identify when and why exchanges are happening and address those areas to decrease the number of machines exchanged.
 

u Credits and Related Contracts: Credit contracts created from a specific contract will now be listed in the Related Contracts list available from the Reports menu option from within a contract.
