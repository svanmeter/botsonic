Launchpad - Operator Set up

 

The permissions that allow for an Operator to be Added or Modified (updated) are in the Operator area of security.

 

There are ‘View’ and ‘Edit’ options for the fields in the Operator form.

  • View controls if the user can see the field.
  • Edit controls if the user can modify the field once the modify option has been selected.

Modifying the email address - When the system has a global organization, the admin updating emails must also have ‘Edit Permissions’ in Global.  They must be logged in with Single Sign On (SSO) credentials.

 

It is recommended to leave the following set as Required:

  • Password
  • Password Level

A Store relationship is a system requirement that cannot be changed.  All Operators must have a relationship to at least one Store.

  • If an Operator with valid credentials logs in without a Store relationship set, the Store set in the User Profile will be auto-set as a related Store and set as the Default.
  • To Add, Remove or change the Default Store, the user must have permissions to Modify an Operator.
    • Select the ‘Store’ related tab
    • Select Modify
    • Select ‘+Add Existing’
    • With the cursor in the displayed search option, you can just select <enter> to display all Stores.  Or, you can search by Store name.
    • Selecting a Store from the grid will add it as a related Store.
    • From the ‘Action’ option menu, you can update each row to Remove or set as Default.  If there is already a Default set, it will deselect the current one and select the new one.

 

 

Shared and Time Clock Only Users

When the Shared User or Time Clock Only option is selected, only the Name and ID are required. 

 

Local Credentials

These can be used to both access the application and validate with user permissions while using the application.  They can be short to make entering them a little easier.

 

Single Sign-On Credentials (SSO)

 

This is where the user’s email is entered. This is also the Login ID for Launchpad for global organizations.  This is required to have access to many features integrated with POROne.
