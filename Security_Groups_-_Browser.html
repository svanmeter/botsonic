Security Groups - Browser

 

This feature is designed to allow for both ‘Security’ and ‘Security and Employee’ Groups to be created, removed and related Operators managed.

 

Definitions

 

Security Group - This is purely used to assign ‘Yes’ and ‘No’ permissions in the Security feature.

 

Security and Employee Group - All Employee Groups are also Security Groups.  Plus, they have the double duty of also being used for things like Tasks and Time Clock.

 

Security

Permission on the browser side to manage Security Groups is located in the Configurations Area: Security Groups.  When permission is set to Yes, the operator will see and can select the Security Groups option in Configurations.  This permission also provides all the underlying permissions to add, remove Groups and assign Operators.

 

Changes to Security Group assignment can also be made from an Operator record.  A user with ‘Update Operator’ or ‘Insert Operator’ will have the ability to update Security Groups related to an Operator.

 

Security Groups

Accessing this feature from Configurations will launch a new dashboard tab.  A list of current Security Groups will be displayed.  One of the default columns here is the ‘Employee Group’ column.  A ‘checked box’ in this column indicates a Security Group that is also marked as an Employee Group.

 

Action options:

  • Remove - a Security Group can be removed from this list.  This will remove any relationships and controls that are associated with that Group.
  • Add - This option will open a new dashboard tab where you can enter the Name of a new Security Group.
 

Adding a New Group

A Name is required.  You can also check the box labeled: ‘Also make Security Group and Employee Group’.  Selecting this will also display features for pay types if you use this.

Operators

You can also manage Security Groups from an Operator.  From the Operator record, select the ‘Security Groups’ related option.  From here you can add/remove Security Groups to a user.
