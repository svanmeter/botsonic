Agreed Rates Will no Longer Require Elevated Privileges if Greater than Max Hire Discount for Equipment - Users will no longer require the "Prv.Front Desk.Override Max Hire Discount" privilege to confirm contracts if a hire item has been added to that contract where the customer's agreed rate for that item is greater than the "Max Hire Discount" value set on that hire item's category.  Users will still be prevented from confirming contracts if they then amend the rate on those hire items to an effective discount that is greater than the customer's agreed rate/discount.

Support For Sage Line 50 v28 - The transfer tool has been updated to include support for Sage Line 50 v28.

Download Timesheets with Invoices via SyrinxCWS - A new system option, "Inc. TimeSheets with Batch Invoice Print?", has been added to the "Syrinx WS" tab of Syrinx SP Configuration\System Options\Syrinx WS.  When enabled, the PDF file containing batch invoices output via the Invoices\Invoice Batch menu in SyrinxCWS will include any scanned or mobile timesheets linked to the invoices in that batch as long as those documents have been made available to the SyrinxCWS site.

Allow Direct Entry of Net Price on Advice Notes - A new system option, "Allow Direct Entry of Net Price", has been added to the "Advice Note Entry" tab of Syrinx SP Configuration\System Options\Sales.   With this option enabled, users will be able to directly enter a "Net Price" value on an advice note line and the system will work out the appropriate discount to apply to the line based on that item's "List Price".

Option to Disable Prompt to Clear Delivery Details - A new system option, "Prompt To Clear Requires Delivery?", has been added to the "Deliveries" tab of Syrinx SP Configuration\System Options\Contract Entry that controls whether or not the system will prompt the user to clear a hire contract's delivery details when changing the contract's type from "Firm Booking" to "Quote".  This option is enabled by default as this was previously the system's default behaviour.  If the option is disabled, the user will no longer be issued the prompt, "The contract has been marked for delivery.  Do you want to clear the 'Requires Delivery?' flag?".

Date and Time Tokens for Scheduled Report Filenames - The Amend Report Schedule screen will now show the "File Output" tab for all export types. For non "export to file" types (e.g. email a PDF copy of the report), the output file path field will not be shown.  For emailed scheduled report entries, if a file name value has been set, this file name will be used in preference to the default report name. This allows the user to use the date and time tokens in the filename of the file that is emailed:

Set Bulk Qty Unavailable from Hire Stock Item Details Screen - Users who have the system privilege "Front Desk.Set Bulk Hire Unavailability" are now able to set an "Unavailable" qty value against bulk hire stock items from the "Hire Stock Item Details" screen. Previously, this field could only be edited from the "Amend Hire Stock Items" screen meaning the user also required Amend Hire Stock menu access and privilege to edit this value.

Purging Staff Member Records will now Remove Associated Staff Audit Log Entries - The "Purge Data?" option on the "Remove Staff Member" tab of Syrinx SP Configuration\Administration\Bulk Update will now remove all staff audit log entries associated with that staff member as these may also hold personally identifiable information.

Restrict Regions During Invoice Batch Creation - A new user privilege, "Prv.Invoicing.Invoice Any Region", has been added which will automatically be linked during the upgrade process to any existing role that already has the existing privilege "Prv.Invoicing.Invoice Any Depot".  When this privilege is revoked from a user's role and the user still has "Prv.Invoicing.Invoice Any Depot" privilege which also allows invoicing by region, the region selection dropdown on the "Create New Invoice Batch" screen will be restricted to only those regions that are linked to the depots to which the user has access as defined on the "Depot Access" tab of the "Amend Logon User" screen.

*Note: If the user is not a depot restricted user (i.e. no depots are defined on the "Depot Access" tab of the "Amend Logon User" screen), revoking this privilege will have no effect on the batch creation process.

Raise Credits Against Part Exchange Advice Notes - The system will now allow users to raise credits against the part exchange component of a part exchange advice note so that this type of advice note can be fully or partially credited.

Option To Add Additional Currencies to the Syrinx Database - Users are now able to add additional currencies to the Syrinx database via the Syrinx SP Configuration\"Accounts/Invoicing"\Currencies menu.  Previously, a patch was required to add additional foreign currencies to the system. 

Post Date added to the Enter Cross Hire Invoice and Enter Transport Invoice Screens - A "Post Date" field has been added to the "Enter Cross Hire Invoice" and "Enter Transport Invoice Screens".  This field will only visible and mandatory if the "Enter Posting Dates?" option has been enabled against the accounting system linked to Syrinx in the Syrinx Configuration program.  The "Post Date" field was previously only available on the "Enter Purchase Invoice" screen when entering purchase invoices for standard consumable purchase orders.

New EquipmentHireRates GET Endpoint on SyrinxWebApi - A new EquipmentHireRates GET endpoint has been added to the API.  This endpoint returns all of the default hire rates in the database.  If “CategoryCode” and “Id” parameters are not passed a value, only hire rates linked to "current" hire categories will be returned.  Non-current category hire rates can be returned if the specific category code and/or hire rate id are supplied.

Hire Rate ID Link added to GET Assets Endpoint on SyrinxWebApi - A "pricingLevelId" value has been added to the output of the GET Assets endpoint.  This ID will be the internal database ID of the hire rate linked to a hire stock item.
