eCommerce | Categories Missing from the Website

INTERNAL

 

Entire categories may be excluded from the website.  This is done during Step 2 of the Update Website process when run in Configuration Mode, but many customers do not know about it.

 

If a customer is having issues with an entire category missing from their website, this is the first thing to check!  Start the Update Website process, then select Configuration Mode.  Once finished, select Step 2.  You'll see a window with included categories down the left side and excluded categories down the right.  If the category you're looking for is on the right side, check it, and click the Apply Selections button.  It should then be visible on the left side of the window.  Next, click the Commit Changes button.  The next step is to select Step 6 of the Update process to Upload to Website.  Once this finishes, the category should appear on the site.

Note:  If the category did not appear on the left side of the window when you clicked the Apply Selections button, this is a minor bug - you'll need to run the update one more time for the change to take effect.  You may run the update in Auto Mode this time; you do not have to run it in Config Mode. 

Next, be sure that the category in question is assigned to the same division as the other categories showing on the same page of the website.

If the category still does not appear on the website, check to be sure that there is at least one item in the category that has:

  • a quantity or subrent quantity
  • a rental rate or a sale price, whichever is applicable
  • a valid header key if applicable (if there is a header key entered in the item record, be sure the header key actually exists)
  • and is not marked "hide on website" or "inactive"

If after checking all of this, the category is still missing from the site, create a ticket for eCommerce Support.
