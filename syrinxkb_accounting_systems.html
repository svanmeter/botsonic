Set or Define an Accounting System in Syrinx


The Accounting System that you are linking Syrinx with is set on the "Accounts" tab of Syrinx Configuration\System Options\Accounts:

If your accounts package is not in the list, it can be added using the "Accounting Systems" screen which is accessed via the "Accounts/invoicing" tab in the Syrinx Configuration program:

The following information can be entered/updated:

 

General

Name

The name of the accounting system.

Allow 3 Address Lines…?

Does the accounting system have 3 address lines (before the town)?  Un-ticking this will prevent users from entering a 3rd address line on a customer’s invoice address in Syrinx.

Has Direct Transfer..?

Is there a direct transfer program for this accounting system?  If set, then the ‘Direct Transfer’ tab is made visible on the System Options screen to allow configuration of the direct transfer program.

Maximum Customer Name Length

The maximum allowed length of customer names in the accounting system.  If set, Syrinx will prevent entry of customer names longer than this length.

 

Account Numbers

Maximum Length

The maximum allowed length of customer/supplier account numbers in the accounting system.  If set, Syrinx will prevent entry of account numbers longer than this length.

Force Uppercase?

Should account numbers on customers and suppliers be converted to uppercase?

Unique Across Customers & Suppliers?

Should the system prevent from adding a customer whose account number is already in use by a supplier, or vice versa?

 

Purchase Invoices

Enter Posting Dates?

Should the user be able to enter a separate posting date when entering purchase invoices?  This should only be set if the transfer program makes use of this information.

Maximum Invoice No. Length

The maximum allowed length of purchase invoice numbers in the accounting system.  If set, Syrinx will prevent entry of invoice numbers longer than this length.
