SmartEquip Procedures

 

 

Point of Rental Tickets Flow

Ticket 1 – Onboarding Configuration for SmartEquip connectivity

                Ticket to include:

  • SmartEquip project manager name and email address (so that the SE PM can be copied on all ticket communications)
  • Provide unique Customer Identifier to the Fleet Client and copy the SE PM.
    • This field is needed to configure the EntFapplication.
  • Point-of-Rental to deploy the code necessary to configure the SmartEquip integration to “ON” for the Fleet Client.
  • Point-of-Rental to provide port and IP information to SmartEquip so that the connection through the VPN can be established.
  • Equipment Number based on ITEMKEY OR ITEMNUMBER – POR to configure based on this number.
  • Point-of-Rental copy the SmartEquipFleetExport.rpt to the Fleet Client production environment 

Ticket 2 – Create Generic test users in the Non-Production environment  

  • Provide SmartEquip with test Userid(s) to this environment after the test users have been created.
  • This ticket will be discussed in advance with the Fleet client and may become optional after time.

SmartEquip Test user setup - Purpose:

Ensure that test users are setup so that SmartEquip QA resources can use to login to

SmartEquip e-FleetPro, successfully pass user authentication, and execute all required use-cases from the SmartEquip EntFapplication.  Provide at least one test user for each user group or role in Point-of-Rental that will be using the SmartEquip EntFapplication. This will vary for each Fleet Client.

Ticket 3 – Enable Equipment Extract process

                Ticket to include:

  • SmartEquip FTP site information:  This information will be provided to the Fleet Client so that it can be included in this ticket.
    • FTP URL:  files.smartequip.net
    • Username:
    • Password: 
  • The Fleet Client will determine how often the extract process should run.
  • The extract process will be scheduled within the Point-of-Rental environment for the Fleet Client.

Ticket 4 – Create/Refresh Point-of-Rental Non-Production environment for the Fleet Client (optional)

                 Ticket to include:

  • Optional ticket needed if the Fleet Client requests the additional environment.
  • Provide Confirmation that the steps in Ticket 1 are complete.
  • Create the Fleet Client Non-Production environment and refresh this environment with the Fleet Client production data.
  • Provide SmartEquip with connectivity parameters to this environment.
