INTERNAL | Credit Card Processor Setup & Terminal Information



This article is going to show you how to set up or update your Cayan (GPI or TSYS) Credit Card Processor and Terminal information.
 

Important!

  • This process MUST be performed after an End of Day has been completed.
  • This process will require you to get a TeamViewer to the local computer, not the server.

 
  

Topics in this article include
 

  • Obtaining Terminal IP information
  • Adding a retrieved VeriFone terminal static IP to a new Cayan/GPI Verifone Terminal
  • Create VeriFone on-screen Terminal-for manually keyed CC transactions (if you do not have physical terminals)
  • Configuring your Point of Rental Software for GPI/Cayan
  • TSYS VARS
  • Troubleshooting
  • Training Video (5/26/2023)





Obtaining Terminal IP information

You must have a TV connected to the LOCAL computer, not the server.


IPv4

 

If the customer is going from Point to Cayan - use the same IP address that is assigned to the Point terminal.

If this is an Initial Cayan Setup:
 

  1. Pick a number under 100 and change the last section of the IP address. For example, 192.168.1.XXX
  2. Ping the new IP address and verify that it is unreachable. We want to verify the IP address is not being used by another device.
  3. If the IP address is reachable, repeat steps 1 and 2.
 
SubNet - This will always be 255.255.255.000
 

Gateway -  You must have a TV connected to the LOCAL computer in order to follow the below steps. 

  1. Open Command Prompt on the Local computer.

  2. Type IPCONFIG

  3. Write down the Default Gate Way. 

    1. If the second to last segment of the CC Terminal IPv4 is different from the Default Gateway you are most likely not on the local computer.

DNS
  1. 8.8.8.8
  2. 8.8.4.4

 

Adding a VeriFone terminal static IP to a new Cayan/GPI Verifone Terminal

You must have a TV connected to the LOCAL computer, not the server.
 

Configure the M400 terminals for POR Enterprise
 

  1. Press the '1,5,9' at the same time on the CAYAN terminal.
  2. “Com Control”
  3.  “LAN”
  4. “Configuration”
  5. Enter the code '200331'.
  6. Select 'IPV4'.
  7. Select 'DHCP' Set to “No”.
  8. Select the individual fields to correct/enter the static IP address settings.
  • The Static IP address, Netmask, and Gateway can all be found by opening the command prompt and typing "ipconfig"
  • DNS1 for the M400 is 008.008.008.008 and DNS2 008.008.004.004
  1. Select Back ONE Time and select Save and Apply. 
  2. Confirm the test passes without issues.
  3. Select the red key on the CAYAN terminal to return to the main screen.

 

Configure the MX915 terminals for POR Enterprise
 

1. Use the command prompt and ping to find an unused static LAN; make sure it is not in DHCP range.

2. Press the ‘0’ key on the physical terminal and, when prompted to log in, use ‘9416557’ for the password. Press the ‘Enter’ or ‘Green’ key.

3. Navigate to Network > Configuration.

4. Click ‘Mode’ and change to ‘StaticIP’ and click OK.

5. Configure the static IP address, netmask, gateway, DNS1, and DNS2 addresses. Each field needs to be selected by touching it with your finger or the stylus.

6. DNS1 and DNS2 need to be set to ‘8.8.8.8’ and ‘4.2.2.2’ respectively.

7. Click the Apply button in the upper right when the information has been entered.

8. Click the ‘Test’ option and run the test. Make sure everything passes before moving on.

9. If the internal test works, select ‘Done' and then select the ‘Red’ button to get back to the Genius screen. Resolve any network issues before moving to the next phase of the setup.



    Create VeriFone on-screen Terminal-for manually keyed CC transactions

    These stops are needed if the customer does not have physical terminals.
     

    1. Browse to Verifone Terminals to create the CAYAN on-screen prompt.  

      • Elite: Configuration Gear > Core Options > Security > VeriFone Terminals

      • Expert: Program Menu > Security > VeriFone Terminals

     

    1. Enter the following information

      • Description: CAYAN-STR[X]-ONSCREEN where [X] is the store number.

      • IP Address: 1.1.1.1. all on-screen/soft terminals must have this address.

      • Integration: Cayan US or CA. Please specify the country if there is an option to do so.

      • Site ID: [same Site ID of the store in question].

      • Merchant Key: [same Merchant Key as the store in question].

     

    If a workstation is assigned this “virtual terminal” or “soft terminal” device in their Point of Rental printer selection, then when they click Manual Entry on a CC transaction they’ll get a secure CC transaction screen pop-up for them to key in the CC transaction.


     

    Configuring Point of Rental Software for GPI/Cayan
     

    Setting up your new CAYAN system
     

    1. Run a credit card inquiry for all stores. Before continuing, there must be a $0 credit card balance in both Point of Rental and CAYAN. If there is a balance, the store(s) must perform an End of Day before they can be set live on CAYAN.

      • From Elite: Left Menu > End-of-Day > Credit Card Inquire

      • From Expert: Program Menu > End of Day Processing > Credit Card Inquire

        • Click on Submit to make sure there is a zero-dollar balance.
           

    2. Update the store’s parameters

      • From Elite: Configurations > Core Options > App Configurations > System Configurations > Parameters > Drawer

      • From Expert: Program Menu > Configurations > System Configurations > Parameters > Drawer

        • The Merchantware Name goes into the ‘Merchant #’ field.

        • The word CAYAN goes into the Processor field.

        • Check the box marked ‘Integrate Credit Card’ if you have not already done so.
           

    • Click on the ‘Apply’ button.
     
    • Close the Point of Rental software completely.



    Add the VeriFone terminal(s) to your Point of Rental Software
     

    1. To set up terminals 

      • Elite: Configuration Gear > Core Options > Security > VeriFone Terminals

      • Expert: Program Menu > Security > VeriFone Terminals
         

    2. Click the Add button

      • The description should always be CAYAN-[last 3-digits of device serial #]

      • Store - The store number that the device will be assigned to.

      • IP Address - static LAN IP address you assigned the device. Verify the IP is not already taken by another device. 

      • Integration - CAYAN

      • Site ID - Part of the CAYAN Merchantware Creds CAYAN emails us and the customer

      • Merchant Key - Part of the CAYAN Merchantware Creds CAYAN emails us and the customer


     

    TSYS VARS 


    If Cayan informed the customer that they will need to get a new VAR from Elavon:
     

      1.  Inform the customer they will need to request a TSYS SIARRA VARS with a TSYS front-end and an Elavon backend (24/h turnaround).

     

      1. Once the customer has received the new VARS, they must submit it to Cayan.

     

      1. After Cayan has uploaded the document, the integration is clear to continue.

     

    Confirm that Agent and Chain are 6 0’s and the bin number will be specific to the customer.
     




    Troubleshooting

    M400

    The customer states that Cayan informed them they have the wrong VAR OR if you find the terminal can process swipes but cannot complete a transaction with the chip or tap function.

    • The Customer will need to contact Elavon and request a TSYS Sierra VARS sheet from Elavon that has a TSYS front end and an Elavon Back end.

    • Once Elavon has processed the new VARS sheet. Submit the new information to Cayan.

    • After Cayan receives the new VARS sheet and inputs it into their system both the Chip and Tap functionality should be available.
       

    Unable to see Terminal on the Network
     

    • Test Connection by using ‘PING’ in the Window Command Prompt (CMD)

    • If you cannot see the terminal have the customer, check the cable.

    • If you have determined the network was caused by a hardware issue. Create a case for CEX Tier 1.

     

    Unable to collect payment from terminals or other issues

    If a customer sees this error, an existing connection was forcibly closed by the remote host. Open a web browser and try to navigate to transport.merchantware.net. It is possible the customer's network administrator has blocked this site. If so, they will need to reach out to their IT administrator or group for a resolution. 

    If the customer cannot come to a resolution after speaking to their IT team, the issue may be on the Cayan side. Have the customer troubleshoot with Cayan.
