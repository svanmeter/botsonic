Transfer tool Support for Sage 50 2023 v29 - Support for Sage 50 2023 v29 has been added to the transfer tool.

Download PDF Copy of Inspection Certificate from Workshop App - Workshop App users will now have the ability to download a PDF copy of the Inspection Certificate for mobile inspection jobs or service reminder jobs that require inspection test results.  Pressing the new download button on the signing page of the mobile job should download a PDF copy of the inspection certificate to the device and open it in the user's default PDF viewer:

The PDF could then be printed to a linked Bluetooth printer.   These PDF inspection certificates will be saved in the user's "Downloads" folder on the device.

In order for this new functionality to work correctly copies of the crystal inspection certificate document, THW02R_DM.rpt, and any alternate versions (e.g. THW02R_DMLOLER.rpt) should be copied to the Syrinx Mobile service's reports directory/virtual directory where the current crystal documents for the mobile service are held (e.g. C:\inetpub\wwwroot\SyrinxAndroidService\Reports).

Set Default Behaviour for Updating Item Meter Reading if Excess Usage Details are Updated via Amend Hire Contract - Users are now able to set the system behaviour as to whether or not an item's current meter reading is updated if that item's excess usage information is amended on an open hire contract via the "Amend Hire Contract" screen with a new system option, "Allow Usage Updates Latest Reading?", on the "Meter Readings" tab of Syrinx SP Configuration\System Options\Hire Fleet:

The default is "Yes" as this was the previous behaviour. If set to "Prompt", the user will be given the choice when amending the contract:

Record Time Element of Date Confirmed for Timesheets - A timesheet's confirmation date will now include the time when the timesheet was confirmed.  Previously only the date was recorded.

Specify Number of Required Photos for Sales Items - Users are now able to enter the number of required delivery/collection photos on sales categories:

Previously this option was on/off for sales categories but now up to a maximum of 10 photos can be required for sales items when they are delivered/collected via the Driver App.

Dedicated Field to Search by Order Number When Entering Purchase Receipts - An "Order" search field has been added to the "Enter Purchase Receipt - Pick Order" screen in order to make it more obvious to the user that they can filter the list by entering a specific purchase order number:

The existing "Find" field can still be used to filter the list by order number, supplier name, supplier account number, the customer's account number with the supplier as well as the name of the user who raised the order.

Option to Make Stock Item User Properties Private - Users are now able to flag individual stock item user property items as "Private" with a new "Priv?" check box on the Stock Item User Properties screen accessed via Syrinx SP Configuration\General\Stock Item Properties:

Users who do not have the new user privilege "Prv.User Properties.View Private Properties" will not be able to view those property items on the hire/sales stock item records:

This has also been extended to the custom crystal reports versions of the Inspection Certificate, Hire Breakdown & Service Job documents.  The inspection certificate should natively handle the change but the Hire Breakdown and Service Job documents would need to be updated to take this change into account and a specific "ItemUserProp" value would have to be suppressed if its corresponding  "StockPropDef" (i.e. its label) value is returned as NULL.  This logic also only applies to stock user properties 1 through 10 on those two documents.

Log Changes to Offhire Date and Time When Those Values are Updated on an Unconfirmed Offhire - Changes to the offhire date and time values on an unconfirmed offhire will now be recorded in the hire contract log.  Previously, the system would only log a change to an offhire's date if it was updated after the offhire was confirmed.

Supply Alternative Values for Contract Service Route Start and End Points - Users are now able to set alternative start/end points for contract service notes when performing route optimisation.  At present, a contract service note route will use the depot's address as the start/end points for the trip and this will still be the default for any new contract service notes that are created.  Users will have the option to set these values to a "driver" staff member's address as long as lat/long values are set against the address on the staff member's record.  If the staff member's address does not contain lat/long values, they will not be displayed in this list.  This is for cases where the staff member may be starting and ending their journey from home instead of the depot to which a contract service note is linked.  There is also a "User Entered" option which will allow the lat/long values to be edited manually:

Note: there is no validation for "User Entered" lat/long values and route optimisation may fail if invalid values are entered into these fields.

In the map displayed using the "RouteSavvy" route optimisation integration, the start and destination descriptions will vary depending on the value set in the "From" dropdown.  When Depot is selected, the start and end points are listed as "Depot name (Depot)" as before. For a driver staff member, it will display the "Line1" value of the staff member's address.  When set as "User Entered" it will simply display "Start" & "End" against these two points on the route:

System Will Now Prompt to Update Site Linked to Invoice for Advice Note - If the user changes the site linked to an unconfirmed advice note and the user has already printed/generated the invoice for that advice note, the system will now prompt the user to update the site linked to the unconfirmed invoice:

Import Supplier Discount Percentage with Sales Items - Users are now able to set or update supplier discount percentages when importing or updating sales items via the "Sales Items" & "Sales Items (Update)" options in Syrinx SP Configuration\Administration\Import Data.  Valid values for the new "Supp Disc" field on those imports are between 1 and 99.   Supplying a blank value for this field on the "Sales Items (Update)" import will not clear an existing supplier discount entry.  These must be cleared manually via the main Syrinx client if the supplier no longer offers a discount on specific items.

Custmer Insurance Email Field Expanded to Allow up to 255 Characters - The "Cust Ins. Email" field on the "Hire Insurance" tab of a customer record has been expanded to allow up to 255 characters.

Option to Syrinx Mobile SMTP Settings in the Syrinx Database -

There is now an option to store the SMTP emailing details for the Syrinx Mobile Service in the database instead of recording them in the smtp.config file on the SyrinxAndroidService site.  These values can be set on a new “Android SMTP” tab in Syrinx SP Configuration\System Options\General:

The system will still continue to use the smtp.config file if these new values are undefined in order to provide backward compatibility following the upgrade to 4.41.

 

Additional Fields Added to POST Customers Endpoint - Users can now set/update the following values on a customer record via the POST Customers endpoint in SyrinxWebApi:

"regNumber"

"cisRegNumber"

"webSite"

"salesperson"

Category Group Added to Various Endpoints in SyrinxWebApi - A new "CategoryGroup" parameter has been added to the Assets, EquipmentHireRates and StockLevels GET endpoints in SyrinxWebApi so that users can extract data from these endpoints from specific category groups.  A "CategoryGroup" field has also been added to the return payload for the Assets and EquipmentHireRates GET endpoints.

Booking Type Indicator Added to GET HireContracts Endpoint - A "bookingType" field has been added to the output of the GET HireContracts endpoint which will return the booking status of an unconfirmed contract and can either be Firm Booking, Provisional Booking, Quote or Prospect Quote.
