POR One | Check Out and Check In



 

This documentation provides an overview for the integration with Elite and POR One for item Check Out and Check In.  



 

Definitions

 

Check Out - performed at the point of handing off the possession of items from the rental store to the consumer.

Types

  • Customer Pickup - this is when a consumer comes to the rental location and takes possession of the rental items.
  • Driver Delivery - this is when the rental store delivers the items to the consumer’s residence or jobsite/venue.
 

Check In - performed at the point of the rental store taking back possession of items from the consumer.

Types

  • Customer Drop Off - this is when the consumer returns the items to the rental location.
  • Driver Pickup - this is when a driver retrieves the items from the consumer’s residence or jobsite/venue.
 

Configurations in Elite

 

Configuration settings in Elite that will drive certain behavior in One:

 
  • Configurations>App Configurations>System Configurations>Parameters>Contracts tab
    • Use Delivery eSign - selecting this will turn on the eSign option on the Contract page in One when the contract is set for Delivery.  The eSign can be performed on the driver’s device or sent by SMS to a consumer.
  • File Maintenance>Item File>Options tab
    • Checking the ‘suppress fulfillment’ option will configure that item to NOT be included in the Check Out process.  
    • Sales items are only included in the Check Out tasks.
    • This can be done in bulk from Database Management as well.
 

Updates made in Elite based on actions taken in One

 
  • Images - Images taken in One are available in Elite in several places.  Part of the Check Out/In task process is taking images of the items.  These images are available from Elite at the item and contract levels.  This includes an image of the signature.
  • Bulk quantity - a user can update the quantity of bulk items during the Check Out/In process.  The Elite contract will update to these quantities.
  • Select Serialized - If a Header record is a line item, the user will have the option to select the serialized item in One.  The Elite contract will update to the selected serialized item.
  • Call Logs - Completed Check Out/In tasks will generate a Call Log in the Elite contract.  The Call Log will detail the values set during the Check Out/In tasks.  Things like quantity, fuel level, meter and notes.  Check In Call Logs will be set to ‘Force acknowledgement’.
  • Multi-Leg Transfers - When the last Logistics ticket in a Multi-Leg transfer is completed in One, the Transfer contract will be closed.
