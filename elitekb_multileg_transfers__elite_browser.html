Multi-Leg Transfers - Elite Browser

 

A multi-Leg Transfer is for companies with greater than two locations that also use the Dispatch Center module.

 
  • This feature is designed to create Logistics tickets related to a Transfer that plan the route of the items on the Transfer through multiple waypoints (store locations).  
  • ALL items on the Transfer must complete the full route.  Items that may be dropped at an interim location would need their own Transfer and Logistics tickets.
  • This process only works with a One Way Transfer.
  • Logistics tickets are created, added, and updated on the Logistics tab in the Transfer.  If there is no Logistics tab showing, that means this system does not have the Global license for Dispatch Center turned on.  This must be turned on, a Global Feature Refresh triggered, then restart the application.
 

Configurations
 

  • There is a configuration option available in the ‘Dispatch Center’ area to ‘auto-create Logistics ticket’.  Selecting this configuration will create a Logistics ticket for every new One Way Transfer. And, the Delivery option will be selected as well.
  • Another configuration this is designed to work with is Transfer Delay.  This is where a value (in hours) is set for the ‘typical’ length of time it takes to move a product from one location to another.  These are user-defined.  Within this configuration, you can also set the Transfers between two locations to ‘No Transfer’. When this is selected, the ‘To’ location will not be available as a destination location.
 

Creating and updating a route with Logistics tickets
 

Logistics tickets are managed from the Logistics tab.  If you are using the ‘Auto-Create’ configuration the first leg of the route will be created when the Transfer is created.  Or, you can select ‘Create a new Logistics Ticket’.  This first leg of the route will be between the origin location (contract store) to the destination location (contract customer).  

  • Start with adding the items to be transferred onto the contract.
  • Adding a leg - When a leg is set, there will be Add and Remove options available.  The Add option will always set a new leg after the one selected.  The leg will now have no destination location.  And, the new leg will have no origin location.  Setting one will update the other.
    • Only available locations can be selected.  If the location is already on the route, it will display but be inactive.
    • Dates and times will update automatically based on existing dates and times as well as the Transfer Delay values. 
    • Conflicts will be noted when they fall outside store hours or when an arrival time and departure time conflict (it is set to leave before it gets there).
  • Transfer Period - The total time to move the items will be determined from the start date of the Transfer to the final arrival date/time of the last leg.  This will affect the availability of the item(s) during the transfer process.
  • The primary origin location can only be changed by changing the contract store.  The origin departure date and time must be updated from the contact header (Start Date). The final destination location can only be changed by changing the customer.
 

Managing Logistics Tickets

 

Updates to Logistics Tickets for Multi-Leg Transfers should be updated and managed from the Transfer.  Once set, and the transportation process begins - the process is managed and executed from the Dispatch Center and the One mobile app.  The Transfer must be ‘Sent Out’ from the Dispatch Center by the origin location.

 


The dispatcher can right-click on the first Logistics ticket and select ‘Send Out’.  Two options will be displayed.  They can be done in any order, but the origin location must send out both the Transfer and their Logistics tickets.

 

Integration with One - Rental items ONLY
 

Any Logistics ticket can be searched and selected from One.  From the contract page, there will be an option to ‘Close Logistics Ticket’. The Transfer must be ‘Sent Out’ in order for the Task option to be available in One.  The Task can be completed in either Reservation or Open status.  This will take the user through an inspection process of the rental items on the Logistics ticket.  Once completed, the Logistics ticket will be closed.  A Call Log is added to the Logistics Ticket as well as image links to the ‘Mobile’ tab in browser contracts.
 

If it is the final Logistics ticket related to the Transfer, the Transfer will also be closed.  Upon close of the Transfer, the item(s) on the transfer will have their Current location updated to the final origin location.
