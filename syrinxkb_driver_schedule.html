Syrinx® - Driver Schedule

 

 

 

 

Overview: This guide is designed to help you with using the driver scheduler.

 

 

Some of the key features of the Driver Scheduler include:

  • The ability to create multiple trips per vehicle without having to create brand new lines.
  • Full-screen capability with the ability to resize and zoom in on the main scheduling area.
  • Drag and drop functionality to enable users to allocate jobs/trips to vehicles quickly and easily.
  • Resizable jobs allow easy modifications of start/end times.
  • Extended system options and user privileges to allow total control of how the app looks and behaves.
  • Sorting and filtering functionality on the jobs list has been greatly enhanced to give the user the total control of what appears.
  • New week and month view which allow the users to see jobs in a wider scope.
  • Streamlined and more efficient
  • Purchase Orders can now be marked for collection and the new Driver Scheduler will pick them up.
  • The ability to view other depots schedule without having to change depot.
  • The ability to set a date range for the unallocated jobs list – no longer stuck to just viewing one day.
  • Users can now search for jobs in a time period and can instantly see where they have been scheduled.

 

 

From the ribbon at the top, click on Depot and then click on view within in the driver schedule section.

 

 

 

This will pop up a log in screen. Enter your normal Syrinx User Name and Password.

 

 

This will then launch your driver scheduler.

 

The driver schedule will display all jobs that are linked to transportation including:

 

  • Hire Delivery
  • Hire Collection
  • Hire Exchange
  • Site Movements
  • Advice Note Delivery
  • Purchase Order Collection
  • Customer Service Job Collection
  • Customer Service Job Delivery
  • Contract Servicing
  • Miscellaneous Jobs

 

If the job you are looking for isn’t displayed, then you just need to make sure the requires delivery/collection box is ticked on the relevant screen.

 

The top section of the scheduler is transportation vehicles. To add vehicles in they need to be created as hire items. On the Misc tab there is tick box that says include in driver schedule. This needs to be ticked.

 

 

The bottom section is unallocated jobs. To see more information, you can click on the unallocated job in the bottom. This will open the properties page to right hand side.

 

 

If you need to change anything on the contract, then you can click Open in Syrinx. This will open the job in the main system for you to make any changes.

 

To allocate jobs to drivers/vehicles, simply drag and drop the jobs to the relevant vehicle in the top section.

 

You can overlap jobs if delivery is the same time. You can also extend the jobs. For example, if a job is going to take 4 hours, you can extend it in the scheduler so you can see what capacity you have that day.

 

 

Once you have dragged jobs up, you will notice a black line. This is called a trip. You can click on the trip to see more information and assign a driver (drivers can be preset up with default vehicles). You can set the load and unload jobs (if enabled). You can see total weight of the delivery and safe working weights if these have been set against the vehicle.

 

 

 

To set the default vehicle against a member of staff, you will need to go into Staff > Staff Members > Search for the person and on the front screen choose the vehicle.

 

You can move jobs to different days. When you click on the job in the bottom section, the job details will be in the properties page. If you click on the arrow, you can change the date.

 

You can create ‘Misc Jobs.’ These can be customer, supplier or other visits. It may be that you want to send the driver to a customer to collect something or the vehicle may be off the road for maintenance.

 

 

The schedule view currently provides three different views to the user: Day, Week and Month. Day view is the only view where you can edit information relating to the jobs and trips – when in week and month view, you cannot drag or resize any jobs and trip information is not shown in these views either. Changing the view is accessed via the navigation bar.

 

Week view shows jobs slightly differently and does not show the trips. The number of days that are displayed in Week view can be changed via a new system option which allows the user to specify if they want to exclude or include weekends or to just include Saturday.

 

Month view shows jobs in a different manner and again does not show trips. Because Month view takes up a large amount of space, each day will only show a max of one job, with an arrow the user can press to expand to show all jobs for that day.

 

You can now search for jobs in the time period they are viewing on the schedule and jobs list by using the search button next to the Add section at the top:

 

 

The search screen shows all the jobs within the time period you are scheduling for and also the time period that is being used for the job list (which can be a set date range now). In this particular example, it is showing scheduled jobs for the 3rd August and unallocated jobs between 1st June – 3rd August:

 

Highlighting a job and pressing OK will instantly highlight that job on the schedule or the jobs list. The list offers fully integrated filtering and search functionality. To filter on a particular column, simply press the filter button and in the example opposite I am filtering on the Doc Type and you can see I can choose individual Doc Types to filter on or I can specify rules on which rows to show:

 

 

Or if a more general search is required, users can use the “Search Term” box at the top to type in what they want to search for. One point of reference is that if the search term contains a space then users should surround the search terms in double quotations. So if you wanted to find all jobs for the customer John Abbott  Ltd, you would have to type “John Abbott Ltd”. If you omitted the double quotations, the search would return all jobs that contained the words John or Abbott or Ltd:

 

 

 

Additional Information

 

  • You can also add a new trip by dragging a ‘new trip’ down. When you add a new trip this will create a new vehicle check that will need to be completed on the app. This could be used if the driver is changing mid shift (vehicle checks need to be created and enabled in the configuration).
  • The Bring Forward button will bring forward any unallocated jobs to the present day. There is a number of days that this will look back. This is set in the configuration.
  • The Map button in the unallocated section will show on a map where the jobs are. You will be limited to the number of points that is displayed. Google Maps allows around 23 different waypoints.
  • All the columns can be resized, filtered and moved around.
  • You can allow users to view the driver scheduler but not amend it.
