Syrinx 365 Minimum Hardware Requirements

Client PC

 

Operating System 

Supported: Windows 10; Windows 11 

Memory

Minimum 8GB (no upper limit)

Disk Space

Minimum 5 GB free disk space

Monitor

Any Size with minimum 1024 x 768 resolution supported

Network

Minimum 100Mbs (Internal Ethernet), Minimum 10Mbs (Broadband)

Microsoft .net Framework

Microsoft .NET Framework 4.8 must be installed before installation.  The latest available version of the Microsoft .Net Framework should also installed.

Microsoft Office

The Syrinx client makes use of Microsoft Access and Office 365 Access Runtime 64 bit will be installed if Office 365 is not currently installed on the PC.  If the full version of Office 365 is installed, the client will make use of the Access component of that installation.  Existing Office installations can be 32 or 64 bit.

 

Main/Server PC

 

Operating System 

Minimum: Windows 10 (1-5 client PCs max)

Supported: Windows 11 (1-5 client PCs max); Server 2016; Server 2019, Server 2022 (Windows Server CALs are not supplied by Point of Rental)

Database Engine

Minimum: SQL 2014

Supported: 2014, 2016, 2017, 2019, 2022

**SQL 2019 Express is included in the Syrinx installation media.

Memory

Minimum 16GB (no upper limit)

Disk Space

Minimum 5GB free
**Ensure enough storage is available for when the database grows and for when the mobile apps a being used and mobile images start being stored on the server.

Monitor

Any Size with minimum 1024 x 768 resolution supported

Network

Minimum 100Mbs (Internal Ethernet), Minimum 10Mbs (Broadband)

Backup

The Syrinx database should be backed up daily to external media such as an external Hard Drive, or onto cloud platform such as one drive/google drive, in order to ensure business continuity in the event of hardware failure, theft or natural disaster.

UPS

(Un-interruptible Power Supply) A UPS is strongly recommended to protect your database as power failures can lead to data corruption and possible loss of data. UPS devices allow the server to shutdown properly and not suddenly.

Microsoft .net

Microsoft .NET Framework 4.8 must be installed before installation.  The latest available version of the Microsoft .Net Framework should also be installed.

Microsoft Office

The Syrinx client makes use of Microsoft Access and Office 365 Access Runtime 64 bit will be installed if Office 365 is not currently installed on the PC.  If the full version of Office 365 is installed, the client will make use of the Access component of that installation.  Existing Office installations can be 32 or 64 bit.

 

Server and network setup 

It is advisable for your Server, network and any client workstations to be set up by an IT professional (Point of Rental will not be able to do this) in good time before the installation is due to take place, the installation will be cancelled if this isn’t in place by the day of the install. 

 

Mobile apps

The Syrinx mobile Apps (Driver, workshop etc) need to be able to communicate with the server while off-site, therefore a broadband connection with a router that can port forward external requests from the web to an internal web server will be required.  The mobile apps are designed to work with internet enabled android/iOS devices which must have a stable 4G/5G connection while off-site.  Separate hardware requirements are available for the Syrinx mobile products.

 

Terminal Services

Syrinx software can run in a terminal services or Citrix environment, but will require additional post installation tasks to function properly. Where you intend to deploy Syrinx in such an environment, please discuss with our team, so that we can supply necessary documentation and/or guidance prior to installation.

 

This documentation contains recommended settings and processes that allow Syrinx to run in a standard terminal services or Citrix environment. However, where there are additional configurations, management policies, or third party software in use that impacts Syrinx, these are not supported.

 

Network

Please ensure that all computers have been properly networked and can communicate effectively with the main/server computer prior to the planned installation date. You should also verify that all clients can access shared information on the main/server computer as Syrinx makes use of shared folders for document and log file storage as well as software distribution. Failure to verify that these aspects of your network are functioning properly may result in severe delays during the installation process.
