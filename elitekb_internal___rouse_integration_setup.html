Rouse Integration Setup 

Overview: This will go through how to set up Rouse and provide basic troubleshooting

Task Runner

Rouse runs off the Point of Rental Task runner. To get there go to:

Expert: Configuration > System Configuration > Task runner

Elite: Configuration Gear > Core Options > App Configuration > System Configuration > Task runner

  • Create a New Task
  • Place the following information in there 
    • Name: Rouse FTP Exporter
    • Description: This task exports the Rouse Data
    • File to run: <DataDrive>:\Program Files (x86)\Rouse\PointofRental\POR.exe
    • Start in Path: <DataDrive>:\POR\Users\
    • Recurrence: Days
    • Interval: 1
    • Priority: Low
    • Start Time: 4:00 AM

 

Check the .config Path

  • Go to the <DataDrive>:\Program Files (x86)\Rouse\PointofRental\POR.exe.config
  • Verify that the source is correct
    • Verify the Source value to <DataDrive>:\por\PORRouse.mdb 
  • Verify the Username and Password values to those provided by Rouse
  • Verify the Target - customer code provided by Rouse
    • If not, replace PointOfRental with the customer code provided by Rouse
  • Verify the Host to “ftp.rouseanalytics.com”
  • Save and close

 

Verify Information in Vendor File

Expert: Program Menu > File Maintenance > Vendor File

Elite: Configuration Gear > Core Options > File Maintenance > Vendor File

Customer switched from JET to SQL

PORRouse.mdb will remain in JET even if the customer is running on SQL. If the customer is running on SQL please contact John Yielding at john.yeilding@pointofrental.com and tell him that this customer needs to have Rouse configured for SQL.

 

KNOWN ISSUES:

Check event viewer on the application logs. Most of the time there is a typo on the config file. Event viewer will tell you which line this is on.

You can also run POR.exe anytime to make sure that Rouse is receiving the database. You can always email the guys mentioned above and asked them if they received the data from the specific company.

Any issues regarding bad data or missing data. Please let John Yielding know.
