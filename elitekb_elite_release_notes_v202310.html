7/19/2023 v2023.10

 




 

We are listening!  Items with this label are a result of your suggestions in uInnovate.  Please continue to share and support those ideas that help your business.

 

  We recommend reading these and understanding the impact prior to updating.  These features may require adjustments to security or configuration and may have implications for your operations or current workflows. 


  

 

Receive PO Security: The receive PO security is intended to prevent users from receiving inventory into the system without proper clearance.  It was discovered that this did not prevent users who were modifying the purchase order from double-clicking on the quantity and receiving the inventory.  We went ahead and caulked this hole. 

 

uLearn:  In the future, uLearn will become the single source of truth for program information and education.  As such, when users click on the NetHelp option, they will be notified of this and given the opportunity to either continue on to NetHelp or go over to uLearn.

 

Mobile Permissions UI Updates: Updates were made to the mobile permissions app that launches from the browser.  Aesthetically the colors were refreshed and the column sizes were adjusted. There are now helpful tooltips that give information about the license when hovered over.  Selecting the ellipse next to the user will spin up a screen that has additional user information.  If there is capacity for more users to be added to a license for an organization, the column name will be green.  If all licenses have been used for that feature, the column name will appear in gray.

 

Bulk Uploads to AWS Servers:  When users upload more than one file to their AWS server, they only receive one prompt for the files being uploaded rather than one for each file being uploaded.  A progress popup will allow the user to see when the upload completes or if one errors out.

 

Closed Contract Mods and the Taxable Flag:  One of our users uncovered a scenario when modifying a closed contract with their contract modifications turned on. They marked the contract to be nontaxable, which created the appropriate credit.  However, any price modifications to the original contract after that point did not take into account the credit that had marked the contract nontaxable and were creating new credits with tax.  Going forward, the program will now handle these closed contracts that have been marked non-taxable.

 

Closed Contracts and External Tax:  Users discovered that they were able to change delivery information on a closed contract after the tax call had been sent.  The issue that this causes is that for destination-based tax, the delivery address and whether or not the contract was delivered play into the tax calculated.  We’ve removed this ability from closed contracts for our external tax users to prevent any ‘oopsies’.

 

 

            

POR One Release Notes
        
These are updates for Elite users utilizing the POR One app.

 

More Details on Stops: The list of Stops on a Route now has more information shown along with an accordion for the delivery details. This allows a driver to know more about a Stop at a glance without having to go into the Stop details. The cards now show Customer Name, Status, Planned Arrival, Required Arrival Time, Address Accordion & Map Link for Navigation, and the first few Line Items on the Contact.
 

Dates on Task Filters: The Tasks filter now has a more robust set of options for dates. This now has all of the date filter options available on other page filters in One such as Today, Last 7 days, Last 30 days, and a Custom Search range. 
 

Failed Task Creation Timeout: If a Task, like Driver Delivery for example, fails to load due to a lost connection to the server the user will be prompted to delete the task and try again. This allows you to quickly see there was an issue creating the Task and to not wait assuming that it is loading. 
 

About Page: An “About POR One'' page has been added to Settings. The page displays device and organization details and other information about the application.

 

 

Memory Lane Monday

This is a fun corner where we republish release notes from previous sprints you may have forgotten about.


u Related Header: This field has been added to the options tab of an item file. The item number of a sales header can be added to a rental header and vice versa. This is used to auto-select a Rental Header when transferring to Rental. This helps automate the process and reduce errors. This feature will also allow the newly created Rental Item to maintain the same KEY and Item NUM as the sale record. This will allow the unit number to be maintained and not have to be changed after being transferred to rental. 


uAuto-Set Item Key: If you would rather just use the System generated number as your unique Key number for Items...now it can be auto-set. We added a Configuration that will set this system generated number into the Key field when creating a new item.
