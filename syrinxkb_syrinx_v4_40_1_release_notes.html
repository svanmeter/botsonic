Invoice Import Tool Can Now be Run as a Service - The Syrinx Purchase Invoice Import tool can now be run as a service so that invoice XML and invoice PDF files can be automatically imported into the system without users having to complete this process via the invoice import tool user interface.  This is enabled by setting an "Auto Invoice Threshold" value in the invoice import tool configuration.   This is the tolerable difference between Syrinx's estimated total value for the consumable or crosshire purchase invoice:

If the values on the supplier's invoice are within this threshold, the invoice will be imported and the purchase invoice values will be allocated proportionally to the nominal codes on the underlying receipt lines.  If the values are outside this threshold, an email will be sent to the account designated in the "Send Auto Failures to" field and the purchase invoice will need to be manually imported through the purchase invoice import tool or rejected.  The "Save Service Credentials" button is used to save the Syrinx logon details to be used by the service when connecting to the Syrinx database and must be pressed before starting the service for the 1st time or if those credentials change.  The tool must be run as Administrator in order to save these values and the service must be restarted to pick up any changes to these values.

Note – Specifics on this new functionality have been included in the Invoice Import Tool factsheet.

Import Purchase Invoice PDFs with Invoice Import Tool - Suppliers can now also include PDF copies of the purchase invoices included in their XML invoice import files and these files can then also be imported into the system and linked to the imported purchase invoice so that it can be viewed when pressing the "Documents" button against the purchase invoice in the Syrinx v4 client.  The associated XSL mapping file will require a new "InvoicePDFName" tag which translates to the appropriate XML field for the PDF filename in the supplier's invoice import file.  The tool will accept a PDF filename with or without the ".pdf" suffix. The PDF file linked to an invoice can be viewed by pressing the "Open PDF" button when viewing a purchase invoice record in the tool:

The PDF file will then be moved to the system "Scanned Documents" folder when the purchase invoice is imported.  The service component of the invoice import tool will also import any linked PDF files for any purchase invoices that successfully pass validation and are imported.

Note – Specifics on this new functionality have been included in the Invoice Import Tool factsheet.

Meter Readings from Driver App Mobile Delivery Jobs will Update Confirmed Contracts - Meter readings entered against items on Driver App mobile delivery jobs will now update hire item records on confirmed contracts.  Previously the meter reading from the Driver App would only update the contract if it was still unconfirmed.  If a meter value has been entered against the hire contract prior to confirmation, it will only be updated if the Driver App value is higher than the current reading.

Note - Items that charge excess usage based off of meter reading values should have this information set prior to confirming the contract as setting/updating the meter reading via the Driver App will not set excess usage values (e.g. "Basic Rate Includes", "Excess Rate", etc.) against the hire contract.  

Define Password for Access Dimensions Logon when Running as a Service - The Access Dimensions transfer tool has been updated to allow users to define a password for the Access Dimensions logon when the transfer tool is run as a service.  Previously the service was hardcoded to only use a logon that had a blank password or a password of "x".  This password value is set in the new "Service Password" field on the "Access Dimensions" tab of the transfer tool "Configuration Settings" screen:

Syrinx 365 Search and Filter Optimisations – When a user selects a filter for a search grid, the focus will now be shifted to that filter so that the user can begin searching instead of remaining on the list of available filters.  When a user begins typing into a specific filter option (e.g. “Is Equal To”, “Contains”, etc.) that filter option will be ticked as active and will be applied to the search grid.  

Ability to Define Map Provider in the Syrinx SalesApp - The SalesApp can now be updated to use a different regional google site for mapping postcodes.  To achieve this, a value will need to be patched against 'Map Web Site-1' internal database option.  For example, Australian databases have this value populated with 'www.google.com.au/maps/place/|FA|' by default so that the Syrinx v4 client will use google.com.au when pressing map buttons.  This change simply allows the SalesApp to also use this internal database option.

Retrieve Sites and Assets by Last Modified Date from SyrinxWebApi - A "ChangesSince" parameter has been added to the "Customer/Sites" and "Supplier/Sites" GET endpoints. This will allow for the retrieval of only records that have been updated/created since the supplied parameter date.  The existing "ChangesSince" parameter on the "Asset" GET endpoint has been updated to include the item's created date.  Each of those endpoints will also now output "dateModified" and "modifiedByUser" values.

New StockMovements Endpoint Added to SyrinxWebApi - A new StockMovements endpoint has been added to SyrinxWebApi which can be used to output the stock movement/adjustment records for stocked bulk hire and sales items.
