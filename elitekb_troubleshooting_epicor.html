Epicor Troubleshooting

 

To test the interface between the Point of Rental server and the Epicor Point of Sale client, use the Epicor 3simr.exe simulator located at C:\Program Files (x86)\E4W Rental Network Server on the Point of Rental server. Opening 3simr.exe displays the dialog window below.

 

 

Enter the IP Address of the Point of Rental server into the LocalIPAddress field, the IP Address of the Point of Sale Client into the RemoteIPAddress field, change the Object Type to Remote Object, and click bConnect2. If a connection is successful, the status label at the bottom of the window will show “True . . .”. Next, click the bConnectedtoE4WPOS button and a successful connection will show “True.”

 

If either test does not work, the interface from Point of Rental to Epicor Point of Sale will not work. 

 

Check the client PC Windows Task Manager for 3cclient.exe and 3posif.exe. If either or both are missing, then Epicor is not licensed for the Point of Rental interface or some other Epicor configuration must happen.

 

 

When the 3simr.exe simulator is tested, Server Message Monitors provide visibility for the connection status. Click the Show Server Messages item in the taskbar to view status:

 

 

The status window below displays a Connected to POS “No” status and there are components missing from the dialog. The Point of Rental to Epicor Point of Sale interface will fail if the message below appears. 

 

 

The status window below displays a Connect to POS “Yes” status and all components are present in the dialog window. The Point of Rental to Epicor Point of Sale interface will succeed if the message below appears.
