Release Notes (informational only)


In the past, we have kept the Expert and the Elite release notes in separate camps to avoid confusion.  However, we think it’s time that the fence comes down between The Sandlot and The Beast.  After all, we’re all on the same baseball..er…software team. Because Elite is the next evolution of Expert, the latest and greatest technology moving toward the future, we thought it would be beneficial for our Expert customers to be in the know.  As Elite features release and mature you may find functionality that you just can’t live without.  While new features in Expert are limited due to its older technology, fear not, Expert will continue to be supported.  Below are the release notes that our Elite customers received for this release; there is also a link to the Elite Release notes on the home page of this site if you are interested in learning more about past Elite releases.

 


 

Contract List from Sales Goal Widget:  Want to know what contracts make up the value portrayed on the graph?  Click on the bar to launch a modal to display a list of these contracts.  From this modal, the user can also open the contract, the customer, or the sales rep to get more detail or see the sales rep goals.

 

Delivery/Pickup Screen:  A slight tweak was made to the query that displays the delivery/pickup window from the contract to make the display quicker for those with large databases and lots of scheduled stops.

 

Importing Files on Cloud Servers:  The process of importing files onto a cloud server has also been amended for users on cloud-hosted servers.  Users will now select the ‘up’ arrow in the top bar of the screen.  A file selection box will appear.  Select the file for upload.  Once uploaded, the file will appear in the ‘Upload’ folder under the POR directory.  The subfolder will be labeled for the user that did the upload.

 

Mobile User Permissions:  In order to make the process of updating Gadmin permissions easier for our users, we have developed a microapp that launches from within the browser.  Security under the configuration section dictates whether or not a user can view or edit permissions.  The default is to have both turned off.  If either is turned on and the user is signed in using SSO credentials, they will be able to launch the mobile permissions option in browser configurations.  From here, the user can easily access/amend this information without having to navigate to a separate website.

 

 Day of the Week Inserted in Day at a Glance and Browser Grids:  By popular demand, the abbreviated day of the week (MON, TUES, WED, etc) has been inserted into the date column in the browser grids and day at a glance.  Stay tuned as we expand this into other core grids.

 

  

 

POR One Release Notes

These are updates for Elite users utilizing the POR One app.
 

Customer Records with large amount of Contracts: When viewing a Customer Record in POR One that has a large number of Contracts, the user will now be taken to the Contract view page so that all Contracts can be viewed. There was a limit of 100 previously but now there can be any number of  Contracts associated with the Customer record. 

Stop Details Line Items: This only affects users with Dispatch Center using Routes (and Stops) on One. The accordion for Line Items on the Stops Details page no longer links to the Inventory item. This was causing erroneous navigation when completing a Stop. If the Line Items need to be viewed, you can click on Contract and then navigate to the Inventory item. 

Bug fix for Labor Tracking: There was an issue where Labor Tracking tasks could not be completed. This was resolved and did not require any update to the system. If you experienced this then you can now submit any Labor Tracking tasks without issue. This includes tasks that failed previously. 

 


    

Memory Lane Monday

This is a fun corner where we republish release notes from previous sprints you may have forgotten about.

  

Modifications Turn Route Number Red: In the Dispatch Center, once a route has been saved the route number will turn red if a modification happens. Clicking on the number will display the changes that have taken place: if a contract was modified if setup time changed. Etc.


Call Logs in a Continuation Series: Any call log written on any contract in a continuation series is now visible on any contract in the series. This is useful so that when a user is in the ‘T’ contract they can go to the call logs tab and see a log from the ‘A’ contract.
