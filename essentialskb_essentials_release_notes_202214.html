10/17/2022 - Version 2022.14


  • Force Purchase Order# - For those of you with customers that require that you get a purchase order# from their employees when you rent to them, here it is. You can mark any customer to ‘Require Purchase Order’. From there it will require the PO# field be filled out before continuing on with a transaction.
 
  • Event Location on Copy Transaction Enhancement - That was a lot to type. We have made a change so that if you copy a transaction, it does not change the Event location on the original contract like it did before.
 
  • Notification Emails Qty text changed - When you got a notification based on a transaction, no matter if it was in reservation/order/received status, it said 'Qty Received' for the item(s). It now says ‘Qty’ in order to be more universal.
 
  • Custom Menu Search - If you are one of our customers using the storefront, and you are using the Custom Menu Setup (Why aren’t you?), we found that when you typed something in the search field, it would not bring that item up, although the ‘categories’ still worked. Now the search field works too.
 
  • Quick Sale Note - If you created a quick sale, then tried to add an item note, clicking ‘add note’ did not bring up the text box for you to type something in.. The link did look awesome though. Now you can add your note to your sale item.
 
  • Conflicting Overbook Report - We found an issue where putting in dates manually on a transaction prevented the conflicting overbook report to pull up, and that has been fixed.
 
  • Visual Improvements - We also made changes to a couple of your ‘views’ that makes for a better experience.
    • In the Inventory screen, we have enlarged the search field on the right, as well as made the magnifying glass larger to make it more visible. 
    • On the Dashboard the ‘search’ has the same enhancement as above, and also now does not have an ‘X’ until you type something, as it’s pretty hard to cancel nothing.
