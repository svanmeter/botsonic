Smart Equip Installation Guide

 

 

 

Overview

 

This guide is to help you install Smart Equip plug in for Point of Rental. 

 

Topics included in this article:

 

  • Configure Smart Equip
  • Configuring IIS
  • Create firewall exceptions
  • Set up production service
  • Set up development service
  • Testing Smart Equip

 

 

Configure Smart Equip

 

Please Note: Admin access is going to be required to complete these steps. 

 

Requirements:

 

  • Windows Server 2012R2 and Above (Note Windows Server 2012 is set to End Life in January 2023)
  • .Net 4.5 and all Current Updates from Windows
  • Internet Information Services (IIS) 
  • Smart Equip Installed

 

 

Configuring IIS

 

1. Go to your Start Menu and search for Internet Information Services (IIS) Manager.

 

 

2. Click on Application Pools

 

 

3. Check to see if “ASP .NET v4.0” and ASP .Net v4.0 Classic” are listed.

 

If not listed, verify to make sure that .Net 4.0/4.5 is installed. If it is installed and does not show up, it may be that it has not been registered. Open Command Prompt and type in the following line:

 

“CD C:\Windows\Microsoft.NET\Framework64\v4.0.30319\” then “aspnet_regiis.exe -I”

 

Please Note: The version number may change, for example: v4.0.30319.

 

 

If listed, click on them both and from the Actions Menu on the right click on ‘Advanced Settings…’ 

 

 

4. In the General area, set the Enable 32-Bit Applications to ‘True.’ Click OK when you are done.

 

 

 

Create firewall exceptions

 

Please Note: If you are running any other program besides Windows Defender, then you will need to go to that program's website on how to create exceptions.

 

1. Go to your Start Menu and search for Windows Defender Firewall with Advanced Security. 

 

 

2. From the left menu, click on ‘Inbound Rules.’

 

 

3. Then from the right menu, click on ‘New Rule.’

 

 

4. Choose Port and click Next. 

 

 

5. Choose TCP and Specific local ports 57368, Click Next

 

 

6. Select 'Allow the connection.'

 

 

7. Select Domain, Private, and Public. 

 

 

8. Name it the following:

 

Name: PORSmartEquip – Production 

Description: Point of Rental SmartEquip Production Port

 

 

9. Do the above process for port 57369 as well and call it the following:

 

Name: PORSmartEquip – Dev

Description: Point of Rental Software Development Port

 

 

 

Set up production service

 

1. Verify that Port 57368 rule was created on the Windows Firewall.

 

2. Go to <DataDrive>:\POR\ and make sure there is a directory called PORSmartEquip. If there is not, create one.

 

 

 

3. From Our PORSvr copy the Webservice Folder and place it in the <DataDrive>:\POR\PORSmartEquip directory. Please Note: Please see IT for the directory path for this folder.

 

4. Open <DataDrive>:\POR\PORSmartEquip\WebService\Web.config with notepad to customize the app settings 

 

 

5. Modify entry to: <add key=”Item Search” Value=”Num” /> Options are NUM for ItemFile NUM or KEY for ItemFile KEY.
 

*Note for Elite:  In the webconfig file if it is set to Lookup by NUM or System Set to AutoKey, Keys for Headers and Parts will use the item number (NUM) as value for the key when creating new parts. If Set to Lookup by Key or Part, and AutoKey = False, keys for headers and parts will use key, but use FLENG - (Store Code char count) - 1 - (3 digit Manf) constraints  to take the Right most characters off the PartNumber.  i.e. FLENG = 16, Store Code = 501, Manf = KUBOTA, PartNumber = 995865342107968754321  makes a store key of KUB968754321 -501. Header logic so the pneumonic aspect is intact, i.e. KUB968754321

 

6. Modify entry to: <add key”PartsCategory” value=”13” /> Set the default parts category for items added through SmartEquip. Determine this category by querying ItemCategory in the POR database – defaults to “Parts – Special Order” unless the customer specifies otherwise. SELECT [Name],[Category] FROM [ItemCategory]

 

7. Modify entry: <Add Key=”DBPath” value=”” /> Set Value= “<DataDrive>:\POR\Users\”

 

8. Go back to IIS Manager.

 

9. From the left menu, go to Sites.

 

 

10. From the right menu, go to “Add Website.” Enter in the following information:

 

 

Site Name: SmartEquip

Application Pool: ASP .Net v4.0

Path: <DataDrive>:\POR\PORSmartEquip\WebService

Type: HTTP

IP Address: <Server IPv4 Address>

Port: 57368

Pool ASP: 4.0/4.5

 

 

 

Set up development service

 

1. Verify that Port 57369 rule was created on the Windows Firewall.

 

2. Open the Webservice and call it WebService – Copy.

 

3. Open <DataDrive>\POR\PORSmartEquip\WebService - Copy\Web.config with notepad to customize the <App Settings>

 

 

4. <add key=”Item Search” Value=”Num” />

Options are NUM for ItemFile NUM or KEY for ItemFile KEY

*Note for Elite:  In the webconfig file if it is set to Lookup by NUM or System Set to AutoKey, Keys for Headers and Parts will use the item number (NUM) as value for the key when creating new parts. If Set to Lookup by Key or Part, and AutoKey = False, keys for headers and parts will use key, but use FLENG - (Store Code char count) - 1 - (3 digit Manf) constraints to take the Right most characters off the PartNumber.  i.e. FLENG = 16, Store Code = 501, Manf = KUBOTA, PartNumber = 995865342107968754321  makes a store key of KUB968754321 -501. Header logic so the pneumonic aspect is intact, i.e. KUB968754321
 

5. <add key”PartsCategory” value=”13” />

Sets the default parts category for items added through SmartEquip  

Determine this category by querying ItemCategory in the POR database – defaults to “Parts – Special Order” unless the customer specifies otherwise

SELECT [Name],[Category] FROM [ItemCategory]

 

6. IMPORTANT: This is for the Training Database Set the Following Key <Add Key=”DBPath” value=”” />

Set Value= “<DataDrive>:\POR\Users\train\”

 

7. Go back to IIS Manager.

 

8. From the left menu, go to Sites.

 

 

9. From the right menu, go to “Add Website.” Enter in the Following Information:

 

 

Enter the following information:

 

Site Name: SmartEquip

Application Pool: ASP .Net v4.0

Path: <DataDrive>:\POR\PORSmartEquip\WebService

Type: HTTP

IP Address: <Server IPv4 Address>

Port: 57369

Pool ASP: 4.0/4.5

 

 

 

Testing Smart Equip

 

To test Smart Equip, we recommend that you install Fiddler. To get the installer, go to https://www.telerik.com/download/fiddler

 

1. Run test on Development (SmartEquipDev – POR 57369).

 

 

2. Click the Composer button to begin the test 

 

 

3. Click on the Parsed Tab – The History section will show historically created test. 

 

 

4. The Point of Rental Test Server has sample tests that can be used on a customer system.

 

Login - Tests User Login – Requires customer user id and password with Smart Equip Role

Create Order - Creates purchase order and adds equipment to work order – requires customer user id with Smart Equip Role, Valid unit and valid Point of Rental transactions

Validate Order - Validates the existence of the rental unit and Point of Rental transactions

 

5. Search Criteria must be taken from the customer’s Point of Rental instance.

 

User ID and Password

Unit

Model

Manufacturer

Serial Number 

Item

Part Number

 

6. Double-click the functionality to test and the Parse Section will update to test criteria

 

7. Click the Execute button to test the criteria. 

 

 

8. The results will appear in the left-hand capture box. A result of 200 means success. 

 

 

 

9. Click on the right-hand Inspectors tab and XML tab to view the results detail. 

 

 

10. Log file is located at <DataDrive>:\POR\PORSmartEquip\Logs\
