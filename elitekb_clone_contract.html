Clone Contract



Cloning a contract allows you to make a copy of a contract and change it into a Quote, Reservation, or an Open contract. No changes are made to the original contract. Any Asset Sale items will not be copied to the cloned contract. Additionally, any serialized items will be replaced with the header item.


Topics in this article include

  • Getting there
  • Cloning a Contract 




Getting there

Here's how to get there
Expert: Program Menu > Counter System > Clone Contract
Elite: Transactions > Clone Contract

 

Cloning a contract

 

1. Search for the contract you want to clone. You can search by the number of the contract. You can enter SSC in the contract number field to search for closed contracts, or SSO to search for open contracts.

 

 

2. Choose the contract you want to clone. If the status of a contract is not bold, then it is a will-call contract.

 

3. Choose Yes when prompted “Is this the correct contract to clone?”

 

 

4. You will next be asked if you want to select a different customer. If the clone is for the same customer, choose No. If the clone is for a different customer, choose Yes.

 

 

5. If you select Yes, enter the name of the new customer in the Customer Selection screen.  

 

 

 

6. Once the customer is chosen, enter your password if prompted to do so.

 

7. If you select No, enter your password if prompted to do so.

 

8. You will then be required to enter the reservation pickup date. Select the appropriate date and time.

 

 
 

 

9. Next, enter the rental period.

 

 

10. When asked “Do you want to recompute the prices?”, you can choose Yes to recompute and No to not recompute. If you choose Yes, all item prices will be recomputed at the new rate(s). If you choose No, the prices will remain as they are.

 

 

11. You will continue through the remaining steps of the contract by clicking OK in the contract processing screen and moving to the payment screen.

 

Note 1: If there are any items that are not available when cloning, the overbook screen will appear to allow you to overbook the item. If any items are inactive, they will be added to the cloned contract. You will not be warned that they are inactive; however, the item will indicate that it is overbooked.  Also, if you have any items set up with critical levels, and you exceed those levels, a warning message will be displayed.

Note 2: If you are in Canada and this is a contract that was cloned before the PST tax law came into effect, you will want to check the PST option box at the top of the contract to bring in the PST taxes for each item. Otherwise, the PST tax will not be added. This is important since there is not a way to go back and add it once out of the cloning process.

Note 3: When cloned, the nontaxable flag of the items will be checked to ensure the items are set to the proper taxable/nontaxable status.
