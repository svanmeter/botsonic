Option to Create Separate Jobs from Multi-Item Customer Service Job in Bulk - A new "Raise Multiple Jobs" button has been added to the "Items" tab of multi-item customer service jobs:

When pressed, a new "Select Items For Job Creation" screen will be opened that will allow the user to either use the "Select All" button to enable the "Raise Job?" flag against all items or manually select the "Raise Job?" flag against specific items:

After selecting the items and pressing "OK" the system will create customer service jobs for each of the selected items.  The multi-item job will also now display the status of those linked jobs against the items on the multi-item job:
Option to Update Status Against Multiple Items - A new "Update Multiple Statuses" button has been added to the "Items" tab of multi-item customer service jobs:

When pressed, a new "Select Items For Status Update" screen will be opened that will allow the user to either use the "Select All" button to enable the "Selected?" flag against all items or manually select the "Selected?" flag against specific items:

After pressing "OK", the user can select a status to assign to all items via a new "Select which status to update items to" screen:

After pressing "OK", this status will be set against the selected items on the multi-item job:
Enter Timesheet - Ability to search for a Staff Member - Users are now able to search/select operated staff members by staff number when entering timesheets by ticking the "Staff No:" option on the "Enter Timesheet" screen:

When this option is ticked the operator staff member names are prefixed with the staff member's staff number.  Users can also begin entering the staff number in to the "Operator" field and the system will display the matching record.
Query Timesheet - Search on Staff Number - Users are now able to search/select operated staff members when querying timesheets by ticking the "Staff No:" option on the "Query Timesheet" screen:

When this option is ticked, the operator staff member names are prefixed with the staff member's staff number.  Users can also begin entering the staff number in to the "Operator" field and the system will display the matching record.
Query Timesheet Screen Will Now Display the Operator's Staff Number - The Query Timesheet screen will now display the "Staff No" value (staff number) that has been set on the operator's staff member record:
App Option for Item Level Notes to be General Text and not Skip Validation - A new "Use Item Notes for General Text?" option has been added to the "Android Apps" tab of Syrinx SP Configuration\System Options\General which controls whether or not the Driver App will validate that required values such as "Processed", "Meter Reading" or "Required Photos" have been set/actioned against an item on the mobile job if the user has entered "notes" against the item:

The previous behaviour was for the app to skip any item level validation if a user had entered notes against that item.  This new setting allows for this behaviour to be changed so that the entry of "notes" against the item will not skip any of the required item level validation.
 App Photo Option to Use Date/Time Photo was Taken Instead of Date/Time Synched - A new system option "Use Date/Time Taken for Photos" has been added to the "Android Apps" tab of Syrinx SP Configuration\System Options\General that will force the system to use the date/time that a photo was taken on the time stamp placed on the photo when it is archived from the database:

The previous behaviour was for the system to use the date/time that the photo was synched to the database on the timestamp and this can be different from the date/time the photo was actually taken if the user had low to no connectivity and had to synch the photo at a later time.
