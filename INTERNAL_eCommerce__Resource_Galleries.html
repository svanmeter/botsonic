The Resources page can be used for many purposes and help keep the navbar from becoming cluttered. You can use it for forms, policies, guides, employment, FAQ, coupons, reviews, etc.



Topics in the article include
  • From the customer side
  • Adding the page to the website
  • Adding other resources to the page

From the customer side

In POR, the customer must prepare PDF docs to populate the various sections on the Resources page on the website.  The PDFs must then be copied to specific folders on the customer’s POR server to upload to our server and be available for the website.

 

  1. On their server, they must locate their POR folder.  Depending on their setup, it could be on their C: drive or any other drive.
  2. The next folder to find is the webcontent folder.
  3. Next is the site001 folder.  And here’s where it gets tricky.  There are installations where there will also be a site002 folder, and maybe even a site003 folder.  We’ll stick to the simplest of situations here, and assume we only see the site001 folder and proceed from there.  We’ll cover the other situations below.*
  4. Next is the gallery folder.
  5. In the gallery folder will be two folders: webdoc and webphoto.  Webphoto is used for images for their photo gallery and images they upload for the slideshow, landing pages, etc.  We’ll cover that in another article.
  6. The webdoc folder is the one we’ll be working with for the Resources page.
  7. The sections of information that they want on their Resources page will determine the folders that need to be created in the webdoc folder.  Almost every customer wants at least one form on their site, so a forms folder is usually the first folder created.  Others can include guides, employment, policies, coupons, etc.

Once they have their folders created, they can copy their PDFs into the associated folders.

 

*if the store has multiple extractions from POR, they will have multiple folders, i.e. site001, site002, site003, etc. in their webcontent folder. In this case, you will need to know which extraction is which so you can be sure you're using the right folder.   {insert how to figure out the right folder for each site here}.
 

If the customer is running Mobile Workforce, is usually used the site001 folder.  If they have only one extraction from POR, then site002 would be the folder you'd want; otherwise you'll use site002, site003, site004, etc.
 

FYI, multiple extractions could be a party website and an equipment website for the same store, or multiple locations each with their own website.



 

Adding the page to the website

Use our one-page listing, rental-resources.asp.

This file is a simple plug & play but does need revisions to customize the page to the customer’s requests.

  • check Title Tag and <metaDescription>
  • update the folder sections in the code to match the setup for the site you’re working on:
    • update the folder name(s) according to the folders that are uploading from the customer’s server
    • update the H2 for the headings on the page to match the folders
 

 

Adding other resources to the page

  • If a store has a policies, FAQ, or employment pages, they may want to add the link(s) to this page instead of including link(s) in their menu navbar.   In this case, you may simply add another H2 tag and hard code the link to the other landing pages.
  • You can also include scripts on this page, such as for newsletter signups, review generators, employment listings, etc.
  • Images can be added or uploaded from gallery folders as well, for coupons or other purposes.
  • Some customers like to include Guides and Instructions.  These can turn into very long lists of docs.  These are docs that can be attached to item records as well.  So there are two schools of thought on this.  One is that having the docs attached to items may make them easier to find.  Another is that stores like to tell people to go to the Resources page on their website and think that will be simpler for customers than having to find their item on the website.  In the end, it's the customer's choice
