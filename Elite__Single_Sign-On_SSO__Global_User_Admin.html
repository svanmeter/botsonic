Elite | Single Sign-On (SSO) & Mobile Permission



 

This article will cover Single Sign On setup and Mobile Permissions (preferred method over Global User Admin). Single Sign-On allows you to have one login ID and password to access different applications within the Point of Rental ecosystem. For instance, you would use Single Sign-On (SSO) credentials to log in to the Elite and POR One mobile app. This makes remembering and updating your password easier.
 

In Elite, you are required to log in with SSO credentials to see and use certain features that interact with the Point of Rental global API. Logging into Elite using Single Sign-On (SSO) should be the default process to ensure access to all features.

Setting up SSO requires the company to have a Global Organization created. This is an automated process that is done when a system is implemented.



Topics in this article include
 

  • Setting Up an Operator to Log In Using Single Sign-On (SSO)

  • Single Sign-On (SSO) vs Local Credentials

  • Creating/Resetting Your Single Sign-On (SSO) Password

  • GAdmin (Global User Admin)



 

Setting Up an Operator to Log In Using Single Sign-On (SSO)


An Operator must have a unique email address in the Operator record. This is located in the Security tab of the Operator record in the Elite browser interface. Users cannot share the same email address. Emails MUST be unique!

 


 

Single Sign-On (SSO) vs Local Credentials


In the Operator record, there is still a requirement to set up Local Credentials. These are still tied to Password Role Level for many security settings on the Core side of the program, for example, to set contract pricing. They do not have the same length and complexity requirements as SSO credentials.

 


 

Creating/Resetting Your Single Sign-On (SSO) Password


Once an Operator has this in their record, they can go to the Login screen for Elite.
 

1. They will enter their email address, and then select Forgot Password to trigger the setting of their SSO password.
 

 

2. This will display the Send Reset Code feature. When this option is selected, an email with a verification code is sent to that email address.

 

 

3. The Operator should check their email and they will find an email with a six-digit verification code. Enter this code, enter a password (must be at least 8 digits long), and select Confirm to proceed.

 

 

4. A success message will be displayed. Selecting Login will log the user into the application.
 

 

5. A user can follow these Forgot Password steps anytime they want to reset their password and in those cases when they forget it.

 
 

 Mobile Permissions


Mobile Permissions is a feature that was released on v2023.08 that allows you to access permissions to launch the mobile permissions microapp.  This is also under the system section of configurations in the browser. that is an alternative way 

The ability of a user to view/edit can be set under the configurations section of security.  If the user does not have access here, they will not see this option in the browser configuration.  These security settings will also correspond to the actions the user can take in the micro app.  Users must be using SSO credentials to log on.

When the micro-app is launched, it will open a new dashboard for the user to view/edit.





Setting Security Permissions with the POR One App and Global User Admin

1. To access these permissions, an administrator must go to Configuration Gear > System Global User Admin. This is located in the middle part of the screen.

 

 

2. This will open your default web browser window where admin credentials can be used to log in.

 

 

3.  On the User List Page, this is where you can set permissions for parts of the POR One Mobile App. You can also give other employees access to View and Edit Permissions from here as well. To give them access click the check box under the appropriate column.

 


Viewing or Editing Permissions
 

Edit Permissions - A user with this option selected can update permissions for other users.

View Permissions - A user with this option selected can log in and view the settings but cannot make any changes.
