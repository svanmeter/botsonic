Program Operation

 

Simultaneous Use of The Point-of-Rental™ Programs

 

The Counter System program has been designed to allow its use at two or more terminals simultaneously.  However, because the terminals share the same disk files there are certain situations that should be avoided:

1.   Do not attempt to access the same contract from two terminals simultaneously.

2.   Do not attempt to access the same customer from two terminals simultaneously.

3.   All terminals at the store must have previously exited from the Counter System Menu and Quit from the Program Menu to perform end-of-day processing. These terminals must not be used to run any POINT-OF-RENTAL™ programs until after the END OF DAY PROCESSING program has completed.

 

Notes Concerning Program Operation

 

1.   Money values may be entered in pennies or a decimal. For example, to enter five dollars and thirty nine cents you can type 539 or 5.39

2.   When you are asked to select from a range of values, the allowed range will be shown in parentheses followed by a ‘default’ value. If you press the ENTER key, the “default” value will be used. For example, let’s assume someone is buying a 12” concrete blade and you have reached the point where quantity must be entered.

            Enter quantity (1-14) 1 :

If you press ENTER then quantity 1 would be used. If more than 1 is desired, you must type the desired quantity followed by ENTER.

 

3.   Most questions that require a “yes” or “no” response may be answered by clicking on Yes or No, typing the letter “y” or the letter “n”, or using TAB to select the appropriate option and pressing the ENTER key.

4.   When you are typing information in, corrections may be made before you press the ENTER key using the BACKSPACE key to position the cursor back to the place where the mistake was made. For example, if you were typing the name Johnson but you  spelled it Johnxon, you would press BACKSPACE key three times to get back to the letter x, then you  would type “son”.

5.   Upper and lower case letters are considered to be the same by the program. For example, if you have previously defined a customer’s search key to be Johnson, that customer will be found if you later enter JOHNSON as the search key.

6.   The following rules apply when you are entering data or modifying data for a record (Customer data, Item data, etc).

a.  The TAB key advances you to the next field of the record. If no data is to be entered for the field, or if the data in the field is not to be changed, simply press TAB. The ENTER key can also be used, but remember that pressing ENTER on a button will be the same as clicking the button.

b.  For fields that are defined as numeric only, your entry will not be accepted if you enter any non-numeric data in the field. The exceptions to this are that spaces and the minus sign (-) will be accepted.

c.  To reverse the direction that fields are entered, hold the SHIFT key while pressing the TAB key.

d.  To duplicate the first field into the current field, press the  = key. If the current field is smaller than the first field the rightmost characters will be dropped.

e.   When you enter customer data, you may place a question mark “?” in any non-numeric field to indicate missing information. This will cause Modify or Close Contract to display the customer data and allow the operator to insert the missing information.


7.   A ‘search key’ is a sequence of letters, numbers or special characters that are used to find a unique record within a data file. For example, a customer’s telephone number may be the search key that is used to find a customer’s data record. This search key must be unique within the data file in order that

The program will know exactly which record to look for. Whenever you’re trying to find a particular customer, item contract, etc. You will be asked to enter the search key. Some of the data files may allow more than one search key For example, you can search for a customer by phone number, name, or driver’s license number. In any event, whenever you are required to enter a search key, the program will first ask for the primary search key. If you only press the ENTER key or TAB key in response, the program will move to the next search key.  If you enter NEXT when a search key is requested, the program will attempt to find the search key following the previous search key that was entered. If you enter SAME, the previous search key will appear. If you enter MORE when a search key is requested, the program will show the next screen of records in a MORE screen.


8.   The ‘status’ of a contract is printed as the first line on the contract form. The possible statuses are:

            QUOTE – A quote that has been generated for a customer.

OPEN – At least on a item is currently rented and has not been returned.

            RESERVATION -  At least one item has been reserved for a future date/time and has not yet been picked up/sent out.

            CLOSED – All rented items have been returned but the total contract amount has not yet been  paid or a deposit is still held.

            COMPLETED – All rented items have been returned and the total contract amount has been paid

            ADJUSTIMENT – An Extra Charge or refund was made to a contract that is no longer on file in the computer.

            WORK ORDER -  A contract to complete repairs on a customer’s equipment.

            MAINTENANCE REQUIRED – Items on a contract that have been scheduled or placed for maintenance. This will not take the item out of inventory.

            SUB RENT OPEN – Items on a contract that have been overbooked and have been sub-rented from another company.


9.   The following special item and customer records have been pre-defined in your system. However, the store owner may have chosen different keys for some of them:

            Customer Key: SALE

            This is used as the customer key for TAXABLE CASH SALE ONLY.

            Customer Key: NOTAX

  This is used as the customer key for NON-TAXABLE CASH SALES ONLY.

This customer is used to take broken items out of inventory and to charge cost to the item’s Repair Cost. There can be several REPAIRS customers. They all just have to begin with REPAIRS in the key.

Customer Key: QUOTE

This customer is used to write a rental quote if you do not wish to use a particular customer.

Item Key: MR

This is used only for MISCELLANEOUS RENTAL ITEMS that are not in the price list.

Item Key: MS

This is used only for MISCELLANEOUS SALES ITEMS that are not in the price list. The program will ask for the sales items name

Item Key: BALANCE

This is used to enter previous account balances for charge account customers. NEVER ATTEMPT TO SEE THIS ITEM as it is for use only by the storeowner in initially setting up account balances.

Item Key: DEFAULT

This is called the default items record. It should only be changed by the store owner. The purpose of this record is to set up the initial values for adding new item to the inventory.

11   Each employee should be assigned an Operator Id Code.  These codes consist of any 25 characters on the keyboard.  For example, 2P is a valid id code.  It must be entered to gain access to the counter system functions.  The counter system program will enter name of operator to the top of the contract.  This name will be printed at the top of each contract, and will also appear in the transaction report at the End of Day Processing.  This enables you to associate any action at the counter with a particular counter person.  You must also define the operator’s name that corresponds to the ID code.

12.  Your computer contains a day/date clock that has a battery backup.  Normally, it will always know the correct time and date.  However, if you need to manually change the time and/or date, click on the time in the bottom right hand corner of the server screen.  Change the date and time accordingly.

13.  U-Haul or Ryder Commissions

The following procedure will allow you to run all U-HAUL revenue through your cash drawer but only include commissions in your total sales.

A.   If you have not already done so, add the following:

•  A disbursement item for “U-HAUL Money In.”  Maximum price negative (-5000.00).

•  A disbursement item for “U-HAUL Money Out.” Maximum price positive (5000.00).

•  A customer for U-HAUL transactions, Cash, Sales Only, tax exempt.  Sample key: UHAUL.

•  A miscellaneous sales item for U-HAUL transactions, type s.  Sample key: UHAUL.  Selling price blank.

B.   When money comes IN from U-HAUL customers, issue a cash disbursement to “U-Haul Money In”.  Enter as money added to the drawer.

C.   When money goes OUT to U-HAUL customers, issue a cash disbursement to “U-Haul Money Out”.  Enter as money removed from the drawer.

D.   When a commission check comes IN from U-HAUL Corporation, open a contract with customer key UHAUL.  Sell item key UHAUL, quantity 1, for the total amount of the commission.  Payment method check.

E.   If applicable, issue cash disbursements for any expenses that were deducted from the commission check.

F.   Item key UHAUL and customer key UHAUL will both reflect UHAUL commissions for income.

 

14.  How to Copy Pictures Into Item Record

Within the item record you have the option of displaying a picture of the item. You can get the pictures from manufacturer’s web sites or from a disc of pictures. The web site pictures are the best to use as they are sized and formatted to work with fewer problems.

15.  In situation where the extended price of a line item is a 1.5 cents, the program will round using the rules set based on bankers’ rounding.

Even cents round down and Odd cents round up.

Example, +.235 becomes +.24, as does +.245; while −.235 becomes −.24, as does −.245.

This method treats positive and negative values symmetrically, and is therefore free of sign bias. More importantly, for reasonable distributions of y values, the expected (average) value of the rounded numbers is the same as that of the original numbers. However, this rule will introduce a towards-zero bias for even numbers (including zero), and a towards-infinity bias for odd ones.
