Automatic Reports

 


This article will discuss Automatic Reports. Automatic Reports allow the user to select any report to print in selected intervals.

 

Topics in this article include
 

  • How to get there
  • How to set up and add Automatic Reports
  • Interval definitions
  • On-demand reports

 


 

How to get there
 

Expert: Program Menu > Configurations > Reports and Notifications > Automatic Reports

Elite: Configuration Gear > App Configurations > Reports and Notifications > Automatic Reports

 

 

How to set up and add Automatic Reports
 

Before we begin, in order to use this module to its fullest potential, you will need to also have the Mass Fax/Email Module to generate automatic reports.

 

  1. By selecting one of these reports or clicking the ‘Add’ button. 


 

  1. Enter the Description, store, and file name. If you're creating an Automatic Report for a custom report, check the Custom checkbox. 
  • Description - This is the name of the report.
  • Store - This is the location where the report is generated for.
  • File Name - This is the name of the file in Crystal Reports.
  • Custom - If you have created this report in Crystal Reports, then you would check this box.
 
  1. In this step, you're going to specify the method of how the report will be sent. You'll see the option for Print, File Save, Email, or Fax. You are not limited to using only one. For each method, specify the interval by using the drop-down (next to the method name). The available intervals are: Never, Daily, Weekly, Monthly, Quarterly, Yearly, twice a Month, and On Demand. If you select On Demand additional setup is needed. To learn more, see topic On-Demand reports.

 
  1. Printer - Select a printer where this will print.
  2. File Name - In most cases, you can leave this field blank; the file will save in your <DataDrive>:\por\End of Period\ folder. You can define the save path and the file type by including the file name with an appropriate extension, for example, <DataDrive>:\Shared\Documents (Red = path, Blue = Report name, Yellow = file type). When defining a path, you must also define the file name and extension.
  • Examples of File Formats: pdf, csv, xls, htm, xml, doc, rtf, rpt, tab, txt, iif. Google any extension for more information on the file type.
  • Important things to know
    • Filename with no extension will default to pdf.
    • Filename.pdf (forces the file type to save as)
    • If you wish to save the report without the report number and date in the file name, enter a pipe symbol between the report name and the format you wish to save it in, for example, AccountSummary|pdf
  1. Email Address - Enter the email address where you want the report emailed. You can enter more than one email by separating the email addresses with a semi-colon. This option is only available if you have the Mass/Fax/Email Module.
  2. Fax Number - Enter the fax number where you want the report faxed. This option is only available if you have the Mass/Fax/Email Module.​​
 
  1. Use the Test Report option in the top toolbar to test your report. Please note, you cannot test a report using 'daybatch' as a criterion. Please see Automatic Reports Criteria for more information on reporting criteria.




 

Interval Definitions

 

Daily

The report will run when you perform End of Day.

Weekly

The report will run at the end of the week after you perform End of Day. End of Week is determined by your Calendar Maintenance.

Monthly

The report will run at the end of the month after you have performed Month End. Month End is determined by your Calendar Maintenance.

Quarterly

The report will run at the End of the Quarter after the End of Month is performed. The end of the Quarter is defined by your Calendar Maintenance.

Yearly

The report will run after the End of the Year is performed. The end of the year is defined by your System Parameters and Calendar Maintenance.

Twice a Month

The report will run on the first of the month and the fifteenth of the month after End of Day is performed.

On-Demand

This will run at a set time that you specify in the Point of Rental Task Runner.


 


 

On-Demand reports
 

For a report to be run ‘on-demand,’ a task must be set up in the task runner. When the task is run, it will find all of the reports that have been set up for ‘on-demand’ and take the appropriate action (print, email, save, or fax).


Here's how to get there

Expert: Configuration > System Configurations > Configure Task Runner > Add Task

Elite: Configuration Gear > App Configurations > System Configurations > Configure Task Runner > Add Task

 

Here is an example of the task with the appropriate setup. Copy the below information. The listed drive may be different than shown in the image. 


 

Name: Report

Description: Report

File to Run: <DataDrive>:\PORV2022LIVE\Program\PrintoutEOMReports.exe

  • It should always trigger the program PrintEOMReports.exe and start in your Users folder. 

Start in Path: <DataDrive>:\PORV2022LIVE\Users\

Command Arguments: The command arguments should be broken up as follows: 

  • Blue - The first 3 numbers represent the store number needing to run the On-Demand reports. For store 1 enter "001", for store 2 enter "002", etc... 
  • Red -  Enter 128 (this denotes ‘on-demand’ and will never change)
  • Purple -  This number represents the batch number. Leaving it at zero will not look at a specific batch. For example, ‘001,128,0.0’
