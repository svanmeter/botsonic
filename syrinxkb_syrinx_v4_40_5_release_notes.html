Option to Assign Failed Fleet Inspection Re-test Jobs to Workshop App User - When a fleet inspection job is failed via the workshop app, the system will now add an entry for the re-test job to the unallocated area of the workshop scheduler and give the user the opportunity to assign that job to a workshop app user for completion.  The original mobile job will now be linked to the failed inspection job record that the system creates to record the failed inspection results.  These failed jobs have the same job number as the original job but are suffixed with "/F".  Previously, the only way to complete the re-inspection after a failed inspection test was to “reopen” the failed mobile inspection job.

Disable Automatic Creation of Pre-Delivery Inspections - A new system option "Manual PDI Only?" has been added to the "Inspections" tab of Syrinx SP Configuration\System Options\Workshop.  If this option is enabled, the system will no longer automatically raise Pre-Delivery inspection tests linked to an item when an offhire inspection test for that item is confirmed as complete and users will have to manually raise PDI inspections via "Enter Inspection", the "Hire Stock List" or the Workshop App as and when they are required.  If there are already inspection tests open for an item the manual PDI inspection can only be raised from the "Hire Stock List" or the Workshop App.

Driver Scheduler Option to not Restrict Scheduling of Items Under Repair/Inspection - A new system option, "Don't Restrict Item Allocation?", has been added to the "Driver Scheduler" tab of Syrinx SP Configuration\System Options\General.  When this option is enabled the system will not restrict the scheduling of items under repair or inspection even though the system has been set to not allow confirmation of hire contracts if the contract contains an item under repair or inspection.  Previously, this behaviour had to be enabled via a patch.

Add Support to Task Scheduler For New Trackunit API ISO 15143-3 AEMP 2.0 Pagination Method - The Task Scheduler will now support AEMP providers where pagination is handled with the page numbers forming part of the fleet endpoint URL (e.g. https://iris.trackunit.com/public/api/aemp/v2/15143/-3/Fleet/1, https://iris.trackunit.com/public/api/aemp/v2/15143/-3/Fleet/2, etc.) and the payload is output in XML format.  The scheduler was previously only handling this method of pagination if the payload was in JSON format.

Version 1.2 of SyrinxWebApi - A new version of SyrinxWebApi has been released.  This version contains the following changes:

All POST endpoints: These will now output the internal IDs of the records that they create/update in a success message and also advise on any records in the payload that were not created/updated due to errors.

POST Customer/Sites: Allows update by ADR_ID rather than relying on a matching site name.

POST Customer: Allows update by CST_ID rather than matching by account number. There is also a new requirement for “company type” to be specified for new customers.

DriverJobs/Remove: Now DELETE rather than POST.

Contract/UpdateContractOrder: Now PATCH rather than POST.

ServiceReminders/UpdateServiceReminder: Now PATCH rather than POST.

StockLevels/AdjustStockLevel: Now PATCH rather than POST.

​*This version contains changes that can break 3rd party integrations that are interfacing with previous versions of SyrinxWebAPI.  Live systems should not be updated to this version until any 3rd parties have had the opportunity to update their solutions to accommodate the changes in this new version.
