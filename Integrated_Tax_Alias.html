Integrated Tax Alias

 

This feature is designed to work with integrated tax services.  Currently, this only works with the CCH integration.

 

Tax definitions that are received from the tax service API response are saved to a table.  These tax definitions are then displayed in the Configuration feature.

 

The configuration feature controls two primary functions:

 

  1. Which tax definitions will print on the contract
  2. What will print on the contract

 

Controlling if a definition will print on a contract

When that option is not selected.  Individual definitions can be selected to print on the contracts.

 

 

Controlling what will print on the contract

 

The format for what prints on the contract is ‘Definition - Tax Amount’ in a single horizontal line.  Since many tax definitions can be lengthy, the configuration has the option to set an Alias.
